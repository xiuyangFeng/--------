> **状态**：2026-06-19 已完成 lazy/eager 审核；修复 2 处小问题后通过，可用于 PINN evaluate 补跑。  
> **背景**：Job **5739** OOM（merged train pkl ~100GB 全驻内存）；离线 evaluate 曾耗时 ~11 min（连带加载 train+val+test ~141GB）。
> **执行更新**：PINN evaluate Job **5750** 已完成；非 PINN lazy 重训 Job **5751** 已提交运行（`partial_cache_cases=4`，Slurm `96G/72h`）。

## 问题

| 环节 | 原行为 | 代价 |
| --- | --- | --- |
| `build_datasets()` | 固定加载 train/val/test 三个 merged pkl | evaluate test 也读 **~141GB** |
| `CrownDataset` | merged pkl 全 list 驻 RAM | train **~100GB** → OOM |
| `evaluate_model()` | batch 内逐 sample `unsqueeze(0)` forward | GPU 利用率低 |

## 改动摘要

### 1. Lazy partial 加载（`data.py`）

- 新增 `CrownLazyDataset`：索引来自 `audit/preprocess_cases.jsonl`，按帧读 `pkl/partial/{case}_{filter}.pkl`。
- `_PartialRecordCache`：默认缓存 **2** 个病例 shard（`partial_cache_cases`），内存 ~1–3GB 量级而非 100GB。
- `build_datasets(config, splits=..., lazy_load=...)`：
  - `splits`：仅构建所需 split（evaluate 只建 `test`）。
  - `lazy_load`：config `data.lazy_load=true` 或检测到 jsonl+partial 时自动启用。
- merged pkl 路径保留（`lazy_load=false` / `--eager-load`），供 audit 或小数据对照。

### 2. 评估加速（`evaluate.py`）

- `evaluate_checkpoint(..., splits=(split_name,))` — 不再加载 train/val merged pkl。
- `_prepare_eval_minibatch` — 与训练一致的 **batch GPU forward**（`eval_batch_size` 可配）。
- CLI：`--eager-load` 强制 merged pkl（回归对照）。
- **评估模式（2026-06-21）**：
  - 默认 `--eval-mode paper`：对齐 `model_test.py` 全点推理（GPU 内 `--forward-chunk-size` 分块）+ `model_error_analysis.py` test 全局 GT min/max 作 NMAE 分母 + 逐样本日志。
  - 旧行为：`--eval-mode subsample`（随机 10000 点 / batch）。
- **进度日志**：分阶段 + paper 模式逐样本 / subsample 逐 batch；`--log-every N`。

### 3. 配置（`configs/local/crown_original_vp*.json`）

```json
"lazy_load": true,
"partial_cache_cases": 2,
"eval_batch_size": 8,
"num_workers": 0
```

PINN 配置同步；`batch_size` 保持 2。
