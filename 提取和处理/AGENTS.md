# [AGENTS.md](http://AGENTS.md)

## 环境

- **测试环境**：Conda，环境名为 `rag_venv`。运行脚本、安装依赖或执行测试前请先激活：`conda activate rag_venv`。

## 项目规则

- **本项目不允许使用任何软件开发类 skills**：Agent 在本项目中不得调用或依赖任何与软件开发相关的 skill（如 brainstorming、TDD、code review、writing-plans、executing-plans、git worktrees、finishing-a-development-branch 等），仅使用基础工具与代码库上下文完成任务。
- `docs/` 目录中包含了实验的项目文档与核心思路，在有需要的时候可以按需加载进行理解。如果代码发生修改，必须同时检查一下 `docs/` 里面相关文档是否需要同步更新。
- 任何会修改代码、脚本、配置或实验文档的任务，完成后都必须同步更新 `docs/02-推进与变更/代码修改与实验推进记录.md`。
- **推进记录书写顺序**：每次新增一条记录时，必须把**整条新记录插入到文件最上方**（紧贴文件开头、位于所有既有内容之前），采用「越新越靠上」；**禁止**仅在文末或文档底部追加。若文档有固定的“说明/目录”等前置块且约定保留在最顶，则新记录插在**该前置块之后、第一条历史记录之前**——如无此前置约定，则默认插在文件最顶端。
- 只有在 `docs/02-推进与变更/代码修改与实验推进记录.md` 完成对应更新后，才允许执行 `git commit` 和 `git push`。
- 所有 git 提交信息必须使用**中文**书写，且**完整、详细**：应清楚说明本次提交修改了哪些内容、解决了什么问题或完成了哪项任务，避免使用模糊或过短的描述（如仅写“更新”“修复”等）。
- 每次更新 `docs/02-推进与变更/代码修改与实验推进记录.md` 时，至少包含以下四部分：
  - 本次主要修改
  - 对应代码/文档
  - 推进到实验步骤
  - 当前状态判断
- 如果本次任务没有产生代码或文档改动，应在响应中明确说明“无需更新推进记录”，不能默认跳过说明。
