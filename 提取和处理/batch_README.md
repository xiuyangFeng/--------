# 显式几何特征工程 - 批量提取工具

本项目用于批量处理血管三维模型，自动提取中心线并计算显式几何特征（如曲率、扭率、径向距离等），并将这些特征映射到对应的 CFD（计算流体力学）点云数据上，为后续的图神经网络（GNN）训练提供丰富的几何先验信息。

## 🚀 功能特点

- **自动化中心线提取**：基于 `vmtk` 自动提取血管中心线。
- **丰富的几何特征**：
  - **Abscissa**: 沿中心线的归一化路径距离（0=入口，1=出口）。
  - **NormRadius**: 归一化径向距离（0=中心线，1=血管壁）。
  - **Curvature**: 中心线曲率。
  - **Tangent**: 中心线切向量 (Tx, Ty, Tz)。
- **批量处理**：支持遍历目录，自动处理多个病例。
- **特征映射**：利用 KD-Tree 高效地将中心线特征映射到体素/点云数据上。

## 🛠️ 环境依赖

本项目依赖以下 Python 库，请确保已安装：

- `vmtk` (Vascular Modeling Toolkit)
- `vtk`
- `numpy`
- `pandas`

> 注意：`vmtk` 通常需要通过 Conda 安装。

## 📂 数据准备指南

为了使用批量处理脚本，请按照以下结构组织您的数据：

### 目录结构

```text
data_root/                  <-- 输入数据根目录
├── Case001/                <-- 病例文件夹
│   ├── surface.stl         <-- [必需] 血管表面模型 (.stl 或 .vtp)，每个病例只能有一个
│   ├── time_0.csv          <-- [必需] CFD点云数据 (.csv 或 .npy)，可以有多个
│   ├── time_1.csv
│   └── ...
├── Case002/
│   ├── model.vtp
│   └── flow_data.npy
└── ...
```

### 文件格式说明

1.  **表面模型 (.stl / .vtp)**:
    *   必须是封闭或开放的血管表面网格。
    *   用于提取中心线和计算血管半径。

2.  **点云数据 (.csv / .npy)**:
    *   **CSV 格式**: 必须包含 `x`, `y`, `z` 列（坐标）。其他列（如速度 `u`, `v`, `w`，压力 `p`）会被保留并合并到输出文件中。
    *   **NPY 格式**: Numpy 数组，前 3 列必须是 `x, y, z` 坐标。

## 💻 使用方法

使用 `batch_process.py` 脚本进行批量处理。

### 命令行参数

- `--input_dir`: 输入数据根目录的路径。
- `--output_dir`: 结果输出目录的路径（脚本会自动创建）。

### 运行示例

```bash
python batch_process.py --input_dir ./my_data --output_dir ./processed_data
```

### 输出结果

处理完成后，`output_dir` 中将生成与输入对应的目录结构：

```text
processed_data/
├── Case001/
│   ├── result_features_time_0.csv  <-- 包含了几何特征的新文件
│   ├── result_features_time_1.csv
│   └── ...
└── ...
```

**输出文件 (CSV) 列说明**:

| 列名 | 说明 |
| :--- | :--- |
| `x`, `y`, `z` | 原始点云坐标 |
| `Abscissa` | 归一化沿程距离 (0.0 ~ 1.0) |
| `NormRadius` | 归一化径向距离 (0.0 = 中心, 1.0 = 壁面) |
| `Curvature` | 对应中心线位置的曲率 |
| `Tangent_X/Y/Z` | 对应中心线位置的切向量 |
| *(其他原始列)* | 如 `u`, `v`, `w`, `p` 等原始物理量 |

## 📝 开发说明

- **核心算法**: `vmtk_core.py` 包含了基于 VMTK 和 Numpy 的几何计算核心逻辑。
- **单例脚本**: `Script_Scenario_B_Volumetric.py` 提供了处理单个病例的函数接口。
- **批量脚本**: `batch_process.py` 负责文件遍历和任务调度。
