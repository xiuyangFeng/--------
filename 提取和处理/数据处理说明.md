# 数据处理说明（编号映射 + 点云合并）

本脚本 `prepare_data.py` 将原有的 `rename_stl.py` 和 `rename_ascii.py` 功能合并，并新增将 STL 放入点云编号目录的步骤，整体流程如下：

## 准备
- 目录结构假设：
  - `stl_data/`：原始或已编号的 STL 文件。
  - `点云/`：按人员姓名或编号的点云目录，每个目录下可有 `ascii/` 子目录及其他文件。
- 依赖：`pandas`。

## 脚本配置（`prepare_data.py` 顶部）
- `DRY_RUN`：预演模式，默认 True，只打印计划，不改动文件。
- `RUN_STL_RENAME`：是否对 `stl_data` 进行编号重命名并生成映射（初次处理姓名版 STL 时设为 True，当前数据已编号可设为 False）。
- `KEEP_STL_SUFFIX`：重命名 STL 时是否保留后缀（如 `-all`）。
- `RUN_POINT_RENAME`：是否重命名点云目录和 `ascii` 内文件。
- `RUN_MERGE_STL`：是否将 STL 拷贝/移动到点云对应编号目录。
- `MOVE_STL`：合并时是否移动（True）或复制（False）。

## 运行步骤
1) 预演（推荐）  
   在脚本内保持 `DRY_RUN = True`，运行：  
   ```bash
   python prepare_data.py
   ```  
   检查输出的“文件/目录重命名计划”和“STL 复制/移动计划”是否符合预期。

2) 实际执行  
   确认无误后，将 `DRY_RUN = False`，根据需要调整其他开关，再运行同样的命令。

## 操作细节
- 映射：若 `stl_data/id_mapping.csv` 已存在则直接读取；否则在 `RUN_STL_RENAME=True` 时根据 STL 文件生成映射并重命名。
- 点云重命名：将人员目录重命名为编号（如 `CHEN_JING_RU` -> `004`），`ascii` 内文件改为 `编号-时间步` 格式，保留后缀数字。
- 合并 STL：遍历 `stl_data/*.stl`，按编号（文件名前缀数字）拷贝/移动到 `点云/编号/` 下；若目标已存在则跳过并提示。

## 常见问题
- 映射缺失：若提示缺少 `id_mapping.csv`，请设置 `RUN_STL_RENAME=True` 并确保 `stl_data` 内文件名包含可解析的姓名。
- 目标已存在：当目标目录或文件已存在且内容不同，会终止操作，请先确认并手动处理。
