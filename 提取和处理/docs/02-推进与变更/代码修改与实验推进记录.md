## 2026-03-19 第 8 次记录

### 本次主要修改

- 针对集群训练时出现的主机内存不足问题，在训练数据集入口增加“运行时最小字段装载”逻辑：
  - `MLP` 仅保留 `x / y / global_cond`
  - 图模型仅保留 `x / y / global_cond / edge_index`
  - 对图文件中训练不会访问的冗余属性，在进入 DataLoader 前直接裁掉，减少 batch 拼接、主机内存驻留和主机到设备的数据搬运
- 预加载模式下同步只缓存裁剪后的样本，避免 `preload=true` 时把无关字段整份留在内存中
- 特征掩码改为原地乘法，减少每个 batch 额外张量分配
- 训练循环补充两项轻量内存优化：
  - `optimizer.zero_grad(set_to_none=True)`，降低梯度缓存占用
  - 每个 batch 结束后更早释放 `batch / pred / breakdown` 等临时对象，并降低 `tqdm` 刷新频率，减少长时间训练的资源抖动
- 同步在训练说明和任务 A 配置说明中补充“运行时最小字段装载”的说明，明确这是代码级内存优化，不改变实验语义

### 对应代码/文档

- `training/core/data.py`（按模型裁剪运行时所需字段、预加载缓存瘦身、特征 mask 原地应用）
- `training/scripts/train_field.py`（训练入口为 train/val/test dataset 注入所需字段集合）
- `training/core/trainer.py`（`zero_grad(set_to_none=True)`、更早释放 batch 临时对象、降低进度条刷新频率）
- `training/README.md`（补充内存优化说明）
- `docs/01-任务/任务A/任务A配置与启动说明.md`（补充按模型自动裁剪图字段说明）
- `docs/02-推进与变更/代码修改与实验推进记录.md`（本记录）

### 推进到实验步骤

## 2026-03-17 降采样策略优化：降低点数 + 壁面点上限采样

### 本次主要修改

**背景**：在服务器上进行训练时，由于数据量过大（每个图 40,000 节点 × 81 时间步 × 多个病例），大量 `.pt` 图文件加载导致系统内存（RAM）被耗尽，进程被 kill。

**核心改动**：

1. **降低目标总点数**：40,000 → 15,000，单图体积从 ~10 MB 降到 ~4 MB，总数据量减少约 60%
2. **新增壁面点上限机制**（`wall_max_points = 10000`）：
   - 原逻辑：壁面点无条件全保留（`wall_ratio: 1.0`），某些网格密集的 case 壁面点可能上万，挤占内部点预算
   - 新逻辑：壁面点超过上限时，使用混合采样（50% FPS + 50% random）降采样，FPS 保证血管分支和高曲率区域不丢失
3. **内部点预算动态分配**：内部点预算 = 15,000 - min(实际壁面点数, 10,000)，壁面点少的 case 自动获得更多内部点
4. **FPS 比例提升**：20% → 50%，更高的 FPS 占比保证血管几何特征的空间覆盖质量
5. **旧配置项 `wall_ratio` 移除**，由 `wall_max_points` 替代

**采样策略总览**：

| 区域 | 点数 | 采样策略 |
|---|---|---|
| 壁面 | min(实际, 10000) | 超出时 50% FPS + 50% random |
| 近壁内部 | (15000 - 壁面) × 70% | 50% FPS + 50% random |
| 核心内部 | (15000 - 壁面) × 30% | 50% FPS + 50% random |

### 对应代码/文档

- `pipeline/config.py` — `SAMPLING_CONFIG` 更新（`target_total_points`、`wall_max_points`、`hybrid_fps_ratio`，移除 `wall_ratio`）
- `pipeline/utils/sampling.py` — `stratified_sampling_by_distance()` 增加壁面点上限采样逻辑，提取 `_build_sampling_func()` 辅助函数
- `pipeline/preprocess.py` — `process_single_frame()`、`process_single_case()`、`process_all_cases()` 和 CLI 参数均传递 `wall_max_points`
- `pipeline/run_all.py` — 步骤1调用同步传递 `wall_max_points`
- `docs/02-推进与变更/代码修改与实验推进记录.md`
