# 实验分析对比图目录说明

> 与 Agent 技能 `.cursor/skills/analyze-experiment/` 配套；每次完成实验结果分析后，建议在本目录落盘至少一张对比图。

## 根目录

```
outputs/field/plots/analysis_compare/<slug>/
```

- `<slug>` 命名：`YYYYMMDD_<subject>_vs_<baseline>`
  示例：`20260530_V3P-F-30a_vs_AsymW4957`
- 代码常量：`training/core/field_plot_paths.py` → `CAT_ANALYSIS_COMPARE`

`outputs/` 在 `.gitignore` 中，图件不进版本库；**路径与 `manifest.json` 须写入** `docs/02-推进与变更/代码修改与实验推进记录.md`。

## 推荐文件（单次分析一套）

| 文件 | 内容 |
| --- | --- |
| `manifest.json` | 分析日期、exp_id、run 路径、基线、主指标数值、判定 |
| `metrics_compare.png` | test 主指标 + 次要指标柱状对比（当前 vs 基线） |
| `compare_val_loss.png` | 验证损失曲线叠加（可选 `val_score` / `wss_loss` 等同目录另存） |
| `metrics_compare.csv` | 与柱图一致的数值表，便于后续拼表 |

## 生成方式

**精度柱图 + manifest**（GNN 环境）：

```bash
python -m training.scripts.plot_experiment_analysis_compare \
  --run-dir outputs/field/<current_run>/ \
  --baseline-run-dir outputs/field/<baseline_run>/ \
  --slug 20260530_V3P-F-30a_vs_AsymW4957 \
  --primary-metric wss_r2_wss \
  --checkpoint best_wss
```

**损失曲线对比**（可与上式同目录输出）：

```bash
python -m training.scripts.plot_training_history \
  --run-dir outputs/field/<current_run>/ \
  --run-dir outputs/field/<baseline_run>/ \
  --output-dir outputs/field/plots/analysis_compare/<slug> \
  --compare-metric val_loss \
  --compare-title "val_loss: current vs baseline"
```

汇报/PPT 用汇总图仍可放在 `docs/03-汇报材料/figures/`；**日常实验分析对比**以本目录为准，避免与 `optimization/`、`multimodel_baseline/` _campaign 混放。
