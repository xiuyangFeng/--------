## 2. 论文总主线

论文最终要证明的是一条完整数字孪生链条成立：

`患者几何 + 边界条件 -> AI快速重建血流场 -> 衍生血流动力学指标 -> 闭塞/破裂风险预测`

因此，整篇论文不是两个独立模型的堆叠，而是一条分阶段验证的证据链。建议把全文组织为三个层次。

### 2.1 层次一：物理场重建

给定患者几何、时间信息和边界条件，快速预测速度场与压力场：

- 输入：`x, y, z`、显式几何特征、`is_wall`、`t_norm`、边界条件
- 输出：`u, v, w, p`

### 2.2 层次二：衍生指标恢复

根据预测得到的物理场，进一步计算：

- `WSS`
- `TAWSS`
- `OSI`
- `RRT`

并与 CFD 真值对比，验证 AI 结果是否足以支撑后续临床使用。

### 2.3 层次三：临床风险预测

以临床、形态和血流动力学特征为输入，预测临床终点风险。当前优先聚焦：

- 髂支闭塞风险预测

后续可扩展：

- AAA 破裂风险预测
- AAA 生长风险预测

## 4. 研究假设

后续所有实验都要为下面 4 条假设服务。

### H1

显式几何先验能够提升患者级血流场重建性能。

### H2

显式几何先验对复杂区域更有效，尤其是：

- 弯曲段
- 分叉区域
- 近壁区域

### H3

AI 重建的物理场能够稳定恢复 CFD 衍生指标，并在病例级排序和风险分层上与 CFD 保持较高一致性。

### H4

AI 衍生指标与临床、几何信息融合后，可获得接近 CFD 衍生指标支持下的临床风险预测性能。

## 5. 任务拆分与研究阶段

建议把项目按四个阶段推进，每个阶段都有明确的产出和停止条件。

### 5.1 阶段 A：基础血流场重建

### 目标

建立稳定、可复现的节点级 `u, v, w, p` 预测模型。

### 输出

- 一套固定数据划分
- 一套统一训练脚本
- 至少 3 个基线模型结果
- 一个主模型结果

### 通过标准

- 患者级测试集可稳定收敛
- 主模型优于简单点级和基础图模型
- 训练与推理流程稳定可重复

### 5.2 阶段 B：衍生指标恢复

### 目标

验证 AI 重建场是否能支持计算临床相关的血流指标。

### 输出

- `WSS / TAWSS / OSI / RRT` 的后处理脚本
- 与 CFD 对应指标的病例级和点级对比

### 通过标准

- 至少一组关键指标在病例级上与 CFD 高相关
- AI 衍生指标能区分部分高风险与低风险病例

### 5.3 阶段 C：风险预测模型

### 目标

建立以临床终点为目标的表格/融合模型，验证血流动力学特征的增益。

### 输出

- `Clinical-only` 基线
- `Clinical + Geometry` 基线
- `Clinical + Geometry + CFD-hemodynamics`
- `Clinical + Geometry + AI-hemodynamics`

### 通过标准

- CFD 衍生特征对风险预测确有增益
- AI 衍生特征能接近 CFD 版本效果

### 5.4 阶段 D：数字孪生闭环验证

### 目标

证明 AI 替代 CFD 后，整条风险评估链路仍然成立。

### 输出

- 端到端对照实验
- 临床可用性讨论
- 推理效率与临床部署价值分析

### 通过标准

- AI 版本相较 CFD 版本只有有限性能差距
- 但推理速度显著优于 CFD

## 6. 数据体系与统一原则

### 6.1 数据来源分层

项目中实际存在三类数据：

1. CFD 图数据
   - 节点级场数据
   - 时间维度快照
   - 几何显式特征
   - 边界条件

2. 衍生血流动力学指标
   - `WSS / TAWSS / OSI / RRT`
   - 可分节点级、区域级、病例级

3. 临床风险数据
   - 髂支闭塞标签
   - 破裂标签
   - 生长标签
   - 解剖参数
   - 临床表格特征

### 6.2 数据主键统一

必须从现在开始统一主键体系，避免后续模块无法联通。

推荐统一主键：

- `patient_id`
- `phase`
  - 例如 `pre`
  - `post_early`
- `time_step`
- `split_version`

所有下游表格、图数据、后处理结果、风险标签都必须能回溯到这套主键。

### 6.3 最关键的原则：病例级划分

所有任务都必须按患者进行划分，不能按时间帧随机划分。

禁止：

- 同一患者的不同时间帧同时出现在训练集和测试集
- 使用全量数据统计量对测试集做归一化
- 风险模型和场重建模型使用不同患者划分但又试图端到端对比

### 6.4 推荐数据划分方案

### 方案一：患者数足够

- 训练：60%-70%
- 验证：10%-20%
- 测试：20%

### 方案二：患者数有限

- 外层：`Leave-One-Patient-Out` 或 `5-fold patient-level CV`
- 内层：训练集内部再划验证集

### 方案三：风险预测样本很少

- 采用重复分层交叉验证
- 保持患者级、标签分层
- 最终报告均值和标准差

## 8. 任务 A：血流场重建实验设计

### 8.1 任务定义

### 输入

- 节点坐标：`x, y, z`
- 时间条件：`t_norm`

- 显式几何特征：`Abscissa, NormRadius, Curvature, Tangent`
- 节点类型：`is_wall`

### 输出

- `u, v, w, p`

### 预测层级

- 节点级回归

### 8.2 基线模型

任务 A 的基线不要过多，必须服务于科学问题。

### Baseline A1：Point-wise MLP

#### 输入

- `coords + t + BC`
- 可做一个增强版 `coords + t + BC + geometry`

#### 目的

证明图结构本身有价值。

### Baseline A2：GCN 或 GraphSAGE

#### 输入

- `coords + t + BC + is_wall`

#### 目的

证明不是任意图模型都能达到你的效果。

### Baseline A3：Transformer without geometry

#### 输入

- `coords + t + BC + is_wall`

#### 目的

控制 backbone，单独验证显式几何先验的贡献。

### Main Model A4：Transformer + geometry

#### 输入

- `coords + t + BC + geometry + is_wall`

#### 目的

作为第一阶段主模型。

### Optional A5：Transformer + geometry + physics

#### 目的

作为增强版，而不是起步版。

### 8.3 任务 A 的评价指标

### 基础指标

## 9. 任务 A 的消融实验清单

任务 A 的消融是整篇论文最重要的证据块，建议分为“必做”和“可选”。

### 9.1 必做消融

### Ablation A1：输入特征层级消融

实验组：

- `coords + t`
- `coords + t + BC`
- `coords + t + BC + is_wall`
- `coords + t + BC + geometry`
- `coords + t + BC + geometry + is_wall`

回答的问题：

- 边界条件是否必要
- 几何特征是否必要
- `is_wall` 是否必要

### Ablation A2：显式几何分量消融

实验组：

- 去掉 `Abscissa`
- 去掉 `NormRadius`
- 去掉 `Curvature`
- 去掉 `Tangent`
- 全部保留

回答的问题：

- 哪类几何先验贡献最大
- 哪类几何信息最稳定

### Ablation A3：坐标归一化消融

实验组：

- 原始坐标
- 仅中心化
- 中心化 + PCA 对齐
- 中心化 + PCA 对齐 + 缩放

回答的问题：

- 标准化坐标系统是否提升跨病例泛化

### Ablation A4：数据增强消融

实验组：

- 无增强
- 旋转
- 旋转 + 平移
- 旋转 + 平移 + 小尺度扰动

回答的问题：

- 泛化能力是否依赖在线增强

### Ablation A5：物理约束消融

实验组：

- 仅数据监督
- 数据 + continuity
- 数据 + continuity + momentum
- 数据 + continuity + momentum + no-slip

回答的问题：

- 物理约束带来的是真提升还是仅训练不稳定

### 9.2 可选消融

### Ablation A6：图构建参数

- `k=4, 6, 8, 12`
- `knn` vs `radius`（如果后续实现）

### Ablation A7：采样策略

- `random`
- `fps`
- `hybrid`

### Ablation A8：backbone 稳定性

- `GCN`
- `GraphSAGE`
- `GAT`
- `Transformer`

### Ablation A9：QN 模块

- 无 QN
- QN 加在消息传递后
- QN 加在解码前

### Ablation A10：隐式几何编码

- 无 PointNet++

- `TAWSS`
- `OSI`

### 10.2 任务 B 的核心目标

2. AI 衍生指标在临床风险排序上接近 CFD 指标。

### 10.3 评价层次

### 层次一：点级或壁面点级

- `RMSE`
- `MAE`
- `R²`

- `WSS`
- 局部 `OSI`

在左右髂动脉、AAA 主体等区域分别计算：

- 平均值
- 最大值
- 95th percentile
- 高风险阈值面积占比

- Pearson 相关
- Spearman 相关

## 12. 端到端数字孪生验证设计

这是全文最关键的整合实验，必须回答一个核心问题：

“如果用 AI 代替 CFD，临床预测是否仍然成立？”

### 12.1 三条核心链路

### Chain D1

`Clinical + Geometry`

### Chain D2

`Clinical + Geometry + CFD-hemodynamics`

### Chain D3

`Clinical + Geometry + AI-hemodynamics`

### 12.2 要回答的问题

1. `D2` 相对 `D1` 是否有明显提升。
2. `D3` 是否接近 `D2`。
3. `D3` 相对 `D1` 是否依然有效。

如果三点同时成立，数字孪生闭环成立。

### 12.3 端到端主结论模板

可以按以下逻辑组织论文主结论：

1. 真实 CFD 衍生特征确实提升风险预测。
2. AI 衍生特征与 CFD 衍生特征具有较高一致性。
3. 因此 AI 可作为快速近似数字孪生引擎，服务风险评估。

## 13. 实验执行顺序

这是最重要的“防止浪费资源”部分。不要跳步。

### Step 1：固定数据版本与 split

### 任务

- 固定 `split_version`
- 固定患者划分名单
- 固定训练/验证/测试表

### 不完成不得进入下一步

因为后续所有结果必须可复用、可复现、可对比。

### Step 2：固定任务 A 的训练框架

### 任务

- 建立统一训练入口
- 统一日志格式
- 统一 checkpoint 命名
- 统一早停策略

### 输出

- 最小可运行版本

### Step 3：跑通任务 A 的最小基线

至少跑通：

- `MLP`
- `GCN/GraphSAGE`
- `Transformer(no geometry)`
- `Transformer(geometry)`

### 目标

不要追求最强结果，先验证 pipeline 和训练链路无误。

### Step 4：完成任务 A 必做消融

优先顺序：

1. 输入特征消融
2. 几何分量消融
3. 坐标归一化消融
4. 数据增强消融

### 目标

把第一篇论文最核心的证据做扎实。

### Step 5：建立任务 B 后处理与验证链

### 任务

- 固定 `WSS / TAWSS / OSI / RRT` 的计算脚本
- 固定区域定义方法
- 固定病例级聚合规则

### 目标

把“场误差”变成“临床相关指标误差”。

### Step 6：确认 AI 指标对 CFD 指标的一致性

### 最低要求

- 至少关键指标有较高病例级相关性
- 风险排序趋势基本一致

如果这一步不过关，先不要进入风险预测大规模实验。

### Step 7：建立任务 C 的表格学习基线

优先跑：

- `Clinical-only`
- `Clinical + Geometry`
- `Clinical + Geometry + CFD-hemodynamics`

### 目标

先确认真实血流动力学特征对风险预测确实有价值。

### Step 8：加入 AI-hemodynamics

跑：

- `Clinical + Geometry + AI-hemodynamics`

### 核心判断

如果 AI 版本接近 CFD 版本，闭环成立。

### Step 9：最后再做高级增强模块

包括：

- physics loss
- QN
- PointNet++
- `h_global`

这些只能在主线闭环成立后再做。

## 15. 统计与报告规范

### 15.1 重复实验

所有主结果建议至少使用多个随机种子重复，报告：

- `mean ± std`

### 15.2 显著性检验

对于关键主结果可做：

- 配对 `t-test`
- Wilcoxon signed-rank test

用于支持：

- geometry 是否显著优于 no-geometry
- AI-hemodynamics 是否显著优于无 hemodynamics

### 15.3 置信区间

风险模型建议报告：

- `AUROC 95% CI`
- `AUPRC 95% CI`

## 16. 资源管理与止损原则

为了避免后续大量浪费资源，必须遵守以下止损策略。

### 16.1 不要同时优化多个维度

禁止在同一轮实验里同时改变：

- 模型结构
- 输入特征
- 损失函数
- 数据划分
- 图构建方式

每轮实验只允许有一个主要变量。

### 16.2 高级模块必须后置

以下模块不能作为第一阶段依赖：

- PINN 物理约束
- QN
- PointNet++
- 多任务端到端联合训练

原因：

- 训练不稳定
- 调试成本高
- 归因困难
- 容易把主结论打散

### 16.3 风险模型前必须先验证物理链

如果任务 B 的关键指标都无法稳定恢复，则不要投入大量资源做风险预测调参。

### 16.4 优先保证“主线可闭环”

比起单点最优精度，更重要的是先跑通：

`场重建 -> 衍生指标 -> 风险预测`

只要闭环成立，后续增强才有意义。

## 18. 当前阶段的明确执行建议

从现在开始，建议按以下顺序落地：

1. 固定患者级划分和数据版本。
2. 补齐任务 A 的统一训练脚本。
3. 先只做 4 个模型：`MLP / GraphSAGE / Transformer(no geometry) / Transformer(geometry)`。
4. 完成任务 A 的 4 组必做消融。
5. 固定衍生指标计算流程。
6. 做 CFD 与 AI 指标一致性验证。
7. 再做闭塞风险模型。
8. 最后再引入 physics loss、QN、PointNet++ 和图级深度特征。

这条顺序的核心原则是：

先证明主链成立，再做增强；先做可解释的简单版本，再做复杂模块。
