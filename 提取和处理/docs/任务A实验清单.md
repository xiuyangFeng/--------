# 任务A实验清单

> 上位文档：[实验设计总纲](实验设计总纲.md) | 相关文档：[任务B](任务B指标计算规范.md) / [任务C](任务C风险建模规范.md) / [任务D](任务D端到端验证清单.md)

## 1. 文档定位

本清单用于执行“血流场重建”阶段的全部核心实验。目标不是继续罗列更多 backbone，而是把任务 A 固定成一条可复现、可归因、可支撑论文叙事的实验路线。

适用范围：

- 节点级速度场和压力场预测
- 精度对比
- 效率对比
- 必做消融
- 可选扩展

---

## 2. 任务定义

### 2.1 输入

- 节点坐标：`x, y, z`
- 显式几何特征：`Abscissa, NormRadius, Curvature, Tangent_X, Tangent_Y, Tangent_Z`
- 节点标签特征：`is_wall`
- 图级条件：`t_norm, BC_Inlet, BC_O1~O4`

### 2.2 输出

- `u, v, w, p`

### 2.3 目标

在患者级泛化前提下，实现高精度、稳定、快速的血流场重建，并明确回答以下问题：

1. 图结构是否优于点级模型。
2. 显式几何特征是否带来稳定增益。
3. 边界条件和 `is_wall` 是否有必要。
4. 坐标对齐和增强是否提升跨病例泛化。
5. 物理约束是否带来额外收益。
6. 在 4 万点大图场景下，多尺度层次化建模是否比单尺度消息传递更优。
7. 精度提升究竟来自 backbone 更强，还是来自层次化表示与更合理的图结构设计。

## 4. 实验组织原则

任务 A 的实验必须按“由简到繁、由主问题到扩展问题”的顺序组织：

1. 先证明图建模必要。
2. 再证明显式几何先验必要。
3. 再证明泛化增强和坐标标准化是否必要。
4. 再判断物理约束是否值得保留。
5. 最后处理 4 万点快照下的效率瓶颈，验证多尺度层次化结构是否值得成为主线模型。

禁止的做法：

- 一次同时改 backbone、输入、图结构和损失，导致无法归因。
- 只汇报整体误差，不汇报效率和区域指标。
- 用“更复杂模型更准”代替“问题被清楚回答”。

## 5. 基线实验清单

以下 5 组是任务 A 第一批必须跑通的实验，其中前 4 组用于建立现有主线，第 5 组用于回答 4 万点大图场景下的层次化建模价值。

## 5.1 A-Base-01：Point-wise MLP

### 目的

建立无图结构下限。

### 输入配置

- `coords + t_norm + BC`

### 固定条件

- 不使用图结构
- 不使用显式几何
- 不使用物理约束

### 输出记录

- `RMSE_u`
- `RMSE_v`
- `RMSE_w`
- `RMSE_|v|`
- `RMSE_p`
- `R2_p`
- 推理时间

## 5.2 A-Base-02：GraphSAGE 或 GCN

### 目的

验证图建模本身是否优于点级模型。

### 输入配置

- `coords + t_norm + BC + is_wall`

### 固定条件

- 使用默认构图参数
- 不使用显式几何
- 不使用物理约束

## 5.3 A-Base-03：Transformer without geometry

### 目的

控制 backbone，隔离几何特征贡献。

### 输入配置

- `coords + t_norm + BC + is_wall`

### 固定条件

- backbone 与主模型保持一致
- 去掉 `Abscissa, NormRadius, Curvature, Tangent`

## 5.4 A-Main-01：Transformer with geometry

### 目的

作为任务 A 第一阶段主模型。

### 输入配置

- `coords + t_norm + BC + geometry + is_wall`

### 固定条件

- 使用当前 pipeline 主图结构
- 使用病例级坐标对齐
- 使用默认增强策略

## 5.5 A-Base-04：Hierarchical GNN

### 目的

验证在 4 万点大图条件下，多尺度层次化建模是否能提供更优的精度-速度-显存折中。

### 推荐结构

- 多尺度编码器
- 粗尺度消息传递
- 细尺度特征传播与解码

### 推荐实现方向

- PointNet++ 风格下采样 + 图消息传递
- 或多分辨率图 U-Net 风格 encoder-decoder

### 输入配置

- `coords + t_norm + BC + geometry + is_wall`

### 固定条件

- 与 `A-Main-01` 保持同一训练预算
- 只改变层次化表示与消息传递方式
- 首轮不叠加 physics loss

## 7. 必做消融实验

以下实验是论文必须保留的核心证据。

## 7.1 A-Abl-01：输入特征层级消融

### 研究问题

边界条件、几何特征和 `is_wall` 分别带来什么贡献。

### 实验组

1. `coords + t`
2. `coords + t + BC`
3. `coords + t + BC + is_wall`
4. `coords + t + BC + geometry`
5. `coords + t + BC + geometry + is_wall`

### 控制变量

- backbone 固定为同一版本 Transformer
- 图构建固定
- 训练预算固定

### 结果要求

- 至少报告整体误差
- 额外报告壁面与内部点误差

## 7.2 A-Abl-02：显式几何分量消融

### 研究问题

哪类几何先验真正有效。

### 实验组

1. Full geometry
2. 去掉 `Abscissa`
3. 去掉 `NormRadius`
4. 去掉 `Curvature`
5. 去掉 `Tangent`

### 结果要求

- 报告总体误差
- 报告高曲率区域误差
- 报告近壁区域误差

## 7.3 A-Abl-03：坐标归一化消融

### 研究问题

病例内坐标标准化是否有利于泛化。

### 实验组

1. 原始坐标
2. 仅中心化
3. 中心化 + PCA 对齐
4. 中心化 + PCA 对齐 + 缩放

### 结果要求

- 报告测试集整体误差
- 报告跨病例方差

## 7.4 A-Abl-04：增强策略消融

### 研究问题

增强是否改善患者级泛化。

### 实验组

1. 无增强
2. 旋转
3. 旋转 + 平移
4. 旋转 + 平移 + 小尺度扰动

### 结果要求

- 报告平均性能
- 报告不同 seed 下稳定性

## 7.5 A-Abl-05：物理约束消融

### 研究问题

物理损失是否真正提升效果，还是仅增加训练复杂度。

### 实验组

1. 数据损失 only
2. 数据 + continuity
3. 数据 + continuity + momentum
4. 数据 + continuity + momentum + no-slip

### 额外记录

- 训练稳定性
- 收敛速度
- 残差分布

## 7.6 A-Abl-06：层次深度消融

### 研究问题

多尺度收益来自“有 hierarchy”本身，还是来自更深更复杂的主干。

### 实验组

1. 单尺度 `40k`
2. 双尺度 `40k -> 10k`
3. 三尺度 `40k -> 10k -> 2.5k`
4. 四尺度 `40k -> 10k -> 2.5k -> 600`

### 结果要求

- 报告整体误差
- 报告推理时间
- 报告训练时间
- 报告峰值显存

## 7.7 A-Abl-07：下采样策略消融

### 研究问题

层次化结构中的收益是否依赖特定采样策略。

### 实验组

1. random
2. FPS
3. hybrid FPS + random
4. 几何感知采样
5. 壁面保留 + 内域分层采样

### 结果要求

- 报告总体误差
- 报告近壁区域误差
- 报告分叉区域误差
- 报告采样后点数分布

## 7.8 A-Abl-08：图结构消融

### 研究问题

性能增益来自 hierarchy，还是来自更合理的图结构。

### 实验组

1. 单尺度 `kNN`
2. 单尺度 `radius`
3. `kNN + axial edges`
4. `kNN + edge_attr`
5. `multi-relational graph`

### 结果要求

- 报告整体误差
- 报告高曲率区域误差
- 报告分叉区域误差
- 报告图规模统计

## 7.9 A-Abl-09：解码与上采样策略消融

### 研究问题

细节恢复来自 encoder 的粗尺度建模，还是 decoder 的信息传播方式。

### 实验组

1. 最近邻上采样
2. 插值传播
3. skip connection + 插值
4. coarse-to-fine residual refinement

### 结果要求

- 报告总体误差
- 报告近壁区域误差
- 报告细小分支误差

## 12. 通过标准与止损点

## 12.1 可以进入任务 B 的最低标准

满足以下至少 4 条再进入任务 B：

1. 主模型稳定优于 MLP
2. 主模型稳定优于无几何的 Transformer
3. 多个 seed 结果方向一致
4. 近壁或高曲率区域性能有明确改善
5. `A-Base-04` 在精度-速度-显存三者中至少两项优于单尺度主模型
6. 推理速度明显优于 CFD

## 12.2 需要止损复盘的情况

出现以下情况时，不要盲目加新模块：

1. 不同 seed 结果波动极大
2. geometry 组不稳定优于 no-geometry
3. 不同 split 结果完全相反
4. physics loss 导致训练普遍崩溃
5. hierarchy 只能提速但精度明显崩坏
6. 多模块同时更改导致无法归因

## 13. 建议执行顺序

1. 先跑 4 个现有基线：`A-Base-01` 到 `A-Main-01`
2. 跑输入特征消融：`A-Abl-01`
3. 跑几何分量消融：`A-Abl-02`
4. 跑坐标归一化消融：`A-Abl-03`
5. 跑增强消融：`A-Abl-04`
6. 跑物理约束消融：`A-Abl-05`
7. 在单尺度主线稳定后，再引入 `A-Base-04`
8. 跑 hierarchy 相关消融：`A-Abl-06` 到 `A-Abl-09`

这条顺序的原则是：

先做最能回答论文问题的实验，再做复杂增强；先把单尺度主线结论站稳，再引入 4 万点场景下的效率优化路线。
