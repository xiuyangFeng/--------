# 任务A实验清单

> 上位文档：[实验设计总纲](实验设计总纲.md) | 相关文档：[任务B](任务B指标计算规范.md) / [任务C](任务C风险建模规范.md) / [任务D](任务D端到端验证清单.md)

## 1. 文档定位

本清单用于执行“血流场重建”阶段的全部核心实验。目标是把任务 A 拆成可直接排期、可直接记录、可直接复现实验的执行单。

适用范围：

- 节点级速度场和压力场预测
- 基线实验
- 必做消融
- 可选扩展

---

## 2. 任务定义

### 2.1 输入

- 节点坐标：`x, y, z`
- 显式几何特征：`Abscissa, NormRadius, Curvature, Tangent_X, Tangent_Y, Tangent_Z`
- 节点标签特征：`is_wall`
- 图级条件：`t_norm, BC_Inlet, BC_O1~O4`

### 2.2 输出

- `u, v, w, p`

### 2.3 目标

在患者级泛化前提下，实现高精度、稳定、快速的血流场重建。

## 3. 执行前固定项

以下项目在进入训练前必须锁定，后续实验不得随意更改。

### 3.1 数据版本

- `data_version`
- 数据源范围
- 病例排除规则
- 时间步过滤规则

### 3.2 划分版本

- `split_version`
- 训练患者列表
- 验证患者列表
- 测试患者列表

### 3.3 预处理版本

- 采样方法
- 目标点数
- 坐标归一化方式
- 归一化统计来源

### 3.4 训练通用设置

- 随机种子集合
- batch size
- epoch 上限
- early stopping
- optimizer

建议默认固定：

- `seed = [1, 2, 3]`
- `optimizer = Adam`
- `scheduler = ReduceLROnPlateau`

---

## 4. 任务A主问题

任务 A 的实验必须围绕以下问题组织：

1. 图结构是否优于点级模型。
2. 显式几何特征是否带来稳定增益。
3. 边界条件和 `is_wall` 是否有必要。
4. 坐标对齐和增强是否提升跨病例泛化。
5. 物理约束是否带来额外收益。

---

## 5. 基线实验清单

以下 4 组是第一批必须跑通的实验。

## 5.1 A-Base-01：Point-wise MLP

### 目的

建立无图结构下限。

### 输入配置

- `coords + t_norm + BC`

### 固定条件

- 不使用图结构
- 不使用显式几何
- 不使用物理约束

### 输出记录

- `RMSE_u`
- `RMSE_v`
- `RMSE_w`
- `RMSE_|v|`
- `RMSE_p`
- `R2_p`
- 推理时间

## 5.2 A-Base-02：GraphSAGE 或 GCN

### 目的

验证图建模本身是否优于点级模型。

### 输入配置

- `coords + t_norm + BC + is_wall`

### 固定条件

- 使用默认构图参数
- 不使用显式几何
- 不使用物理约束

## 5.3 A-Base-03：Transformer without geometry

### 目的

控制 backbone，隔离几何特征贡献。

### 输入配置

- `coords + t_norm + BC + is_wall`

### 固定条件

- backbone 与主模型保持一致
- 去掉 `Abscissa, NormRadius, Curvature, Tangent`

## 5.4 A-Main-01：Transformer with geometry

### 目的

作为任务 A 第一阶段主模型。

### 输入配置

- `coords + t_norm + BC + geometry + is_wall`

### 固定条件

- 使用当前 pipeline 主图结构
- 使用病例级坐标对齐
- 使用默认增强策略

## 7. 必做消融实验

以下实验是论文必须保留的核心证据。

## 7.1 A-Abl-01：输入特征层级消融

### 研究问题

边界条件、几何特征和 `is_wall` 分别带来什么贡献。

### 实验组

1. `coords + t`
2. `coords + t + BC`
3. `coords + t + BC + is_wall`
4. `coords + t + BC + geometry`
5. `coords + t + BC + geometry + is_wall`

### 控制变量

- backbone 固定为同一版本 Transformer
- 图构建固定
- 训练预算固定

### 结果要求

- 至少报告整体误差
- 额外报告壁面与内部点误差

## 7.2 A-Abl-02：显式几何分量消融

### 研究问题

哪类几何先验真正有效。

### 实验组

1. Full geometry
2. 去掉 `Abscissa`
3. 去掉 `NormRadius`
4. 去掉 `Curvature`
5. 去掉 `Tangent`

### 结果要求

- 报告总体误差
- 报告高曲率区域误差
- 报告近壁区域误差

## 7.3 A-Abl-03：坐标归一化消融

### 研究问题

病例内坐标标准化是否有利于泛化。

### 实验组

1. 原始坐标
2. 仅中心化
3. 中心化 + PCA 对齐
4. 中心化 + PCA 对齐 + 缩放

### 结果要求

- 报告测试集整体误差
- 报告跨病例方差

## 7.4 A-Abl-04：增强策略消融

### 研究问题

增强是否改善患者级泛化。

### 实验组

1. 无增强
2. 旋转
3. 旋转 + 平移
4. 旋转 + 平移 + 小尺度扰动

### 结果要求

- 报告平均性能
- 报告不同 seed 下稳定性

## 7.5 A-Abl-05：物理约束消融

### 研究问题

物理损失是否真正提升效果，还是仅增加训练复杂度。

### 实验组

1. 数据损失 only
2. 数据 + continuity
3. 数据 + continuity + momentum
4. 数据 + continuity + momentum + no-slip

### 额外记录

- 训练稳定性
- 收敛速度
- 残差分布

## 11. 通过标准与止损点

## 11.1 可以进入任务 B 的最低标准

满足以下至少 3 条再进入任务 B：

1. 主模型稳定优于 MLP
2. 主模型稳定优于无几何的 Transformer
3. 多个 seed 结果方向一致
4. 近壁或高曲率区域性能有明确改善
5. 推理速度明显优于 CFD

## 11.2 需要止损复盘的情况

出现以下情况时，不要盲目加新模块：

1. 不同 seed 结果波动极大
2. geometry 组不稳定优于 no-geometry
3. 不同 split 结果完全相反
4. physics loss 导致训练普遍崩溃
5. 多模块同时更改导致无法归因

---

## 12. 建议执行顺序

1. 先跑 4 个基线。
2. 跑输入特征消融。
3. 跑几何分量消融。
4. 跑坐标归一化消融。
5. 跑增强消融。
6. 最后再碰 physics loss。

这条顺序的原则是：

先做最能回答论文问题的实验，再做复杂增强。
