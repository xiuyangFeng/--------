# 整理思路

### 总记

用“显式几何先验 + **物理约束的图神经网络主干** + 多模态临床融合”的多任务框架，预测时空血流动力学场（速度、压力、WSS、OSI 等），并进一步完成基于数字孪生体的临床终点风险预测（如EVAR后的髂肢闭塞、AAA破裂），实现预测时空场的秒级响应和数字孪生在医疗中的使用。

---

### 目标与子任务

- 物理场重建：基于AAA的点云数据集，预测时空速度与压力场，派生WSS、OSI、RRT 等指标。
- 临床终点预测：将图池化的全局物理表征与临床表格特征融合，输出风险概率或连续结局（如年生长率，破裂）。
- 结构优化：物理约束、显示几何特征、GNN 消息传递

---

### 模型架构（PI-GNN 多任务）

- 主干：**Graph Transformer** （图卷积神经网络），输入增强节点特征，输出节点嵌入 H∈R^{N×F}。
- 头A（物理场）：两个独立 MLP 分别回归速度场（N×3）与压力场（N×1）。由此可派生 WSS/OSI/RRT。
- 头B（临床）：池化得到 h_global，与临床向量拼接，经 MLP 输出概率（Sigmoid）。支持风险分层与可解释性分析（SHAP、校准曲线、DCA）。[[13]]({\{{notion-3}\}})

## **模块一：**预测时空血流动力学场

### 1.数据板块

**1.1 显式几何特征工程（Explicit Geometric Feature Engineering）**

目的：为复杂的动脉瘤几何建立一个标准化的“骨架”，提取具有明确物理意义的先验几何，解决不同患者几何形态各异、难以比较的问题，为模型提供强大的几何先验。

具体操作：对输入的3D模型（网格/点云），提取血管中心线，并为每个点计算出曲率、沿程位置等特征。

**1. `Abscissa` (归一化弧长坐标)**

- **源代码逻辑**：
    
    ```
    python
    
    geo_abscissa[i]= arr_abscissa[closest_id]# 获取中心线上最近点的弧长
    # ... 后续进行了归一化处理 ...
    geo_abscissa= (geo_abscissa- ab_min)/ (ab_max- ab_min)
    
    ```
    
- **几何意义**：它表示该点在血管**“长度方向”上的相对位置**。
    - **0.0**：通常表示血管的起点（入口端）。
    - **1.0**：通常表示血管的终点（出口端）。
    - **中间值**：表示该点轴向位置在血管总长度中的百分比。
- **作用**：让模型知道这个点是在血管的上游、中游还是下游。

**2. `NormRadius` (归一化径向距离/归一化半径)**

- **源代码逻辑**：
    
    ```
    python
    
    dist= np.linalg.norm(pt- cl_points[closest_id])# 点到中心线的绝对距离
    local_r= arr_radius[closest_id]# 中心线该位置的最大内切球半径
    geo_norm_dist[i]= dist/ local_r# 归一化：距离 / 半径
    
    ```
    
- **几何意义**：它表示该点在血管**“横截面方向”上的相对位置**。
    - **~ 0.0**：该点非常靠近中心线（血管核心区域）。
    - **~ 1.0**：该点位于血管壁附近（边缘区域）。
    - **> 1.0**：说明该点在内切球之外，通常发生在血管形状不规则或有分叉的地方。
- **作用**：反映点在径向上的位置，即它是靠近管腔中心还是靠近血管壁。

**3. `Curvature` (曲率)**

- **源代码逻辑**：
    
    ```
    python
    geo_curv[i]= arr_curv[closest_id]# 直接获取中心线在该位置的曲率
    
    ```
    
- **几何意义**：它是中心线在该点处的**弯曲程度**。
    - **定义**：它是曲率半径的倒数 。
    - **0.0**：表示这一段血管是笔直的。
    - **数值越大**：表示血管在这里弯曲得越厉害（转弯处）。
- **作用**：提供局部的弯曲几何信息。在血流动力学中，高曲率区域通常会产生复杂的二次流或涡流，对压力和速度分布有重大影响。

显示几何特征工程的建设需要依赖于VMTK这个python库进行，具体的操作需要研究一下中心线如何提取和构建批量的自动化格式。

**1.3 数据处理和对齐**

目的：为了解决组内的多尺度数据差异问题

具体操作：

**1.3.1 采样**

为了综合考虑训练的成本和数据预处理的成本，必须将内部的大量随机采样的点云数据做处理。处理的核心思路：**对数据进行分层采样。**

- **输入参数**：`wall_points`（壁面点），`internal_points`（内部点），`max_total_points`（默认40000）。
- **核心目标**：将总点数限制在 `max_total_points` 以内，同时保证血管几何结构的完整性（防止截断），并优先保留高价值区域的点。

- **采样逻辑**：
    - **优先级 1（壁面）**：无条件保留所有 `wall_points`。计算剩余预算 `budget = max_total - len(wall_points)`。
    - **优先级 2（内部点分层）**：使用 `scipy.spatial.cKDTree` 计算 `internal_points` 到壁面的距离。将内部点划分为两类：
        - **近壁层 (Boundary)**：距离 < 2.0mm。
        - **核心层 (Core)**：距离 >= 2.0mm。
    - **动态配额分配**：将剩余预算 `budget` 按 **7:3** 的比例分配给近壁层和核心层。
        - **关键逻辑**：必须实现“动态补位”。如果某一层的实际点数少于其分配的配额，必须将多余的配额转移给另一层，确保最大限度利用 40000 个点的空间。
- **采样方法**：对于需要减少点数的层，必须使用 `np.random.choice` (replace=False) 进行**全局随机抽取**，严禁使用数组切片（如 `[:N]`），以防止血管在空间上被切断或覆盖不全。采样方法具体可采取的方法有：
    - **全局随机采样**：可以完成95%效能的FPS采样，但是效率性能极高
    - **FPS（最远点采样）**：可以保证采样更均匀，可以完整覆盖所有的血管，但是计算成本较高
- **输出**：返回合并后的 `wall_points` 和采样后的 `internal_points`。

### 2.模型架构

**图数据（Graph Data）- 描述几何与物理**

**节点（Nodes）**：重建的腹主动脉瘤（AAA）的点云数据集，一个点云代表一个节点。

**边（Edges）（对齐 pipeline）**：当前 pipeline 使用 **k 近邻（k-NN）** 在坐标空间构图，默认 `k=6`（可通过 `convert_to_graph.py --k` 或 `pipeline/config.py` 的 `GRAPH_CONFIG["k_neighbors"]` 调整）。

半径邻域/更大 k 的图构建策略可作为后续研究扩展，但不是当前 pipeline 的默认实现。

**节点特征（Node Features）**：这是一个 N x F_node 的矩阵，其中 F_node 是每个节点的特征维度。对于每一个节点，其特征向量应包含：

**时空坐标（对齐 pipeline）**：`[x, y, z, t_norm]`。

其中 `t_norm` 由时间步 `step` 线性归一化得到：`t_norm = (step - step_min) / (step_max - step_min)`（见 `pipeline/convert_to_graph.py`）。

**边界条件信息（对齐 pipeline）**：`[BC_Inlet, BC_O1, BC_O2, BC_O3, BC_O4]` 为全局标量（每个时间步 5 个值），在构图时复制给每个节点。

- `BC_Inlet`：入口体积流量（从 `Global_conditions/vf-in-rfile.out` 读取，或从质量流量文件换算）。
- `BC_O1~O4`：四个髂支出口压力（固定从对应 `p-out*-rfile.out` 读取）。
- 当前 pipeline **不再使用 `BC_Flag`** 来区分流量/压力两种出口类型。

**几何增强特征（对齐 pipeline）**：`[Abscissa, NormRadius, Curvature, Tangent_X, Tangent_Y, Tangent_Z]`（6 维）。

**输入特征总览（对齐 pipeline）**：当前 pipeline 的 `data.x` 为 **15 维**：

- 坐标 (3)：`x, y, z`
- 时间 (1)：`t_norm`
- 几何 (6)：`Abscissa, NormRadius, Curvature, Tangent_X/Y/Z`
- 边界标志 (1)：`is_wall`
- 边界条件 (5)：`BC_Inlet, BC_O1~O4`
（PointNet++ 局部几何嵌入等更高维特征属于未来扩展，当前 pipeline 未接入。）

**节点标签**（Node Labels / Targets）- 仅用于计算数据损失：这是一个 N x F_label 的矩阵，包含了CFD仿真出的标准物理场数据[velocity_u, velocity_v, velocity_w, pressure]。

**点标签：**每个节点增加一个“类型”特征：**壁面点**还是**内部点** `is_wall=1/0`。

对齐当前 pipeline 的实现：`is_wall` 在清洗阶段由速度阈值推断（近似认为壁面速度为 0），即 `speed = sqrt(u^2+v^2+w^2)`，若 `speed < 1e-6` 则 `is_wall=1`（见 `pipeline/utils/io.py`）。

这告诉GNN：“这个节点是一个壁面点，你需要在满足无滑移条件”，或者“这是一个内部点，你需要满足Navier-Stokes方程”。

**物理约束（PINN）**：PINN的物理损失函数依然作用于GNN的最终输出上，保证物理规律的满足。

**图神经网络主干（GNN Backbone）**

作用：负责从输入的图数据中学习几何和物理场的时空表示。

输入：上述的图数据（节点、边、节点特征）。

**架构**：可以采用多层的Graph Transformer。在每一层，GNN通过消息传递机制，聚合邻居节点的信息来**更新自身的节点表示，不仅使用标准的MLP，还引入二次残差模块（QN）。**经过 L 层后，每个节点都会得到一个最终的、高度浓缩的特征嵌入向量 hv。这个向量 hv 已经隐式地包含了该点周围的几何信息和复杂的流场信息。

输出：一个 N x F_embed 的节点嵌入矩阵，其中 F_embed 是最终嵌入向量的维度，记该节**点嵌入矩阵为H**。

- 节点嵌入矩阵 H 被**同时**发送给两个独立的预测子头。
    - **子头A（速度预测头）**：
        - **架构**：一个独立的MLP，例如 MLP_velocity。
        - **操作**：它接收**整个** H 矩阵，并对**每一行 h_v** 进行处理。
        - predicted_velocity_for_node_v = MLP_velocity(hv)
        - **输出**：由于速度是三维的，这个MLP的输出层应该有**3个神经元**。因此，它的最终输出是一个 N x 3 的矩阵，代表了对**每一个节点**的速度向量 [pred_u, pred_v, pred_w] 的预测。**这就是一个完整的、预测出的速度场。**
    - **子头B（压力预测头）**：
        - **架构**：**另一个独立的** MLP，例如 MLP_pressure。
        - **操作**：它也接收**同一个 H 矩阵**，并对**每一行 hv**进行处理。
        - predicted_pressure_for_node_v = MLP_pressure(hv)
        - **输出**：由于压力是标量，这个MLP的输出层应该有**1个神经元**。因此，它的最终输出是一个 N x 1 的向量，代表了对**每一个节点**的压力值 [pred_p] 的预测。**这就是一个完整的、预测出的压力场。**

**结构总结**

1. **输入唯一且不含标签**：GNN主干只有一个统一的输入，只包含描述问题的特征。
2. **主干负责通用理解**：GNN主干的输出是“高浓缩的通用特征向量”（节点嵌入）。
3. **子头负责专业预测**：不同的预测子头（速度、压力）都是**独立的MLP**，它们都接收**同一个**通用特征向量作为输入，然后各自输出自己负责预测的物理量。
4. **最终输出是完整的物理场**：两个子头的输出合在一起，就是模型预测出的**一个完整的速度场和一个完整的压力场**。它们**不是**包含其他特征的矩阵或向量，而是**最终的、具有物理意义的预测结果**。

**损失函数（Loss function）设计**

L_physics_task = L_data (CFD) + λ * L_physics (Navier-Stokes)

**1. 总损失函数结构**

总损失 = 数据损失（所有点的速度 / 压力预测误差） + ω × 物理损失（按节点类型分配的约束残差）其中：

- **ω**：物理损失的总权重（推荐 1~5，可通过网格搜索优化，避免物理约束过强掩盖数据规律）；
- 核心改进：物理损失不再是 “所有点的统一残差求和”，而是 “按节点类型拆分残差，并赋予不同权重”。

**2. 各部分损失的具体计算（分节点类型）**

**（1）数据损失：所有点统一计算，保证预测值与 CFD 标签一致**

数据损失 = (1/N_total) × [ Σ（内部点：|u_pred - u_cfd|² + |p_pred - p_cfd|²） + Σ（壁面点：|u_pred - 0|² + |p_pred - p_cfd|²） ]

- **说明**：
    - 内部点：u 和 p 均以 CFD 标签为基准（u_cfd≠0，p_cfd 为真实压力）；
    - 壁面点：u 以 “无滑移条件” 为基准（目标值 0），p 仍以 CFD 标签为基准（壁面压力有真实物理意义，需拟合）；
    - N_total：所有点的总数，保证数据损失是全局平均误差。

**（2）物理损失：按节点类型拆分，避免过定**

物理损失 = (1/N_inner) × Σ（内部点：残差_NS） + (α/N_wall) × Σ（壁面点：残差_no_slip）

- **各残差定义**：① 残差_NS（内部点的 Navier-Stokes 残差）：残差_NS = |∇・u_pred|²（质量守恒残差） + |ρ(∂u_pred/∂t + u_pred・∇u_pred) - ∇・σ_pred|²（动量守恒残差）（σ_pred 为预测的 Cauchy 应力张量，σ = -pI + 2μD，D 为速度梯度张量）② 残差_no_slip（壁面点的无滑移残差）：残差_no_slip = |u_pred - 0|²（仅约束速度为 0，不涉及压力）
- **权重与数量说明**：① N_inner：内部点总数，N_wall：壁面点总数（按类型统计，避免某类点数量过多导致残差主导）；② α：壁面约束的权重系数（推荐 0.5~1）—— 因壁面点的无滑移约束已在 “数据损失” 中体现，物理损失中无需过高权重，避免重复约束；
- **关键优化**：仅内部点计算 Navier-Stokes 残差，壁面点仅计算无滑移残差，从根本上解决过定问题。
1. **标签生成**：从组内 CFD 数据或血管几何模型中提取 “壁面标签”——
    - 若有血管 STL 模型：计算每个点云点到 STL 表面的距离，距离 < 0.1mm（可根据血管尺寸调整）的点标记为`is_wall=1`，其余为`is_wall=0`；
    - 若仅有 CFD 数据：CFD 计算时会区分 “壁面边界” 与 “内部流场”，可直接导出每个点的 “边界类型” 标签，映射为`is_wall`。
2. **模型适配**：在 GNN 的输入层加入 “节点类型特征”（`is_wall`或`node_type`），并在 PINN 的损失计算层添加 “类型判断逻辑”——
    - 用 PyTorch 的条件判断（如`torch.where(is_wall == 1, 残差_no_slip, 残差_NS)`）实现差异化残差计算；
    - 避免硬编码：将标签作为模型输入的一部分，而非在代码中固定，方便后续扩展 “入口 / 出口点” 等其他类型的约束（如入口点的流量约束）。
3. **验证效果**：训练后通过 “残差分布可视化” 验证约束有效性 ——
    - 内部点：Navier-Stokes 残差应均匀分布在低数值区间（如 < 0.1），无明显高残差聚类（说明流场符合物理规律）；
    - 壁面点：无滑移残差应接近 0（如 < 0.01），且壁面附近内部点的速度梯度应连续（如从壁面 u=0 过渡到内部 u = 正常流速，无突变）。

**1. 目标与问题定义**

- **临床问题**：对于接受EVAR治疗的AAA患者，能否在术后早期，就根据其解剖形态和血流动力学特征，预测其远期发生髂动脉支架肢体闭塞（ILO）的风险
- **机器学习任务**：这是一个**二分类问题**。
    - **输入（X）**：一个描述单个患者“解剖+血流动力学+临床”状态的综合特征向量。
    - **输出（Y）**：一个二元标签，1代表未来发生ILO，0代表保持通畅。

**2. 数据准备与特征工程（核心步骤）**

- **数据集**：来自论文的75名患者（25例闭塞和50例通畅）。
- **特征构建**：对于这75名患者中的**每一位**，需要构建一个特征向量。这个向量至少包含以下三组特征：
    - **组A：AI驱动的深度血流动力学特征（来自模块一中的神经网络）**
        1. **全局隐式特征 (h_global)**：将患者的术后早期三维几何模型输入到您冻结的PI-GNN引擎中，对GNN主干输出的所有节点嵌入进行图池化（如平均池化，最大池化），得到一个高维向量（例如，512维）。这个向量是AI对整个流场模式的高维特征学习。
        2. **局部/区域显式特征**：同样基于引擎的预测结果，计算出针对ILO问题的、更具空间特异性的量化指标。例如：
            
            在左右髂动脉（IA）区域分别计算：
            
            - mean_WSS_left_IA（左侧髂动脉平均壁面剪切应力）
            - mean_WSS_right_IA（右侧髂动脉平均壁面剪切应力）
            - high_OSI_area_ratio_left_IA（左侧髂动脉高振荡剪切指数区域比）
            - high_OSI_area_ratio_right_IA（右侧髂动脉高振荡剪切指数区域比）。
            - 计算双侧的**不对称性指标**：abs(mean_WSS_left - mean_WSS_right)。
    - **组B：知识驱动的验证性特征**
        1. 需要精确地复现该论文中被证明最有效的预测因子
            - DRI_of_pre_EVAR_OSI：术前双侧IA的OSI差异比。
            - RRT_IA_post：术后IA的相对驻留时间。
            - TAWSS_pre：术前整体TAWSS。
            - Radius_EIA_pre：术前髂外动脉半径。
            - Torsion_CIA_DRE_DRI：一个复杂的、描述双侧CIA扭转变化的衍生指标。
            
            **注意**：这些特征同时包含了术前（pre-EVAR）**和**术后早期（post-EVAR）的信息，特征提取流程需要能处理这两个时间点的几何和物理场。
            
    - **组C：基础临床与解剖特征**
        1. 患者的人口统计学信息：Age（年龄），height（身高），weight（体重）。
        2. 常规解剖学指标：Proximal_neck_angulation（近端颈部成角）, Aneurysm_max_diameter（动脉瘤最大直径）。
        3. 手术相关信息：Stent_graft_diameter（支架移植物直径）, Stent_oversizing_ratio（支架过度尺寸比）。
        4. 合并症：Hypertension（高血压）, Smoking_status（吸烟状况）等。

**3. 模型选择与训练**

- **模型选择**：由于现在处理的是一个**表格数据**的分类问题（输入是一个融合后的特征向量），可以选择多种强大的机器学习模型：
    - **首选：梯度提升树模型（如 XGBoost, LightGBM）**。这类模型在处理异构的表格数据时通常表现最佳，并且自带特征重要性排序功能，可解释性强。
    - **备选：随机森林（Random Forest）**。同样稳健，且不易过拟合。
    - **对比选项：多层感知机（MLP）**。可以验证简单的神经网络在融合特征上的表现。
- **训练流程**：
    - **数据划分**：将75名患者按比例（如80/20）划分为训练集和测试集。为了结果的稳健性，最好采用**交叉验证**（如5折或10折交叉验证）。
    - **特征缩放**：对所有数值型特征进行标准化（StandardScaler）。
    - **模型训练**：在训练集上训练您选择的模型。
    - **超参数调优**：使用网格搜索（Grid Search）或贝叶斯优化来寻找模型的最佳超参数（如树的深度、学习率等）。

**4. 评估与解释**

- **性能评估**：在测试集或通过交叉验证，评估模型的性能，指标包括：
    - **AUC (Area Under the ROC Curve)**：最重要的、综合评估模型区分能力的指标。
    - 准确率 (Accuracy), 精确率 (Precision), 召回率 (Recall), F1分数。
- **特征重要性分析（回答“为什么”）**：
    - 使用**SHAP (SHapley Additive exPlanations)分析。这是目前最先进的可解释性工具，它能告诉您，对于每一个独立的预测**，每个特征（无论是AI提取的，还是专家知识）是如何将预测结果从基线值“推”向最终值的。

| **原始数据列** | **数据类型** | **如何构造成特征** | **特征价值** |
| --- | --- | --- | --- |
| 闭塞与否 | 类别型 (是/否) | 二进制编码：is_occluded = 1 (是), 0 (否)。 | 极其重要的特征。直接决定了血流路径，是影响术后血流动力学的关键因素。 |
| 手术方式 (针对闭塞的骼脉) | 文本描述 | 独热编码 (One-Hot Encoding)：如果方式不多（如“取栓”、“溶栓”、“搭桥”），可以为每种方式创建一个特征列。例如：is_thrombectomy=1/0, is_thrombolysis=1/0, is_bypass=1/0。 | 不同的管理方式对血流动力学恢复的影响巨大，是预测远期通畅性的核心变量。 |
| 股动脉-股动脉人工血管搭桥 | 类别型 (是/否) | 二进制编码：has_fem_fem_bypass = 1 (是), 0 (否)。 | 这是一个重大的血流动力学改变，引入了新的血流路径，必须作为一个重要特征。 |
| 髂动脉支架置入术 | 类别型 (是/否) | 二进制编码：has_iliac_stent = 1 (是), 0 (否)。 | 表明了髂动脉本身存在狭窄或夹层等问题，是重要的病变信息。 |
| 取栓/溶栓 | 类别型 (是/否) | 二进制编码：is_thrombus_removal = 1 (是), 0 (否)。 (如果与“手术方式”列重复，可整合) | 表明术前存在急性血栓事件，可能与更高的炎症反应和更差的远期预后相关。 |
| 是否使用球囊扩张 (是为1，否为0) | 数值型 (0/1) | 直接使用：used_balloon = 1 或 0。 | 球囊扩张可能会对血管壁造成微小损伤，影响远期重塑，是非常有用的特征。 |
| 支架品牌 (Cook, Zenith, Medtronic) | 类别型 (多分类) | 独热编码 (One-Hot Encoding)：brand_cook=1/0, brand_zenith=1/0, brand_medtronic=1/0。 | 不同品牌的支架具有不同的柔顺性、径向支撑力和材料，这些物理特性会直接影响血流动力学。 |
| 支架的尺寸 (近端直径-远端直径-长) | 复合文本/数值 | 数值拆分：将其拆分为三个独立的数值特征：stent_proximal_diameter, stent_distal_diameter, stent_length。 |  支架的尺寸与主动脉的匹配程度（Oversizing率）直接决定了密封效果和内漏风险。 |
| 髂总近端直径 / 髂总远端直径 | 数值型 | 直接使用：cia_proximal_diameter, cia_distal_diameter。 | 这是计算支架与血管匹配度的基础数据，对于预测锚定区并发症至关重要。 |
| 抗血小板用药情况 | 类别型/文本 | 编码：可以根据用药方案进行编码。例如：drug_aspirin=1/0, drug_clopidogrel=1/0, is_dual_antiplatelet=1/0。 | 影响血栓形成和内膜增生，与远期通畅性密切相关。 |
| 共病 (除了AAA以外其他病) | 文本/多选 | 多标签二进制编码：为每种常见的共病创建一个特征列。例如：has_hypertension=1/0, has_diabetes=1/0, has_pad=1/0 (外周动脉疾病)。 | 共病情况是影响患者整体健康和远期生存的重要因素。 |

- **特征重要性分析（回答“为什么”）**：
    - 使用SHAP分析。可以分析：新模型中，**是低TAWSS_of_AAA更重要，还是AI提取的某个关于涡流的隐式特征h_global更重要**
    - **分层解释**：您可以对不同直径组的患者，分别进行SHAP分析，看看对于中等大小的AAA和超大的AAA，导致破裂的关键预测因子是否发生了变化。

对于输出的考虑，还要判断，是直接预测标签还是预测可能性

**1.预测结果（Classification）**

- **模型输出**：一个**硬性标签（Hard Label）**，例如 1 (会闭塞) 或 0 (不会闭塞)。
- **如何实现**：通常是模型内部先预测一个概率，然后根据一个固定的阈值（通常是0.5）来做出最终的“是/否”判断。
- **优点**：简单直接，易于计算准确率。
- **缺点**：
    - **信息丢失**：一个预测为“会闭塞”的患者，其真实风险可能是51%，也可能是99%。这两种情况对于临床决策来说是天差地别的，但硬性标签完全无法区分。
    - **武断与僵化**：0.5这个阈值是武断的。一个风险为49%的患者被划为“低风险”，而一个51%的患者被划为“高风险”，这在临床上是不合理的。
    - **临床不适用**：医生需要的不是一个AI来替他们做决定，而是一个**提供量化证据的工具**来辅助他们做决策。

**2. 预测风险（Risk Stratification / Probability Estimation）**

- **模型输出**：一个**连续的概率值（Soft Label）**，范围在 [0, 1] 之间。例如，输出 0.75，代表模型预测该患者有**75%的概率**会发生髂肢闭塞。
- **如何实现**：对于分类任务，我们让模型的输出停留在**Softmax层或Sigmoid层之后**，直接获取那个概率值，而**不进行最后一步的阈值判断**。
- **巨大优势**：
    - **信息完整**：完整地保留了模型对不确定性的评估。75%的风险和55%的风险可以被明确区分。
    - **支持个体化决策**：
        - 对于一个预测风险为90%的患者，医生可能会建议立即进行更积极的干预。
        - 对于一个预测风险为60%的患者，医生可能会建议加强随访频率，而不是立即干预。
        - 对于一个预测风险为20%的患者，医生可以更放心地让其进行常规随访。
    - **灵活性**：临床医生可以根据不同的临床情景，**自己设定不同的风险阈值**。例如，对于一个后果极其严重的事件，医生可能会将风险阈值设得很低（如>30%就视为高危）。
    - **评估更科学**：模型的性能不再用简单的“准确率”来衡量，而是用**AUC、校准曲线（Calibration Curve）、Brier分数**等更能评估概率预测质量的指标来衡量。

  如果要进行可能性的预测需要在预测框架中做一些微小的改动

**1. 模型架构的修改**

- 在临床预测头中：
    - 确保其**最后一层**是一个**Sigmoid激活函数**（对于二分类问题）。Sigmoid函数的作用就是将任意的输入数值，压缩到 (0, 1) 的区间内，这天然就代表了概率。
    - **模型的最终输出就是这个Sigmoid函数的结果**，而不是之后再加一个阈值判断。

**2. 损失函数的选择**

- 仍使用**交叉熵损失（Cross-Entropy Loss）**。交叉熵损失本身就是用来优化概率预测的，它会惩罚那些“模型预测概率很低，但真实发生了”和“模型预测概率很高，但真实没发生”的情况。

**3. 训练流程：保持不变**

- 整个特征工程、PI-GNN引擎的训练、特征融合等流程完全不需要改变。

**4. 评估方式的重大转变**

- **区分能力（Discrimination）**：
    - **能够将真正会发生闭塞的患者，从不会发生闭塞的患者中区分出来。**
- **校准度（Calibration）**：
    - **当把所有预测风险为70%左右的患者放在一起时，他们中确实有大约70%的人最终发生了闭塞。**
    - **校准曲线**（Calibration Plot）。一个好的校准曲线应该紧贴对角线。
- **临床实用性（Clinical Utility）**：
    - **“进行了决策曲线分析（Decision Curve Analysis, DCA）。结果表明，在合理的风险阈值范围内（例如，当医生认为风险超过20%就需要干预时），使用我们的模型进行决策，其带来的净收益远高于‘所有人都干预’或‘所有人都不干预’的策略。”**

- 先采用“处理后几何 × 多时相快照”的扩展方式快速放大样本量，优先复用组内成熟数据，避免重新CFD的高成本。

# 数据处理

1. **几何文件**：一个经过三维重建、平滑处理后的**表面网格文件**。最常见的格式是 .stl, .ply, .obj 等。这个文件本质上记录了两样东西：
    - **顶点列表 (Vertices)**：一个 N x 3 的列表，包含了 N 个顶点的 (x, y, z) 坐标。
    - **面列表 (Faces)**：一个 M x 3 的列表，记录了 M 个三角面片分别是由哪三个顶点组成的。
2. **物理场标签文件**：一个与上述几何文件**完全对应**的CFD仿真结果文件。这个文件记录了在那 N 个顶点上，每个点的物理量值。可以是一个 .csv 或 .txt 文件，格式类似于：
    - Node_ID, pred_u, pred_v, pred_w, pred_p
    - 0, 0.1, 0.2, 0.0, 1500.0
    - 1, 0.12, 0.18, 0.01, 1450.0
    - ...
3. **其他特征文件**：
    - **显式几何特征**：一个 N x F_geom_explicit 的文件，记录了您计算好的 radius, curvature 等。
    - **PointNet++特征**：一个 N x F_geom_implicit 的文件，记录了您用PointNet++提取好的局部几何嵌入。
    - **全局边界条件**：几个单一的数值，如 inlet_flow 在当前时间点的值。

### **第二部分：如何将“原始材料”加工成PyG的“图数据对象”**

PyTorch Geometric的核心，是一个名为 torch_geometric.data.Data 的数据对象。它就像一个“集装箱”，把所有与一个图相关的信息都打包在一起。我们的目标，就是把您的原始材料，填充到这个“集装箱”里。

**一个 Data 对象主要包含以下几个关键属性：**

**1. data.x：节点特征矩阵 (Node Feature Matrix)**

- **这是什么**：这就是我们之前反复讨论的 N x F_input 的矩阵。它的**每一行**，都代表**一个节点**的完整输入特征向量。
- **如何构建（对齐 pipeline 的伪代码）**：
    
    `import torch
    import numpy as np
    import pandas as pd
    
    # 假设 N 是节点数
    num_nodes = N
    
    # 1) 读入“归一化后的 CSV”（pipeline 输入：processed/normalized/result_features_*.csv）
    df = pd.read_csv(csv_path)
    coords = df[['x', 'y', 'z']].to_numpy()                     # (N, 3)
    t_feat = np.full((num_nodes, 1), t_norm)                    # (N, 1)
    geom = df[['Abscissa','NormRadius','Curvature',
               'Tangent_X','Tangent_Y','Tangent_Z']].to_numpy() # (N, 6)
    is_wall = df[['is_wall']].to_numpy()                        # (N, 1)
    bc = df[['BC_Inlet','BC_O1','BC_O2','BC_O3','BC_O4']].to_numpy()  # (N, 5)
    
    # 2) 拼接得到 x (N, 15)
    all_features = np.hstack([coords, t_feat, geom, is_wall, bc])
    
    # 6. 转换为 PyTorch Tensor
    data_x = torch.tensor(all_features, dtype=torch.float)
    # data_x 的最终维度是 N x 15`
    

**2. data.edge_index：边的连接信息 (Edge Index)**

- **这是什么**：这是告诉GNN“哪些节点是邻居”的关键信息。它是一个 2 x E 的长整型张量，E 是边的数量。
- **如何构建（对齐 pipeline 的伪代码）**：pipeline 当前使用 **kNN 构图**（不依赖网格面片）。

```python
import numpy as np
import torch
from sklearn.neighbors import NearestNeighbors

# coords: (N, 3)
nbrs = NearestNeighbors(n_neighbors=k + 1, algorithm="ball_tree").fit(coords)
_, indices = nbrs.kneighbors(coords)

# indices[:, 0] 是节点自身，忽略；其余 k 个为邻居
row = np.repeat(np.arange(coords.shape[0]), k)
col = indices[:, 1:].reshape(-1)
edge_index = torch.tensor(np.stack([row, col]), dtype=torch.long)
# edge_index 维度为 2 x (N*k)
```
    

**3. data.y：节点标签 (Node Labels / Targets)**

- **这是什么**：这是模型在训练时需要拟合的“答案”，也就是CFD的仿真结果。
- **如何构建**：codePython
    
    `# 假设 cfd_results 是从.csv文件加载的 N x 4 numpy数组 (u, v, w, p)
    # 确保已经进行了标准化！
    standardized_cfd_results = your_standardization_function(cfd_results)
    
    data_y = torch.tensor(standardized_cfd_results, dtype=torch.float)
    
    # data_y 的最终维度是 N x 4`
    

**4. data.pos：节点位置 (Node Positions)（可选）**

- **这是什么**：PyG 建议将节点的原始坐标单独存到 `pos` 便于几何计算与可视化。

  当前 pipeline 默认只保存 `x/edge_index/y`（未显式写入 `pos`），如需可额外补充。
    
    `# 假设 vertices 是从.stl文件加载的 N x 3 numpy数组
    data_pos = torch.tensor(vertices, dtype=torch.float)`
    

**最后，将所有部分打包：**

codePython

`from torch_geometric.data import Data

# 假设 data_x, edge_index, data_y, data_pos 都已准备好
single_graph_data = Data(x=data_x, edge_index=edge_index, y=data_y, pos=data_pos)

# 您现在得到了一个完整的图数据对象！
# 对于一个患者的20个时间点，您就需要创建20个这样的Data对象。
# 将所有这些Data对象保存在一个列表中，就构成了您的数据集。`

---

**每一个样本（每个患者的每个时间点）**，都执行上述的完整流程，创建一个 Data 对象。

**整个流程可以总结为**：

1. **循环遍历每个患者**。
2. 在患者循环内部，**循环遍历每个时间点**。
3. 在时间点循环内部：
    
    a. 加载该患者的**唯一几何文件**（.stl）。
    
    b. 从几何文件中提取 vertices -> data.pos。
    
    c. 从几何文件中提取 faces -> data.edge_index。
    
    d. 加载该时间点对应的**CFD标签文件** -> data.y。
    
    e. 加载/计算所有需要的**节点特征**（时空坐标、边界条件、几何特征等），将它们**拼接**起来 -> data.x。
    
    f. 用以上所有部分，创建一个 Data 对象。
    
    g. 将这个 Data 对象添加到一个总的数据集列表中。
    
4. 将最终的数据集列表保存到磁盘（例如使用 torch.save），以便后续快速加载进行训练。

### **全局属性：流体属性特征 (Fluid Property Features)**

- **特征**：
    - **血液密度 (Blood Density ρ)**：通常是一个常数（如 1060 kg/m ³）。
    - **血液粘度相关的参数**：您的CFD仿真很可能使用了非牛顿流体模型（如Carreau-Yasuda模型）。这个模型本身由一系列参数定义（如零剪切粘度μ₀、无穷剪切粘度μ∞、松弛时间λ等）。您可以将这些**参数**作为特征输入。
- **为什么重要**：虽然在您的研究中，这些参数对所有患者可能都是一样的，但将它们作为输入，可以**增加您模型的通用性**。未来，如果您想研究不同血液粘度（如贫血或红细胞增多症患者）对血流动力学的影响，您的模型将能够**直接适应**，而无需重新训练。这让您的模型更像一个通用的“物理求解器”。

**2. 物理环境特征 (Physical Environment Features)**

- **特征**：
    - **重力加速度 (Gravity g)**：虽然在主动脉这种大血管中，重力的影响通常被忽略，但从完备性的角度可以加入。
- **为什么重要**：同上，增强模型的通用性和物理完备性。

**3. “归一化”相关特征**

- **背景**：在CFD和深度学习中，我们经常使用无量纲化的参数，如**雷诺数 (Reynolds Number Re)** 和 **文策尔数 (Womersley Number α)**。
- **特征**：
    - **入口雷诺数 Re(t)**：ρ * v(t) * D / μ。其中v(t)是入口平均速度，D是入口直径。这是一个随时间变化的特征。
    - **文策尔数 α**：D/2 * sqrt(ω * ρ / μ)。其中ω是心动周期的角频率。这是一个描述搏动流特性的常数。
- **为什么重要**：这些无量纲数是流体力学中的**“通用语言”**。它们将不同尺寸、不同流速、不同粘度的流动，归一化到了一个可比较的尺度上。让模型直接学习这些无量纲数与流动模式之间的关系，可能会比学习原始的、有单位的物理量**更容易，泛化能力也更强**。

**如何实现**：

将这些**全局特征**（通常是标量），在数据预处理阶段，与我们之前讨论的边界条件一样，**复制N遍**，然后拼接到每个节点的特征向量中。

**更新后的节点特征向量 xv 可能的样子：**

xv = [x, y, z, t, flow_bc, pressure_bc, **ρ, μ₀, λ, Re(t), α**, geom_features, ...]

### **如何训练XGBoost/MLP模型**

这个训练过程，遵循的是非常经典、标准的机器学习流程。它比训练庞大的PI-GNN要**简单得多、快得多**。

我们以**“髂肢闭塞（ILO）风险预测”**任务为例，假设您已经完成了特征工程，得到了一个 75 x 542 的“超级特征表格”（75个病人，每个病人542个特征）。

**第一步：数据准备与划分**

1. **准备输入X和标签Y**：
    - X：就是那个 75 x 542 的特征矩阵。
    - Y：一个包含75个元素的向量，元素为 1 (闭塞) 或 0 (通畅)。
2. **划分训练集和测试集**：
    - **关键原则**：这是为了评估模型的最终性能。您需要将您的75个病人**完全分开**。
    - **操作**：使用 train_test_split 函数（例如，来自sklearn.model_selection），将X和Y按照病人的行，随机划分为训练集（例如，80%的病人，即60个）和测试集（例如，20%的病人，即15个）。
    - X_train, X_test, y_train, y_test = train_test_split(X, Y, test_size=0.2, stratify=Y)
    - （stratify=Y 是一个好习惯，它能保证在划分后，训练集和测试集中的“闭塞”和“通畅”病人的比例与原始数据集大致相同。）
3. **特征缩放（对MLP尤其重要）**：
    - 由于您的特征（如年龄、WSS、h_global）具有不同的数值范围，您需要对它们进行标准化。
    - **操作**：
        1. 在**训练集 X_train**上 fit 一个 StandardScaler（来自sklearn.preprocessing）。
        2. 用这个scaler去 transform **训练集 X_train** 和 **测试集 X_test**。
    - **注意**：scaler必须只在训练集上学习（fit），以防止测试集的信息泄露到训练过程中。

**第二步：模型选择与训练（以XGBoost为例）**

XGBoost 是一个非常强大的梯度提升树模型，通常是处理表格数据分类问题的首选。

1. **模型初始化**：codePython
    
    `import xgboost as xgb
    
    # 初始化一个XGBoost分类器
    model = xgb.XGBClassifier(objective='binary:logistic', eval_metric='auc')`
    
2. **模型训练**：
    - **核心操作**：调用模型的 fit 方法，将训练数据“喂”给它。
    - **代码**：codePython
        
        `model.fit(X_train_scaled, y_train)`
        
    - **发生了什么？** 在这个 fit 的过程中，XGBoost会迭代地构建一系列的决策树。每一棵新树的目标，都是去修正前面所有树预测结果的总和与真实标签之间的残差。通过这个过程，模型逐渐学会了特征与结果之间的复杂非线性关系。

**第三步：超参数调优（提升模型性能的关键）**

- XGBoost有很多超参数（如树的数量 n_estimators、学习率 learning_rate、树的最大深度 max_depth 等），它们会极大地影响模型性能。
- **目标**：找到一组能让模型在**未知数据上**表现最好的超参数组合。
- **方法：带交叉验证的网格搜索（GridSearchCV）**：
    1. **定义搜索空间**：为您想调优的超参数，定义一个候选值的网格。
    2. **交叉验证**：GridSearchCV 会在**训练集内部**进行交叉验证。例如，对于5折交叉验证，它会把训练集（60个病人）分成5份，轮流用4份训练，1份验证，对**每一组超参数组合**都这样做，最后计算出每组组合的平均验证性能。
    3. **找到最佳模型**：选择平均验证性能最好的那组超参数，用**整个训练集**重新训练一个最终模型。
    - **代码示例**：codePython
        
        `from sklearn.model_selection import GridSearchCV
        
        param_grid = {
            'max_depth': [3, 4, 5],
            'learning_rate': [0.01, 0.1, 0.2],
            'n_estimators': [100, 200]
        }
        
        grid_search = GridSearchCV(model, param_grid, cv=5, scoring='roc_auc', n_jobs=-1)
        grid_search.fit(X_train_scaled, y_train)
        
        # 最佳模型已经由 grid_search 内部训练好了
        best_model = grid_search.best_estimator_`
        

**第四步：最终模型评估**

1. **在测试集上进行预测**：
    - 现在，我们拿出那个从一开始就**完全没有碰过**的**测试集 X_test** 来模拟真实世界的新病人。
    - **获取风险概率**：使用 predict_proba 方法。codePython
        
        `y_pred_proba = best_model.predict_proba(X_test_scaled)[:, 1] # 获取类别为1的概率`
        
2. **计算评估指标**：
    - 使用 y_test（真实标签）和 y_pred_proba（预测概率）来计算我们之前讨论的所有评估指标：
        - **AUC** (roc_auc_score)
        - 绘制**ROC曲线**
        - 绘制**校准曲线**
        - 进行**决策曲线分析**

**第五步：模型解释**

- 训练完成后，使用**SHAP**库来解释您的 best_model。codePython
