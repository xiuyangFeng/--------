## 总纲

用"显式几何先验 + 物理约束的图神经网络主干 + 多模态临床融合"的多任务框架，预测时空血流动力学场（速度、压力、WSS、OSI 等），并进一步完成基于数字孪生体的临床终点风险预测（如 EVAR 后的髂肢闭塞、AAA 破裂），实现预测时空场的秒级响应。

### 目标与子任务

- **物理场重建**：基于 AAA 的点云数据集，预测时空速度与压力场，派生 WSS、OSI、RRT 等指标。
- **临床终点预测**：将图池化的全局物理表征与临床表格特征融合，输出风险概率（如年生长率、破裂、闭塞）。
- **结构优化**：物理约束、显式几何特征、GNN 消息传递。

### 模型架构概览（PI-GNN 多任务）

- **主干**：Graph Transformer，输入增强节点特征，输出节点嵌入 \(H \in \mathbb{R}^{N \times F}\)。
- **头 A（物理场）**：两个独立 MLP 分别回归速度场 \((N \times 3)\) 与压力场 \((N \times 1)\)。由此可派生 WSS/OSI/RRT。
- **头 B（临床）**：池化得到 \(h_\text{global}\)，与临床向量拼接，经 MLP 输出概率（Sigmoid）。支持风险分层与可解释性分析（SHAP、校准曲线、DCA）。

## 模块一：预测时空血流动力学场

### 1. 数据板块

#### 1.1 显式几何特征工程

目的：为复杂的动脉瘤几何建立标准化"骨架"，提取具有明确物理意义的先验几何，解决不同患者几何形态各异、难以比较的问题。

具体操作：对输入的 3D 模型（网格/点云），通过 VMTK 提取血管中心线，并为每个点计算几何特征。

**`Abscissa`（归一化弧长坐标）**

```python
geo_abscissa[i] = arr_abscissa[closest_id]
geo_abscissa = (geo_abscissa - ab_min) / (ab_max - ab_min)
```

- 表示该点在血管"长度方向"上的相对位置（0.0 = 入口端，1.0 = 出口端）。
- 作用：让模型知道该点在血管的上游、中游还是下游。

**`NormRadius`（归一化径向距离）**

```python
dist = np.linalg.norm(pt - cl_points[closest_id])
local_r = arr_radius[closest_id]
geo_norm_dist[i] = dist / local_r
```

- 表示该点在血管"横截面方向"上的相对位置（~0.0 = 靠近中心线，~1.0 = 靠近血管壁）。
- 作用：反映点在径向上的位置，即靠近管腔中心还是靠近血管壁。

**`Curvature`（曲率）**

```python
geo_curv[i] = arr_curv[closest_id]
```

- 中心线在该点处的弯曲程度（0.0 = 笔直，数值越大弯曲越厉害）。
- 作用：高曲率区域通常产生复杂的二次流或涡流，对压力和速度分布有重大影响。

#### 1.2 隐式几何特征工程（暂不使用）

使用 PointNet++ 分层级提取局部几何特征，作为 GNN 输入节点特征之一。当前标记为可选扩展模块。

#### 1.3 数据处理

**采样策略**

- 壁面全保留 + 内部点按近壁层/核心层分层采样 + 动态补位。
- 默认采样方法：`hybrid`（部分 FPS + 部分随机），目标点数 40000。
- 近壁层（距壁面 < 2.0mm）与核心层按 7:3 动态分配预算。

**坐标对齐**

- 每个时间帧独立读取、清洗、采样，不做跨帧点级配准。
- 同一病例内通过 `coord_normalize.py` 做中心化 + PCA 主轴对齐 + `[-1,1]` 缩放。
- 时间信息 `t_norm` 作为图级属性写入 `data.global_cond`。

**归一化策略**

| 特征类别 | 参数 | 方法 | 备注 |
| --- | --- | --- | --- |
| 几何拓扑 | `Abscissa` | 保持不变 | 提取阶段已归一化到 `[0, 1]` |
| | `Tangent_X/Y/Z` | 保持不变 | 单位切向量分量 |
| | `NormRadius` | Min-Max | 映射到 `[0, 1]` |
| | `Curvature` | Z-Score | 均值 0，标准差 1 |
| 物理流场 | `u, v, w, p, wss*` | Z-Score | 可选 log 变换后再标准化 |
| 边界条件 | `BC_Inlet` | × 1e5 | 将 \(10^{-5}\) 量级提升到便于训练的尺度 |
| | `BC_O1~O4` | 默认 Z-Score | 全局统计量标准化 |
| 分类标记 | `is_wall` | 保持不变 | 按数据来源赋值：`ascii/` = 1, `ascii_in/` = 0 |
| 坐标 | `x, y, z` | 病例内坐标归一化 | m→mm 后做中心化 + PCA + `[-1,1]` 缩放 |

#### 1.4 数据数量与分布

| 数据集 | 总样本 | 分组 |
| --- | --- | --- |
| 腹主动脉瘤破裂 | 82 例（破裂 41 / 未破裂 41） | 50-65mm: 22, 65-80mm: 32, ≥80mm: 28 |
| 髂支闭塞 | 72 例（闭塞 24 / 通畅 48） | — |
| AAA 生长 | 87 例（快速 25 / 缓慢 62） | — |

总计约 300 例，每例 81 个时刻快照，共 ~24000 个时空快照。

### 2. 模型架构

#### 当前图数据结构

| 属性 | 维度 | 内容 |
| --- | --- | --- |
| `data.x` | `[N, 10]` | `x, y, z, Abscissa, NormRadius, Curvature, Tangent_X/Y/Z, is_wall` |
| `data.global_cond` | `[1, 6]` | `t_norm, BC_Inlet, BC_O1, BC_O2, BC_O3, BC_O4` |
| `data.y` | `[N, 4]` | `u, v, w, p` |
| `data.edge_index` | `[2, E]` | kNN 构图（默认 k=6，双向边） |

时间和边界条件不逐点复制到 `data.x`，而是在模型前向时通过 `data.batch` 索引广播到节点再拼接。

#### GNN Backbone

采用多层 Graph Transformer，通过消息传递聚合邻居节点信息，可选引入二次残差模块（QN）。经过 L 层后，每个节点得到嵌入向量 \(h_v\)。

- **速度预测头**：独立 MLP，\(h_v \to [u, v, w]\)，输出 \(N \times 3\) 速度场。
- **压力预测头**：独立 MLP，\(h_v \to [p]\)，输出 \(N \times 1\) 压力场。

#### 结构要点

1. GNN 主干只接收描述问题的特征（不含标签）。
2. 主干输出通用节点嵌入，子头各自独立预测。
3. 两个子头输出合在一起即为完整的速度场和压力场。

### 3. 损失函数设计

当前路线不建议做“纯 PINN”，而是做 `CFD 监督主干 + PINN 物理约束增强` 的混合训练。原因是我们已经拥有大量、质量较高的 CFD 标签，数据监督应作为主优化目标，物理项用于提升泛化、稳定性和物理一致性。

$$L_\text{total} = L_\text{data} + \lambda_c L_\text{cont} + \lambda_b L_\text{bc} + \lambda_m L_\text{mom}$$

其中：

- `L_data`：主损失，直接拟合 CFD 真值场。
- `L_cont`：连续性约束，优先级最高的物理正则。
- `L_bc`：边界条件约束，首版先显式做无滑移和入口条件。
- `L_mom`：动量方程残差，价值高但数值最难调，后置。

#### 数据损失

第一版统一采用 CFD 监督，不区分“内部点是否用标签”。壁面速度虽然理论上满足无滑移，但仍建议保留 CFD 监督，让网络先学会真实离散场分布，再叠加边界正则。

$$L_\text{data} =
\lambda_u \operatorname{MSE}(\hat{u}, u_\text{CFD}) +
\lambda_v \operatorname{MSE}(\hat{v}, v_\text{CFD}) +
\lambda_w \operatorname{MSE}(\hat{w}, w_\text{CFD}) +
\lambda_p \operatorname{MSE}(\hat{p}, p_\text{CFD})$$

建议初值：

- `λ_u = λ_v = λ_w = 1.0`
- `λ_p = 0.1 ~ 0.5`

#### 物理损失

首版不建议一次性上完整 Navier-Stokes 残差，而是按稳定性递进：

1. `L_cont = ||\nabla \cdot \mathbf{u}||^2`
2. `L_bc = L_\text{no-slip} + L_\text{inlet}`
3. `L_mom = ||R_\text{NS}||^2`

其中：

- `L_no-slip`：仅在壁面点上约束 \(|\mathbf{u}|^2\)。
- `L_inlet`：如果入口边界条件已知且稳定，可对入口速度或流量做约束。
- `L_mom`：内部点的动量守恒残差，只在数据主线稳定后再接入。

#### 推荐训练顺序

不要直接做 full PINN。建议采用课程式混合训练：

1. `Stage 1`：只开 `L_data`
2. `Stage 2`：`L_data + L_cont`
3. `Stage 3`：`L_data + L_cont + L_bc`
4. `Stage 4`：`L_data + L_cont + L_bc + L_mom`

推荐把 `L_data` 视为主干，物理项仅占辅助权重。首轮可从以下范围试起：

- `λ_c = 1e-3 ~ 1e-2`
- `λ_b = 1e-3 ~ 1e-2`
- `λ_m = 1e-5 ~ 1e-3`

#### 实现要点

- 先记录每一项 loss 的原始数量级，再决定权重，不要拍脑袋设大权重。
- 优先在验证集上同时看 `val_data_loss` 和物理残差，避免“物理更小但场预测更差”。
- 当前若仍使用 `kNN` 图，物理导数只是近似正则；后续若切到 mesh-aware graph，物理项解释性会更强。
- 训练后要输出残差分布图，而不只看总 loss。

## 模块二：基于数字孪生的风险预测

### 1. 髂支闭塞（ILO）风险预测

**文献基础**：术前高 TAWSS、CIA 高扭转度、EIA 小半径，术后高 OSI、高 RRT，以及双侧 OSI 不对称（DRI）是 ILO 的核心风险因素。

#### 关键风险指标

| 类别 | 指标 | 对闭塞风险的影响 | 来源 |
| --- | --- | --- | --- |
| 解剖学 | Torsion-CIA-pre | 风险因素（扭转度越大越高） | Mimics 重建 |
| | Radius-EIA-pre | 保护因素（半径越大越低） | Mimics 测量 |
| | Torsion-CIA-DRE-DRI | 风险因素（差异比越大越高） | 术前/术后计算 |
| 血流动力学 | TAWSS-pre | 风险因素（数值越高越高） | CFD 模拟 |
| | OSI-pre-DRI | 风险因素（差异比越大越高） | CFD + 双侧计算 |
| | RRT-IA-post | 风险因素（数值越高越高） | CFD 模拟 |
| | RRT-DRE | 风险因素（变化比越大越高） | 术前/术后计算 |
| 衍生差异 | OSI-post-DRI | 风险因素（差异比越大越高） | CFD + 双侧计算 |

#### 特征组设计

- **组 A：AI 驱动的血流动力学特征**
  - \(h_\text{global}\)：图池化后的全局深度表征。
  - 区域显式特征：左右髂动脉区域的 mean_WSS、high_OSI_area_ratio 及双侧不对称性指标。
- **组 B：知识驱动的验证性特征**
  - 复现文献关键风险因子：OSI-pre-DRI、RRT-IA-post、TAWSS-pre、Radius-EIA-pre、Torsion-CIA-DRE-DRI。
- **组 C：基础临床与解剖特征**
  - 人口统计学、常规解剖指标、手术信息、合并症。

#### 临床特征编码参考

| 原始数据 | 编码方式 |
| --- | --- |
| 手术方式 | One-Hot |
| 股-股人工血管搭桥 | 二进制 |
| 支架品牌 (Cook/Zenith/Medtronic) | One-Hot |
| 支架尺寸 | 拆分为近端直径、远端直径、长度 |
| 球囊扩张 | 二进制 |
| 抗血小板用药 | 多标签二进制 |
| 共病 | 多标签二进制 |

### 2. AAA 破裂风险预测

**文献基础**：AAA 低 TAWSS、高 OSI/RRT，AA 高曲折度，CIA 小最大直径是核心风险因素。不同直径 AAA 的关键预测因子不同（50-65mm 组为 OSI，65mm 以上组为 TAWSS/RRT）。

#### 关键风险指标

| 类别 | 指标 | 对破裂风险的影响 | 适用直径范围 |
| --- | --- | --- | --- |
| 形态学 | AA 迂曲度 | 风险因素（越高越大） | 全组 |
| | CIA 最大直径 | 保护因素（越大越低） | 全组、≥80mm |
| | 瘤颈长度 | 风险因素（越短越大） | 全组 |
| 血流动力学 | TAWSS of AAA | 保护因素（越低越高） | 全组、65-80mm、≥80mm |
| | OSI of AAA | 风险因素（越高越大） | 50-65mm |
| | RRT of AAA | 风险因素（越高越大） | 65-80mm、≥80mm |
| | TAWSS of IA | 风险因素（越高越大） | ≥80mm |

#### 特征组设计

- **组 A**：\(h_\text{global}\) + AI 衍生的区域血流指标。
- **组 B**：低 TAWSS、高 OSI/RRT、AA 曲折度、CIA 最大直径 + 直径分组变量。
- **组 C**：人口统计学、解剖指标、手术信息、合并症。

### 3. 风险模型通用设计

#### 输出策略

输出连续概率值 \(p(y=1) \in [0, 1]\)（Sigmoid），而非硬标签。

- 信息完整：75% 和 55% 的风险可被明确区分。
- 临床灵活：医生可自行设定风险阈值。

#### 模型选择

- 首选：XGBoost / LightGBM（表格数据首选，可解释性强）。
- 备选：Random Forest、Logistic Regression、MLP。

#### 评估体系

- **区分能力**：AUROC、AUPRC、F1、Sensitivity、Specificity。
- **校准度**：Brier Score、校准曲线。
- **临床实用性**：DCA（决策曲线分析）。
- **可解释性**：SHAP 全局特征重要性 + 个体病例解释。

## 风险点与对策

| 风险 | 对策 |
| --- | --- |
| 物理项导致训练不稳 | 预热/课程学习，先数据项后逐步增大物理权重；规范化残差尺度 |
| 标签噪声与域移 | 统一重建与光滑流程，严格对齐时间与边界条件；对不同来源数据做归一与分层评估 |
| 数据增强的真实性 | 避免与临床标签绑定的"伪患者"；统计形状建模仅用于物理预训练 |

以下内容仅在主线闭环成立后考虑：

- **PointNet++ 隐式几何编码**：学习局部几何嵌入替代/增强显式特征。
