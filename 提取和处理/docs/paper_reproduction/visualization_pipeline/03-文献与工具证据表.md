## 3. 对当前项目的具体判断

1. **点云架构的关键不是“回构面片”，而是不要丢失原面片到采样点的映射。** PointNetCFD、MeshGraphNet、X-MeshGraphNet、PI-GNN coronary WSS 都是逐点/逐节点输出物理量，再绑定到已有几何承载上。
2. **最适合本项目论文图的通用格式是 VTP/VTU。** 它能携带几何拓扑、point data、cell data 和单位说明，也方便 PyVista/ParaView 批处理。
3. **CFD-Post 交付应走 CSV surface/point cloud。** 官方文档明确支持导入 CSV 并生成 surface 或 point cloud；这比尝试改写 Fluent case 更直接。
4. **Fluent 回写只作为后续增强。** 如果需要把预测值作为 Fluent 内变量，必须通过 UDM/UDF/profile 等机制，并解决 face/node 对齐；第一阶段不应把它作为硬依赖。
5. **WSS 从 velocity 后处理必须先做 CFD oracle。** 不能只因为速度/压力云图平滑，就默认能恢复壁面剪切应力。

## 4. 后续需继续补证的方向

- 定位每个外部 baseline 是否公开了可直接导出 `.vtp/.vtu/.csv` 的后处理脚本。
- 核查本项目现有 `.pt/.npz/.csv` 数据是否保留了原始 `node_id/face_id`。
- 若数据只保留 xyz，需要先写一个 mapping audit，量化最近邻距离和跨区域错配风险。
- 若老师指定必须在 Fluent 内查看，需单独建立 UDM/UDF 最小样例，不应和论文图批处理混在一起。
