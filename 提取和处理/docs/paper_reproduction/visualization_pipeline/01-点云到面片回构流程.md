# 点云到面片回构流程

## 1. 先分清三类“点”

| 点集类型 | 典型来源 | 能否直接回面片 | 推荐处理 |
| --- | --- | --- | --- |
| 原始 surface vertex | Fluent/CFD-Post/VTK 导出的壁面节点 | 可以 | 保留 `node_id`，预测值直接写回 point data |
| surface face center | 壁面面片中心、WSS 标签常见位置 | 可以，但要按 face data 写入 | 保留 `face_id`、面积、法向，画图时做 cell data |
| 重采样 point cloud | FPS、均匀采样、固定点数输入 | 不能无损直接回原面片 | 需要保存采样到原 mesh 的映射，或做最近邻/插值 |

本项目后续所有点云数据导出都应优先保留以下字段：

```text
case_id, point_id, source_node_id, source_face_id,
x, y, z, nx, ny, nz, area, region,
pressure_true, wss_true, pressure_pred, wss_pred
```

`source_node_id` 和 `source_face_id` 可以为空，但不能完全不记录。只保存无序 xyz 会让后续回映射变成不稳定的几何猜测。

## 2. 最稳方案：ID join

适用条件：模型输入点就是原始 surface vertex 或 face center 的子集/全集。

流程：

1. 从 CFD surface 导出 `node_id/face_id + xyz + normal + area + true fields`。
2. 训练、验证、预测全过程保留同一 `node_id/face_id`。
3. 预测结束后按 ID 合并 `pred_*`。
4. 写回 `.vtp` 的 point data 或 cell data。
5. 画图时同一 target surface 上比较 `true_wss`、`pred_wss`、`abs_error`。

优点是没有插值误差，指标和展示可以统一。缺点是要求前处理阶段就把 ID 管住。

## 3. 次稳方案：源 mesh 到目标点的 VTK probe/sample

适用条件：源数据有 cell topology，例如 `.vtu` 体网格、`.vtp` 表面网格。

流程：

1. 把源 CFD 或预测结果写成带变量数组的 VTK mesh。
2. 把目标 surface / slice / centerline 写成 VTK dataset。
3. 用 VTK `ProbeFilter`、PyVista `sample()` 或 ParaView `Resample With Dataset` 从 source 采样到 target。
4. 记录 `vtkValidPointMask` 或等价命中标记。

优点是插值遵循源单元拓扑，适合体场切面、中心线和表面重采样。缺点是需要源数据有真实拓扑。

## 4. 可用但要降级标注：最近邻 / IDW / RBF

适用条件：源数据只有散点，没有 cell topology。

| 方法 | 使用建议 | 主要风险 |
| --- | --- | --- |
| 最近邻 | 快速预览、检查坐标是否对齐 | 分叉、薄壁或近邻跨面会错配 |
| kNN + IDW | 论文云图展示可用 | 会平滑 hotspot，p95/peak WSS 会变低 |
| RBF | 局部 patch 或切面小样本 | 大点数内存高，外推不可靠 |
| Delaunay | 切面或局部 3D 插值 | 大规模血管点云容易退化 |

这些方法得到的 surface 云图原则上只作为展示。若要在插值后统计区域均值或 p95，必须报告插值参数和映射距离分布。

## 5. 面片统计要做面积加权

如果预测值被映射到 surface mesh 的 cell data，区域均值应按面积加权：

```text
mean_region = sum(value_face * area_face) / sum(area_face)
p95_region = weighted_percentile(value_face, area_face, 0.95)
```

否则高密度采样区域会被重复加权，导致热点、狭窄区域或分叉区域的统计偏移。

## 6. WSS 的特殊边界

WSS 不等同于“壁面速度插值值”。如果模型输出 velocity 后再想恢复 WSS，需要单独建立后处理链：

1. 在 CFD 真值 velocity 上用同一法向采样方法反算 WSS。
2. 与 Fluent/CFD 原生 wall shear 比较，先验证 oracle 是否可信。
3. oracle 过线后，才把同一方法用到模型预测 velocity。
4. 对每个壁面点沿法向采样多个近壁点，拟合切向速度梯度。
5. 乘以动力黏度得到 WSS，并与原生 WSS 同点比较。

如果 oracle 本身误差大，不能把 WSS 恢复失败归因于网络。

## 7. 本项目建议的文件产物

| 文件 | 用途 |
| --- | --- |
| `case_surface_base.vtp` | 原始壁面几何、节点/面片、法向、面积 |
| `case_prediction_points.csv` | 模型输出点、ID、预测变量 |
| `case_surface_pred.vtp` | 已写入预测变量的目标 surface |
| `case_mapping_report.json` | 映射方法、参数、距离、命中率 |
| `case_cfdpost_surface.csv` | 给 CFD-Post 导入的 surface/point cloud |

## 8. 最小验收标准

- 同一 case 的坐标单位明确为 m 或 mm。
- surface 法向方向一致，不能混入内外翻转。
- 预测点数、目标面片点数、命中点数可追踪。
- `true` 与 `pred` 使用同一 target geometry 出图。
- 误差指标优先来自插值前同点结果；插值后指标必须单独标注。
