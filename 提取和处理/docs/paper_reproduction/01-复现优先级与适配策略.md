# 复现优先级与适配策略

## 1. 基本原则

外部 baseline 的目标不是把当前 V3 小修小补，而是回答：

> 在相同私有数据、相同 split、相同 WSS/TAWSS/病例级指标下，哪类公开模型家族最可能突破当前点级 WSS 精度天花板？

因此每个 baseline 必须固定：

- 同一训练/验证/测试病例划分；
- 同一归一化与反归一化记录；
- 同一 wall / near_wall / interior mask；
- 同一 WSS 点级、病例级、区域级指标；
- 同一可视化映射口径。

## 2. P0：先跑能最快形成论文 baseline 的模型

### 2.1 MultiViewUNet AAA TAWSS

**目的**：给 G4 2D 壁面展开提供外部非图 baseline。
**适配思路**：

- 不从 3D 全场入手，先只做壁面 TAWSS；
- 使用本项目 centerline / Abscissa / branch 信息构建 2D 展开；
- 输出映射回 3D 壁面点后再算指标，禁止只报 2D grid 指标。

**Go 条件**：3D 壁面点 TAWSS 指标优于当前点云路线，且反映射 gap 可控。

### 2.2 Coronary mesh convolution / SE(3) artery wall WSS

**目的**：建立成熟代码可跑的 artery wall mesh WSS baseline。
**适配思路**：

- 先审计 `sukjulian/coronary-mesh-convolution` 输入 mesh 和标签格式；
- 用本项目 STL/壁面点构建 surface mesh 或 surface graph；
- 输出 WSS vector 后与本项目 wall WSS 分量对齐；
- 同时保留 DiffusionNet baseline，作为非等变 surface baseline。

**Go 条件**：能稳定吃入本项目壁面 mesh/point，并在 `wss_x/y/z` 分量上给出可解释信号。

### 2.3 LaB-GATr

**目的**：作为 E(3)-equivariant AAA WSS 论文的底座或替代实现。
**适配思路**：

- 优先检查 point-cloud pooling/tokenisation 与原分辨率插值接口；
- 如果 AAA WSS 作者仓库代码不完整，可先用 LaB-GATr 搭建等变 surface baseline；
- 与 PointNeXt/V3P 结果分表，禁止把架构换轨写成 V3 小优化。

**Go 条件**：同 split 下 WSS 点级或分量指标超过 post5463 band，且推理/显存可接受。

### 2.4 E(3)-equivariant AAA WSS

**目的**：直接挑战本项目最核心的 AAA WSS 终点。
**适配思路**：

- 优先定位作者代码和输入格式；
- 若目标仓库代码不完整，先记录为“paper-specific code incomplete”，再转 LaB-GATr 底座；
- 若代码依赖 surface mesh，先从 STL/壁面点建立最小 surface graph；
- 输出必须在物理空间报告 WSS / TAWSS / OSI，不只看归一化 loss。

**Go 条件**：能在本项目私有数据上稳定输出壁面 WSS，并且横向/周向分量不再长期接近 0。

### 2.5 PointNetCFD

**目的**：建立“不建图、只用点云”的公开代码 baseline。
**适配思路**：

- 输入：`x,y,z,t,BC,geometry,is_wall`，先做最小版本 `x,y,z,t,BC,is_wall`；
- 输出：先对齐 V1 的 `u/v/w/p`，再决定是否加 WSS head；
- 指标：先看 `near_wall` 和 `interior`，再进入 WSS/TAWSS。

**Go 条件**：比当前内部 MLP/PointNeXt 简单基线更稳，或能以低成本复现速度/压力场。

## 3. P1：需要 mesh 或物理约束接口确认后再跑

### 3.1 MeshGraphNet

**适配前置**：必须明确本项目能否提供真实 mesh connectivity 或可审计的替代拓扑。
**不建议**：直接把当前 kNN 图包装成 MeshGraphNet baseline，这会重复 V1 的物理邻接问题。

### 3.2 PINGS

**价值**：医学血管、4D flow MRI、physics-informed。
**风险**：它的 physics loss 与 PointNet++/GNN 结构有关，不能直接替代本项目 V1/V3 中的 GNN autograd PINN 口径。

### 3.3 Lannelongue et al. physics-constrained IA GNN

**价值**：105 例公开颅内动脉瘤 benchmark dataset，与“医学血管 transient hemodynamics”高度相关。
**风险**：若代码不可得，只能先作为引用和数据集对照。

### 3.4 AneuG-Flow / IA WSS benchmark

**价值**：外部颅内动脉瘤 CFD/WSS 数据和 benchmark 可帮助校准“医学血管 WSS surrogate”社区口径。
**风险**：IA 与 AAA 的几何、边界条件和风险终点不同，不应把 IA 结果直接写成本项目 AAA 结论。

## 4. P2：作为策略或讨论，不抢第一轮 GPU

- MeshMask：优先作为 masked pretraining 策略，等 P0/P1 模型能跑通后再做二阶段增强。
- cerebrovascular PointNet PINN：用于解释“小样本点云 + PINN + v/p”的参考价值，同时明确其没有直接解决 WSS。
- DeepONet / FNO：用于 Discussion，不作为第一轮私有数据 baseline。
