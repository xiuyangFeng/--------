## 2. 编写过程记录

### Step 1：盘点现有 training 代码

已确认当前 `training/` 已具备以下基础能力：

- `train_field.py`：单实验训练入口
- `trainer.py`：基础训练循环、早停、best checkpoint
- `eval_field.py` / `predict_field.py`：独立评估与预测导出
- `configs/field/`：4 组初始 baseline 模板

当前主要缺口：

- 文档里的实验矩阵还没有代码化
- 实验索引字段偏少，不足以支撑后续大规模实验记录
- 训练日志能看到 loss，但对“收敛速度 / 是否刷新 best / 实验组标签”支持还不够

### Step 2：补实验计划生成层

新增：

- `training/plan.py`
- `training/make_field_plan.py`
- `training/run_field_plan.py`

本轮已固化的实验组：

- baseline：`A-Base-01` 到 `A-Main-01`
- 输入特征消融：`A-Abl-01-*`
- 几何分量消融：`A-Abl-02-*`
- 增强策略消融：`A-Abl-04-*`

额外预留：

- 坐标归一化消融通过 `--coord-variant name=subdir` 接入，不强绑当前本地目录结构

这样做的原因：

- 避免继续手写大量 JSON
- 让实验 ID、命名、seed 和实验组口径统一
- 后面批量上服务器时可以直接用生成好的配置集
- manifest 可以直接喂给批量执行入口
