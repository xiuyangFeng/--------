## 1. 文档定位

本清单用于执行“血流场重建”阶段的全部核心实验。目标不是继续罗列更多 backbone，而是把任务 A 固定成一条可复现、可归因、可支撑论文叙事的实验路线。

适用范围：

- 节点级速度场和压力场预测
- 精度对比
- 效率对比
- 必做消融
- 可选扩展

## 2. 任务定义

### 2.1 输入定义与特征索引

根据 PyG 数据集，送入模型的特征组装应遵循以下硬性约束（拼接后为 16 维）：

- **节点特征 (`data.x`, 10维):**
  - `[0:3]`: `x, y, z` (空间坐标)
  - `[3:9]`: `Abscissa, NormRadius, Curvature, Tangent_X, Tangent_Y, Tangent_Z` (显式几何先验)
  - `[9]`: `is_wall` (节点边界标签: 0 为内部，1 为壁面)
- **图级条件 (`data.global_cond`, 6维):**
  - `[0]`: `t_norm` (归一化时间)
  - `[1:6]`: `BC_Inlet, BC_O1, BC_O2, BC_O3, BC_O4` (边界条件)
  - **网络内注入方式**: `global_cond = batch.global_cond[batch.batch]`, 然后 `torch.cat([batch.x, global_cond], dim=-1)`。

### 2.2 输出定义与目标函数

- **目标预测 (`data.y`, 4维):**
  - `[0:3]`: `u, v, w` (速度)
  - `[3]`: `p` (压力)
  - 主线先使用 CFD 监督损失：`L_data = α*MSE(pred_u, u) + α*MSE(pred_v, v) + α*MSE(pred_w, w) + β*MSE(pred_p, p)`，建议 `α=1.0, β=0.1~0.5`。
  - 物理项不与第一版 baseline 同时耦合，而是在主线稳定后按 `continuity -> no-slip -> momentum` 递进引入。

### 2.3 目标

在患者级泛化前提下，实现高精度、稳定、快速的血流场重建，并明确回答以下问题：

1. 图结构是否优于点级模型。
2. 显式几何特征是否带来稳定增益。
3. 边界条件和 `is_wall` 是否有必要。
4. 坐标对齐和增强是否提升跨病例泛化。
5. 物理约束是否带来额外收益。
6. 在 4 万点大图场景下，多尺度层次化建模是否比单尺度消息传递更优。
7. 精度提升究竟来自 backbone 更强，还是来自层次化表示与更合理的图结构设计。
8. 在大量 CFD 标签存在时，混合监督是否比直接 full PINN 更稳。

7. 归一化统计量是否只用训练集计算。
8. 损失函数中 `u,v,w,p` 的权重是否写死在配置中。
9. 评估脚本是否能单独读取 checkpoint 对验证/测试集复算指标。
10. 首个 smoke test 是否能在 1 个 seed、1 个 epoch 下完整跑通训练到评估闭环。

## 4. 实验组织原则

任务 A 的实验必须按“由简到繁、由主问题到扩展问题”的顺序组织：

1. 先证明图建模必要。
2. 再证明显式几何先验必要。
3. 再证明泛化增强和坐标标准化是否必要。
4. 再判断物理约束是否值得保留。
5. 最后处理 4 万点快照下的效率瓶颈，验证多尺度层次化结构是否值得成为主线模型。

禁止的做法：

- 一次同时改 backbone、输入、图结构和损失，导致无法归因。
- 只汇报整体误差，不汇报效率和区域指标。
- 用“更复杂模型更准”代替“问题被清楚回答”。

### 4.1 单个实验的标准执行流程

任务 A 从现在开始，建议所有实验都按同一模板执行。这样你本周开始跑的时候不会混乱。

#### Step 1：创建实验卡

在正式训练前，先写清楚下面 8 项：

- `Exp ID`
- 研究问题
- 唯一变化项
- 固定项
- 使用的数据版本
- 使用的 split 版本
- 使用的 seed
- 预期判断标准

如果一句话说不清“这个实验相比上一组只改了什么”，说明设计还不够干净。

#### Step 2：先跑 smoke test

每个新模型或新配置先做一次小规模闭环测试：

- 只跑 1 个 seed
- 只跑少量 epoch
- 只验证代码正确性，不看最终精度
- 检查是否出现：
  - loss 为 `nan`
  - 输出维度错误
  - `global_cond` 广播错误
  - `edge_index` 未生效
  - 评估脚本读取 checkpoint 失败

通过后才能进入正式训练。

#### Step 3：正式训练

正式训练时建议固定下面流程：

1. 先跑 `seed=1`，确认方向和日志正常。
2. 再补 `seed=2,3`。
3. 训练时同步记录：
   - 最优验证 epoch
   - 最优验证指标
   - 最终测试指标
   - 单 epoch 时间
   - 峰值显存
4. 训练中如果连续多次出现不稳定，不要继续补 seed，先排查。

#### Step 4：统一评估

每组实验训练完后，必须用同一评估脚本统一导出：

- 整体误差
- 壁面点误差
- 内部点误差
- 高曲率区域误差
- 分叉区域误差
- 推理时间
- 显存占用

不能接受“训练日志里看起来差不多”这种结论，必须落成表。

#### Step 5：写一句话结论

每个实验完成后必须写一句只能回答一个问题的结论，例如：

- 图结构相对 MLP 在患者级测试集上稳定降低速度模长误差。
- geometry 主要改善高曲率区域而不是整体平均误差。
- physics loss 提升不稳定，不值得进入主线。

如果一句话里同时出现三个结论，通常说明实验归因还不够干净。

### 4.2 每组实验的通过标准

为了避免本周陷入“跑了很多但没有能用结果”，建议给每类实验都设门槛。

- `基线实验通过`：至少 1 个 seed 完整跑通，测试集指标可读，日志完整。
- `可进入主表`：至少 3 个 seed，均值和标准差可计算。
- `可进入论文结论`：不仅整体指标有差异，而且方向在多个 seed 上一致。
- `可进入后续任务`：已经回答对应研究问题，不再继续无节制调参。

## 5. 基线实验清单

以下 5 组是任务 A 第一批必须跑通的实验，其中前 4 组用于建立现有主线，第 5 组用于回答 4 万点大图场景下的层次化建模价值。

## 5.1 A-Base-01：Point-wise MLP

### 目的

建立无图结构下限。证明只靠点自身特征能学到何种程度。

### 输入配置

- `coords + t_norm + BC`
- **代码做法**: 取 `data.x[:, 0:3]` (Coords) + 广播后的 `data.global_cond`。送入维度: 3 + 6 = 9维。其余几何特征与 `is_wall` 屏蔽。

### 模型结构建议

- `Linear(9, Hidden) -> ... -> Linear(Hidden, 4)`

### 固定条件

- 不使用 `edge_index` 进行网络传播
- 不使用显式几何 (`x[:, 3:9]`)
- 不使用物理约束

### 输出记录

- `RMSE_u`, `RMSE_v`, `RMSE_w`
- `RMSE_|v|` (速度模长误差)
- `RMSE_p`, `R2_p`
- 推理时间

### 执行步骤

1. 先单独检查 MLP 输入是否只包含 `coords + global_cond`。
2. 确认 DataLoader 仍然能正常返回 `batch.y`，但模型前向不使用 `edge_index`。
3. 跑 `seed=1` 的 smoke test，确认纯点级路径无报错。
4. 完成正式训练后，把它作为任务 A 的最低下限，不再反复调参。

### 通过标准

- 能稳定收敛，不出现严重过拟合或 loss 爆炸。
- 结果可作为后续所有图模型的参照下限。
- 输出目录和指标字段与其他模型完全一致。

## 5.2 A-Base-02：GraphSAGE 或 GCN

### 目的

验证“图（Graph）连接”建模本身是否有收益，是否优于点级模型。

### 输入配置

- `coords + t_norm + BC + is_wall`
- **代码做法**: 取 `data.x[:, [0,1,2,9]]` (Coords+Wall) + 广播后的 `data.global_cond`。送入维度: 4 + 6 = 10 维。

### 固定条件

- 必须传入 `edge_index` 做消息传递。
- 不使用显式几何 (`x[:, 3:9]`)
- 不使用物理约束

### 执行步骤

1. 优先选你当前代码里最稳的 `GraphSAGE` 或 `GCN`，不要一开始两者都开。
2. 确认图结构使用和主模型相同的图构建版本。
3. 先验证 `edge_index` 已真正参与前向传播，而不是名义上传入。
4. 训练完成后，第一时间与 `A-Base-01` 对比，先回答“图是否有必要”。

### 通过标准

- 相对 `A-Base-01` 至少在一类核心指标上有稳定改善，或者明确证明图收益有限。
- 作为“图建模本身”的证据组，不能混入 geometry、physics 等额外变量。

## 5.3 A-Base-03：Transformer without geometry

### 目的

控制 backbone 水平与主模型一致，以此来绝对隔离几何特征（NormRadius等）所带来的贡献。

### 输入配置

- `coords + t_norm + BC + is_wall`
- **代码做法**: 同 A-Base-02，10 维输入。

### 固定条件

- Backbone 与 `A-Main-01` 主模型结构完全保持一致（层数、隐藏维度、注意力头数等）。
- 依然屏蔽显式几何提取通道 `x[:, 3:9]`。

### 执行步骤

1. 直接复用主模型代码，只改输入掩膜，不要新写一套 Transformer。
2. 确认除输入维度外，其余超参数与 `A-Main-01` 完全一致。
3. 训练后立刻与 `A-Main-01` 做一对一比较，这一组就是 geometry 的直接对照。

### 通过标准

- 成为后续所有 geometry 结论的主参照组。
- 如果该组与 `A-Main-01` 差异不稳定，优先复核特征注入和坐标标准化，而不是继续加新模块。

## 5.4 A-Main-01：Transformer with geometry

### 目的

作为任务 A 第一阶段的高精度主模型。验证显式几何与GNN复合结构的优越性。

### 输入配置

- `coords + t_norm + BC + geometry + is_wall`
- **代码做法**: `data.x` 全部特征 + `data.global_cond`。送入全维度：10 + 6 = 16维。

### 固定条件

- 使用当前 pipeline 的 KNN 主图结构
- 使用病例级坐标对齐（`coord_normalize.py`输出）
- 使用默认增强策略（旋转+平移开启）

### 执行步骤

1. 先把它定义为“单尺度主线基座”，后续单尺度消融都从它派生。
2. 完成 `seed=1` 后，先检查：
   - geometry 是否真正拼接进入模型
   - `is_wall` 是否被正确使用
   - 默认增强是否只作用于训练集
3. 再补齐 `seed=2,3`，形成第一版主结果。
4. 把这组结果作为第 7 节单尺度消融的共同起点。

### 通过标准

- 稳定优于 `A-Base-03`，否则不能宣称 geometry 有效。
- 近壁区域或高曲率区域至少有一类误差改善。
- 训练、评估、落盘流程完整，能作为阶段 A 的主模型。

## 5.5 A-Base-04：Hierarchical GNN

### 目的

验证在 4 万点大图条件下，多尺度层次化建模是否能提供更优的精度-速度-显存折中。

### 推荐结构

- 多尺度编码器
- 粗尺度消息传递
- 细尺度特征传播与解码

### 推荐实现方向

- PointNet++ 风格下采样 + 图消息传递
- 或多分辨率图 U-Net 风格 encoder-decoder

### 输入配置

- `coords + t_norm + BC + geometry + is_wall`

### 固定条件

- 与 `A-Main-01` 保持同一训练预算
- 只改变层次化表示与消息传递方式
- 首轮不叠加 physics loss

### 执行步骤

1. 不要一开始追求最优 hierarchy，先做最小可运行版本。
2. 首版建议只验证三件事：
   - 显存是否下降
   - 推理是否提速
   - 精度是否没有明显崩坏
3. 如果首版已经显著退化，先检查采样和上采样链路，不要马上增加层数或模块。

### 通过标准

- 至少形成一组和 `A-Main-01` 可直接比较的精度-速度-显存数据。
- 能支持你后续决定“hierarchy 值不值得继续做”。

## 5.6 A-Base-05：MeshGraphNet (Encode-Process-Decode)

### 目的

作为文献对照基线，复现 Pfaff et al. (ICLR 2021) 的 Encode-Process-Decode 架构，验证该经典物理仿真代理模型在本任务上的表现。

### 输入配置

- `coords + t_norm + BC + geometry + is_wall`
- 与 `A-Main-01` 保持相同输入

### 模型结构

- Node encoder MLP + Edge encoder MLP
- 多层 message passing (node update + edge update)
- Node decoder MLP
- Edge features: 相对位移 + 距离

### 固定条件

- 与 `A-Main-01` 保持同一训练预算和隐藏维度
- 不使用物理约束

### 通过标准

- 能与 `A-Main-01` 做公平精度-速度对比
- 提供是否需要显式 edge encoding 的证据

## 5.7 A-Base-06：PointNet++ style

### 目的

验证经典点云深度学习方法（Qi et al., NeurIPS 2017）在血流场重建任务上的竞争力，测试 local max-pooling 聚合是否能替代显式消息传递。

### 输入配置

- `coords + t_norm + BC + geometry + is_wall`

### 模型结构

- Per-point feature projection
- 多层 local neighbourhood max-pooling + MLP
- Per-point output projection

### 固定条件

- 与 `A-Main-01` 保持同一训练预算
- 使用相同图结构定义邻域

### 通过标准

- 能回答 "显式消息传递 vs 简单邻域聚合" 的问题
- 如果性能接近 Transformer, 说明聚合方式不是瓶颈

## 7. 必做消融实验

### 7.0 消融实验统一要求

所有消融都遵守以下规则，否则结果不进入最终表格：

- 每次只改一个因素
- 默认从 `A-Main-01` 或 `A-Base-04` 派生
- 图构建、训练预算、评估脚本全部固定

## 7.1 A-Abl-01：输入特征层级消融 (优先级极高)

### 研究问题

边界条件、几何特征和 `is_wall` 分别带来什么贡献。

### 实验组及代码做法
**建议在 Dataset / Model 前向传播时使用一个 boolean 掩膜控制以下特征维度组装：**
1. `coords + t`: 仅合并 `x[:, 0:3]` + `global_cond[:, 0:1]` (强制屏蔽后面所有，送入 4 维)。
2. `coords + t + BC`: 增加 `global_cond[:, 1:6]` (回归 9 维)。
3. `coords + t + BC + is_wall`: 增加 `x[:, 9]` (回归 10 维，同 `A-Base-03`)。
4. `coords + t + BC + geometry`: 增加 `x[:, 3:9]`，屏蔽 `is_wall` (送入 15 维)。
5. **Full (A-Main-01)**: 全部拼接 (16 维)。

### 控制变量

- Backbone 固定为同一版本 Transformer
- 图构建固定
- 训练预算固定

### 结果要求

- 至少报告整体误差
- 额外报告壁面与内部点误差

### 执行建议

这一组优先级最高，建议你这周 baseline 跑完后第一时间做：

1. 先跑 `coords + t`
2. 再补 `+BC`
3. 再补 `+is_wall`
4. 最后对比 `+geometry` 和 Full

这样你会最快知道：

- 边界条件是否必要
- `is_wall` 是否真正有效
- geometry 的收益是不是独立存在

## 7.2 A-Abl-02：显式几何分量消融 (论证明确的几何因素)

### 研究问题

哪类几何先验真正有效，模型究竟学到了管型扩张还是壁面弯曲的影响？

### 实验组及代码做法

保留能力极强的基座 (等同 A-Main-01)，仅将对应的列手工置 0 (Mask) 或不拼接：
1. Full geometry (16 维)
2. 去掉 `Abscissa` (屏蔽/Mask `x[:, 3]`)
3. 去掉 `NormRadius` (屏蔽/Mask `x[:, 4]`)
4. 去掉 `Curvature` (屏蔽/Mask `x[:, 5]`)
5. 去掉 `Tangent` (屏蔽/Mask `x[:, 6:9]`)

### 结果要求

- 报告总体误差
- 报告高曲率区域误差
- 报告近壁区域误差

### 执行建议

建议只做“去掉一组”的减法实验，不要同时尝试多种组合，否则组合数会迅速失控。

## 7.3 A-Abl-03：坐标归一化消融

## 7.4 A-Abl-04：增强策略消融 (泛化论证)

### 研究问题

动态的数据增强手段（坐标扰动）是否改善了测试集患者级泛化能力。

### 实验组及代码做法

利用 pipeline 中 `dataset.py` 内部的 `CFDAugmentedDataset` 进行设置控制：
1. **无增强**: `augment=False`
2. **旋转**: `augment=True`, 概率 0.5 绕三轴旋转 (关闭平移 `translation_prob=0.0`)
3. **旋转 + 平移**: `augment=True`, 恢复默认旋转及局部位移率 (0.5/0.1)。
4. **旋转 + 平移 + 小尺度扰动**: `augment=True`, 在 3 基础上额外添加坐标微扰（幅度 ≤ 0.01）。

*(注意：微小缩放被验证会改变雷诺数等物理一致性，不建议轻易尝试。)*

### 结果要求

- 报告平均性能
- 报告不同 seed 下稳定性

### 执行建议

如果增强对均值提升不大，但明显降低不同 seed 波动，仍然值得保留。

## 15. 结果呈现与统计规范（2026-03-14 补充）

### 15.1 数值精度规范

| 指标类型 | 保留位数 | 示例 |
|---------|---------|------|
| RMSE (各变量) | 4 位小数 | 0.0123 |
| MAE | 4 位小数 | 0.0098 |
| R² | 3 位小数 | 0.987 |
| p-value | 3 位小数或科学计数法 | 0.003 或 < 1e-3 |
| 参数量 | 千/万级取整 | 125K |
| 推理时间 | 2 位小数 (ms) | 12.34 ms |

### 15.2 统计报告格式

- 所有主结果：`mean ± std` (至少 3 seeds)
- 消融对比：附 paired t-test p-value 和 Cohen's d
- 如果 seed 数 < 5：优先用 Wilcoxon signed-rank test
- Bootstrap 95% CI (n=10000) 补充不确定性量化
- 显著性标记：`*` p<0.05, `**` p<0.01, `***` p<0.001

### 15.3 Per-Case 指标分布

- 每个基线/消融实验应报告 per-case RMSE 的 boxplot
- 识别最差 case 并分析原因（几何复杂度、网格质量、BC 异常）
- 推荐图表：`training.analysis.visualization.plot_per_case_boxplot()`

### 15.4 K-Fold CV 规范

- 正式投稿前建议补充 5-fold patient-level CV 结果
- 使用 `training.core.splits.generate_kfold_splits()` 生成划分
- 每个 fold 至少跑 1 个 seed；主结果表报告 5-fold 均值
- 单次划分结果可作为主表，k-fold 结果放附录作为稳健性验证
