# 任务 A (V2P-WSSP) 优化与修改建议方案

## 现状痛点诊断

通过分析 `代码修改与实验推进记录.md` 和 `任务A实验状态表.md` 的最新进展，V2P-WSSP（点云壁面血流动力学专线）遇到了明显的瓶颈：
- **WSSP-01**（仅 p + WSS）取得了 V2P 线上暂时的最佳 WSS 结果（`wall.r2_wss ~ 0.44`）。
- **V1 WSS 多任务基线**（`A-Opt-05-wss-multi` / `A-Main-01-wss-multi`，Transformer + 全场监督 + `wss_loss_weight=0.1`）已达 **`wss_r2_wss ~ 0.46`（3 seed 均值）**，高于 WSSP-01 的 ~0.44。后续 WSSP 线的改进目标应以此为真正的最高基线锚点。
- **采样口径已统一确认**（2026-04-26 核实）：**WSSP-01 / 02 / 03 / 04 均使用相同的图资产**，对应 `pipeline/config.py` 全局默认 `wall_max_points=13000`、`target_total_points=15000`（壁面点达上限时即为 **13000 壁面 + 2000 近壁内部**）。早期文档和配置 `meta.notes` 中部分 run 标注为「标准采样」属描述误差，不代表实际数据口径不同。此项已在实验状态表中同步修正。
- 在相同采样口径下，**WSSP-02 / 03 / 04** 的负结果（`r2_p` 约 0.004~0.025，`wss_r2_wss` 约 -0.017~-0.018）应归因于**监督目标、loss 权重、早停策略、WSS 头表达能力与归一化机制**共同作用下的多任务优化失败，而非采样差异。
- **（2026-04-27）** 同批与 **V1 Line W 权重复扫**（`A-Opt-W03-p025-w02` / `w02` / `w05`）的 **`regional_eval` 分区域**对比已写入 [任务A实验状态表](03-共享执行与状态/任务A实验状态表.md)：**V2P-WSSP-04** 的 **`interior`/`wall` 场 `r2_p`** 与 **Transformer+场全监督+WSS** 线**不可无注释同表**；W03 三线 **WSS 权重扫参未突破**略负的 **`wall.r2_wss` 平台**（`w02` 略好）。多模型总览图 **`outputs/field/plots/wssp04_w03_line_seed1_20260427/`**；推进记录见 [代码修改与实验推进记录](../../02-推进与变更/代码修改与实验推进记录.md) 文首 **2026-04-27** 条目。
- **（2026-04-28）** **`V2P-WSSP-05`（MSE WSS）/ `V2P-WSSP-06`（Huber WSS）** 在 **`split_AG_v1`、三 seed** 上已训练并完成主后处理：**`wss_r2_wss`** 仍略负，**Huber 相对 MSE 未形成稳定优势**；**`r2_p` 跨 seed 方差大**。**`V2P-WSSP-06` seed=3** 尚缺 **`predictions_test` 导出**（可单跑 `predict_field`）。结论与数字见 [任务A实验状态表](03-共享执行与状态/任务A实验状态表.md)「实验记录摘要 · V2P-WSSP-05 / V2P-WSSP-06」。

**核心矛盾分析：**
1. **多任务梯度劫持**：压力（标量、全场平滑）和 WSS（矢量、仅限壁面、局部高方差）的优化方向冲突。强行用同一个主干网络和简单的 MSE 加权，会导致高方差的 WSS loss 把压力的梯度覆盖掉（如 WSSP-02 的 `wss_loss≈19.5 × 0.5=9.77` 是 `data_loss≈1.95` 的 5 倍）。
2. **监督目标切换带来的归因混淆**：WSSP-01 与 02/03/04 同时改变了速度监督、压力权重、WSS 权重与早停策略；因此不能只根据单次 seed 结果判断"速度监督有害"，必须用只改一个变量的最小实验重排。
3. **损失函数对异常值可能敏感**：WSS 可能存在局部极端的高剪切力区域（如颈动脉分叉处 WSS 可达 10+ Pa，动脉瘤囊处 <1 Pa）。使用 MSE Loss 会放大这些异常点的误差。**但此假设需先经数据分析验证**——在引入 Huber Loss 之前，应统计训练集壁面 WSS 各维的分布与异常值比例（如 >3σ 占比），确认异常值确实是导致训练不稳定的主因之一，而非多任务梯度冲突本身。
4. **WSS 4 维冗余**：当前 `wss_dim=4`（`wss_mag, wss_x, wss_y, wss_z`），但 `|WSS| = √(wss_x² + wss_y² + wss_z²)` 是确定性关系。同时预测 4 维且等权处理可能导致 magnitude 与分量预测不一致。应讨论是否只预测 3 个分量后事后计算 magnitude，或引入一致性约束。

## 核心优化建议 (拟执行路线)

### 1. 数据基座：锚定 Wall-rich + Near-wall 采样
WSS 的预测绝对依赖近壁网格的高分辨率。后续 WSSP 线不再把采样作为可自由变化项，而是固定采用 **13000 壁面点 + 2000 近壁面点**。
- **动作**：所有新 WSSP 实验必须在配置、manifest、run 名称和实验记录中显式标注 `wall13000_nearwall2000` 或等价字段。旧记录中标注为「标准采样」的 WSSP-02/03/04 已核实实际图资产与 WSSP-01 一致（均为 `pipeline/config.py` 默认 `wall_max_points=13000`），文档口径已统一修正。
- **归因原则**：采样已固定；下一轮只允许逐项改变 loss、监督权重、早停或 WSS head，避免继续把多个变量混在一起解释。

### 2. 模型架构：任务解耦 (Decoupled Heads)
压力和 WSS 的物理特性差异过大，需要给它们独立的空间去表示。
- **动作**：模型架构不作为下一轮第一优先改动；只有在固定采样、鲁棒 WSS loss 与最小权重对照仍失败后，才进入 WSS head / decoder 解耦。
- **方案 A (多分支 Decoder)**：让 PointNeXt 的前段共享上下文，但在上采样的最后两级分成两个完全独立的 Decoder 分支：一个专门负责平滑的压力/速度重建，另一个专门关注局部壁面梯度的 WSS。
- **方案 B (Stop-Gradient)**：在 WSS 头的输入处阻断梯度回传（`detach()`），让 Backbone 完全由速度和压力等"场"物理量驱动，WSS 头仅作为一个附加的非线性观测器。**前置条件**：当前 WSS 头仅为单层线性映射（`nn.Linear(hidden_dim, wss_dim)`），若采用 Stop-Gradient，WSS 头必须同步升级为**多层非线性网络**（至少 2-3 层 MLP + 激活函数），否则线性头无法从冻结特征中提取足够的 WSS 信息。

### 3. 损失函数：引入鲁棒损失
解决梯度失衡。
- **动作 1（前置分析）**：在 WSSP-06 之前，先统计训练集壁面点的 WSS 各维度分布（直方图 / CDF / 超过 3σ 的异常值比例），确认 MSE 是否因异常值而失衡。如果异常值显著（如 >5%），Huber Loss 方案成立；如果不显著，应优先考虑梯度冲突等其他归因。
- **动作 2**：修改 `losses.py`，对 `wss_loss` 使用 **Huber Loss (Smooth L1 Loss)** 替代 MSE Loss。这能降低局部极端 WSS 值对全局梯度的破坏，提升训练稳定性。
- **动作 3（低优先级扩展）**：压力梯度损失或相邻点压差约束。注意：WSSP-01 在仅 p+WSS 监督下已达 `r2_p ~ 0.911`，说明**压力学习本身不是瓶颈**；WSSP-02/03/04 中 `r2_p` 崩到 0.004~0.025 的原因是多任务梯度冲突而非压力损失函数不够好。因此压力梯度损失**仅在**固定采样 + Huber + 权重调整后 `r2_p` 仍无法恢复到 ~0.91 时才考虑。

### 4. 训练策略：两阶段渐进式训练 (Two-Stage)
两阶段训练是可选的第二阶段方案，不应在下一轮一开始就和 Huber / 权重重排同时叠加。
- **Stage 1 (流场基建)**：在 Wall-rich 采样下，关闭 WSS 监督，用压力和速度监督（建议 `[1.0, 1.0, 1.0, 1.0]` 或 `[0.5, 0.5, 0.5, 1.0]`），训练出一个高质量的近壁流动上下文基座。**物理依据**：WSS 的本质定义为 WSS = μ · (∂u/∂n)|\_wall，即壁面处速度法向梯度乘以动力黏度，因此**近壁速度场的准确性直接决定了 WSS 的可预测性**。Stage 1 使用过弱的速度权重（如 `[0.1, 0.1, 0.1, 1.0]`）会让 backbone 偏向编码压力信息而忽略速度梯度，与 Stage 2 从 backbone 特征中提取速度梯度预测 WSS 的目标矛盾。
- **Stage 2 (WSS 精调)**：加载 Stage 1 的权重，冻结大部分 Backbone，开启 WSS 头（使用 Huber Loss），以较小的学习率专门精调 WSS。

### 5. WSS 归一化策略（待讨论）
WSS 在不同血管几何中的量级差异可以很大。当前代码中已实现 WSS 的壁面统计归一化，但归一化方式（全局统计 vs 逐样本统计 vs 逐区域统计）会直接影响模型的泛化能力。后续若异常值分析结果支持，可考虑在归一化层面做调整。

## 实验矩阵重排建议

建议废弃"同时改监督、loss、早停"的混合试法，转而按以下逻辑重构下一步实验（全部固定为 **13000 壁面点 + 2000 近壁面点**，采样口径不再变化）：
1. **WSSP-05 (复核基线)**：复刻 WSSP-01（p+WSS，速度 loss=0），补 seed=1/2/3，确立 V2P-WSSP 在固定采样下的真实基线。**同时执行 WSS 分布分析**：统计训练集壁面 WSS 各维直方图与异常值比例，为 WSSP-06 的 Huber 决策提供数据依据。
2. **WSSP-06 (鲁棒损失)**：在 05 基础上仅将 WSS MSE 替换为 Huber / Smooth L1，其他不动，用于验证异常值敏感性是否是主瓶颈。
3. **WSSP-07 (速度辅助)**：在 06 基础上加入速度辅助。建议至少设置两组权重做对比（如 `[0.1, 0.1, 0.1, 1.0]` vs `[0.5, 0.5, 0.5, 1.0]` 或 `[1.0, 1.0, 1.0, 1.0]`），用于区分「速度无增益」和「速度权重不够高」。单一权重设置在归因上无法做出该区分。
4. **WSSP-08 (两阶段或解耦结构)**：只有 06/07 仍不能稳定超过 WSSP-05 时，再进入两阶段训练、Stop-Gradient 或多分支 Decoder。

**Go / No-Go 口径**：
- **WSS 绝对门槛**：新方案三 seed 的 `wall.r2_wss` 均值须稳定超过 **V1 最佳基线 ~0.46**（`A-Opt-05-wss-multi`），而不仅是 WSSP-01 的 ~0.44。同时 `r2_p` 不得明显低于 ~0.91。
- **多任务成功标准**：若新方案 `wall.r2_wss > 0.46` 且 `r2_p > 0.91`，视为 WSSP 主线成功；若同时 `r2_vel_mag > 0.5`，视为多任务全面成功（WSSP-01 无速度监督、`r2_vel_mag ≈ 0`，Go/No-Go 应覆盖速度维度以避免退回单任务模型）。
- 若速度指标改善但 `wall.r2_wss` 或 `r2_p` 下降，则不算 WSSP 主线成功。
- 若单 seed 有正信号但跨 seed 方差大，先补 seed，不继续堆结构。
