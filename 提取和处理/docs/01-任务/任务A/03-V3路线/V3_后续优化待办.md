## TODO-11：压力弱化或 WSS-only fallback

- [ ] 状态：未开始
- 优先级：P1
- 类型：loss / 训练策略
- 当前依据：
  - 判断：该想法**不能直接按“压力和 WSS 都一般就改 WSS-only”执行**。如果 WSS 本身在单目标下也一般，去掉压力不会自动解决 WSS 不可学问题。
  - 合理触发条件应是：`P+WSS` 多任务下 WSS 明显低于 `Probe-WSS-01` 或 `WSS-Mag-01`，且压力目标对下游不是硬需求，说明压力与 WSS 存在任务竞争或早停目标偏移。
  - 当前压力单目标可学性很强，压力本身不是弱项；是否保留压力应由“是否干扰 WSS 主目标”和“论文/任务是否需要壁面压力”共同决定。
- 计划做法：
  - 新增 `V3P-WSSOnly-01`：关闭压力和速度监督，仅保留壁面 WSS 或 WSS magnitude。
  - 新增 `V3P-WSS-Pweak-01`：保留压力但降低 `lambda_p_int/lambda_p_wall`，作为 WSS-only 与 P+WSS 之间的折中。
  - 与 `V3P-Probe-WSS-01`、`V3P-Probe-PWSS-01`、`V3P-Main-01-PW` 做对照，判断压力是否真实干扰 WSS。
- 预期收益：
  - 在 WSS 是唯一主目标时，减少多任务竞争。
  - 帮助区分“WSS 本身不可学”和“压力/WSS 多任务冲突”。
- 风险与注意事项：
  - WSS-only 若提升 WSS 但完全丢失压力能力，只能作为 WSS 专线，不应继续命名为双域 P+WSS 主线。
  - 若 WSS-only 与 P+WSS 差异很小，则不应继续沿该方向扩展，应回到数据规模和 WSS 表达。
