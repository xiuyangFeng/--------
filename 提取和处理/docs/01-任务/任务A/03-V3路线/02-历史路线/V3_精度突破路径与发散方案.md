## 2. 瓶颈根因（严谨判读）

### 2.1 主因：全局坐标系下 wss_x/y 缺乏跨 case 共性

- **wss_z（轴向）**：与主流速度梯度、几何大尺度相关 → train/val 有可学信号。
- **wss_x/y（横向）**：二次流（Dean 涡、分叉诱导）主导 → 对局部几何与入口微扰极度敏感；60–179 case 不足以在**全局 xyz** 下学到泛化规律。
- **证据**：Main-PW history 中 train `wss_x/y` R² 升至 0.24–0.30，val 降至 **−0.17 / −0.07**（记忆而非泛化）；AsymW 缓解 val 过拟合但 test x/y 仍 ≈0。

### 2.2 次因：AsymW 的增益本质是「只学 z」，非表达力突破

- 权重 `[1, 0.05, 0.05, 0.90]` 实质放弃 x/y → 标量 WSS R² 由 z 主导抬升。
- **结论**：在**不改 WSS 表示**的前提下，继续微调权重/正则，**信噪比已接近极限**（~0.36–0.40 带宽）。

### 2.3 V3D 特有：跨域分布异构 + 单一全局 z-score

- AAA / AG / ILO 壁面 `p` / `wss` 物理分布差异大；单一 `wss.std≈3.14` 对三域不平等。
- 归一化修复主要救 **P**，**未**抬升 WSS；数据量增大反而引入域偏移。

### 2.4 结构与信号缺口

- 当前 backbone：**kNN mean/max pool**，无显式**壁面法向梯度**信息；WSS 物理定义即 μ·(∂v/∂n)|_wall。
- 节点特征缺：`dist_to_wall`（连续值）、壁面法向 **n̂**、分叉距离等；`Tangent_*` 已有但未用于 **WSS 目标** 对齐。

### 2.5 评估与选优噪声

- val 仅 5–8（V3P）/ 25（V3D）例 → `best_wss_model` 选取方差大（4957 ep24 vs 4958 ep57）。
- 主报仅点级 R²；缺病例级 Pearson / 高 WSS top-k Dice（路线 §7.1 / §8.5 已规划未落地）。

## 4. 路径 A：WSS 表示改造（最高优先级）

> **判断**：冲破 0.40 的**必要条件**是 (i) 局部坐标系 或 (ii) magnitude-only + 一致性；继续在全局 4 维上扫权重收益递减。

### A1 · TODO-5：WSS 局部坐标系（**推荐下一项 P0 实验**）

**问题**：血管走向不同 → 「轴向 WSS」落在 x/y/z 任意分量 → 网络需隐式学坐标系。

**方案**（预处理 + 训练开关，**不破坏**现有 `y_wss` 全局字段）：

1. 新增 pipeline 步骤（或 `extract_features` 扩展）：
   - 输入：壁面点坐标、已有 `Tangent_X/Y/Z`、壁面法向 **n̂**（CFD 或几何估计）。
   - 输出：`wss_axial = WSS·t̂`，`wss_circ` / `wss_rad`（Frenet-Serret 或 n̂×t̂ 投影）。
2. 图资产：新增 `y_wss_local`（或 CSV 列 + convert 写入）；配置 `data.wss_target_frame: "global" | "local"`。
3. `wss_head` 维度：local 模式下 3 维 + 可选 magnitude 第 4 维（由 sqrt 推导，见 A2）。
4. 归一化：local 分量 **单独** 在 train 壁面节点上统计 mean/std，写入 `normalization_params_global.json` 扩展段。

**实验设计**：

| Exp ID | 目的 | 对照 |
| --- | --- | --- |
| `V3P-Probe-WSS-Local-01` | 单目标上限（仅 WSS，local frame） | Probe-WSS-01 ~0.364 |
| `V3P-Main-01-PW-Local` | PW 主线 + local | Main-PW / AsymW-a |

**Go 判据（seed=1）**：`wss_axial` test R² **> 0.50** 或标量 `wss_r2_wss` **> 0.45** → 补 seed2/3，升 PW 母版。

**风险**：法向估计噪声、分叉处 t̂ 不连续 → 需在 Diag 输出 local 分量 train/test 分位 diff。

### A2 · TODO-9 / TODO-10：magnitude 与 4 维冗余消解

- **现状**：4 维 `[mag, x, y, z]` 冗余；AsymW 已「半 magnitude-only」。
- **方案 A2a（TODO-10）**：`wss_head` 仅 1 维 magnitude + loss 仅监督 `wss` 标量（x/y/z 由 post-hoc 不报告或置 0）。
- **方案 A2b（TODO-9，V3P-WSS-04）**：监督 3 局部分量 + `|pred_mag − sqrt(Σ pred_i²)|` 一致性项。
- **触发**：A1 验证后或与 A1 **组合**单变量对照；禁止与 global AsymW **混表**。

### A3 · TODO-11：WSS-only / 压力弱化（条件对照）

- **目的**：验证 P+WSS 是否仍有多任务轻微冲突（当前 P 0.94+，冲突可能已弱）。
- **触发**：A1 后若 PW-Local 仍低于 Probe-WSS-Local → 试 `lambda_p_*` 降档或 WSS-only 短训。
- **非主线**：论文仍需 P+WSS 双 readout。

### B2 · TODO-17：近壁速度差分 / 梯度上下文 ✅ **强 No-Go（5170）**

- **结果（2026-05-27）**：`V3P-Main-01-PW-AsymW-a-VelGradB1` test **0.369**（best_wss）vs AsymW **0.399**（**−0.030**）；val 0.473→test 0.369 落差大 → **显著负增益**，**维持关闭** `wss_vel_context`。

### B3 · TODO-18：多半径 / 多 k 邻域聚合（轻量 Arch）✅ **No-Go（5202）**
