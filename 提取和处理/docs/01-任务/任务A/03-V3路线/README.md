## 2. 路径传递关系

### 2.1 从 A 到 G 的总传递

```text
A 表示改造（local / magnitude） ┐
B 近壁几何/梯度/多 k             ├─ 已基本 No-Go → F 发散探索 / oracle
C 三域数据 / split / norm        ┘
                  │
                  ├─ F loss / 结构小探针多数 No-Go，PGradFeat 只有病例叙事信号
                  │
                  └─ G 下一代路线：
                       G0/QA/oracle
                         ├─ G1 等变表示（坐标系负担）
                         ├─ G2 物理残差 / Cf / 压力耦合
                         ├─ G3 数据 / 预训练
                         ├─ G4 壁面算子换轨
                         └─ G5 区域/病例 surrogate 兜底
```

### 2.2 路径传递规则

| 来源路径 | 当前结论 | 传递到哪里 | 传递条件 |
| --- | --- | --- | --- |
| **A：WSS 表示** | local / magnitude / 分量约束已 No-Go 或未升母版 | **G1 / H11** | 只保留“坐标系负担可能仍存在”的问题，不再回头扫 global loss 权重 |
| **B：近壁几何与梯度** | `dist_to_wall`、VelGrad、MultiK 全线 No-Go | **G2 / H3** | 从“无序近壁特征”转成“法向 ray / 物理算子 / 残差” |
| **C：三域数据** | DomainNorm 和 val15 No-Go；数据扩池仍是 V3D 主方向 | **G2 / G3 / H1** | 数据问题不再靠单一 z-score 修；后续走 Cf/Re/Wo、主动选数、多保真 |
| **D：评估与选优** | 病例级、区域级仍是第二门禁 | **所有路径** | 任一路线点级升但病例/区域退化，不升母版 |
| **F：发散探索** | loss 探针多数 No-Go；oracle v2 留下压力/近壁差分/POD 信号 | **G0 / G1 / G2 / G4** | 只把已通过 oracle 的信号传给 G，不把失败 loss 方案原样重跑 |
| **G：下一代路线** | 当前主规划 | **执行计划 / 待办 / 实验日志** | G0 过线后才开训练；G1/G2 必须按阶梯单变量归因 |
| **H 观察池** | 暂不占 TODO | **G0 或 G5** | 只有低成本 oracle 成立后，才升级到正式 TODO |

### 2.3 写文档时的传递规则

- **新实验提交/完成**：先写 [V3_实验执行跟踪日志.md](01-执行与待办/V3_实验执行跟踪日志.md)，再同步 [V3_后续优化待办.md](01-执行与待办/V3_后续优化待办.md) 状态。
- **路线判断变化**：先改 [V3_下一代架构与精度突破方案_2026-06-05.md](00-当前主线/路径G_下一代架构与精度突破方案_2026-06-05.md) 或对应历史路线文档，再改本 README 的当前状态/路径传递。
- **数据口径变化**：必须同步 [V3_三域数据.md](03-数据与质量/V3_三域数据.md) 和 `training/splits/README_V3_splits.md`；禁止只在实验日志里写。
- **仅规划层调整**：必须在推进记录标注“规划层 / 未改代码 / 未跑实验”，避免被误读成实验结果。
