# 任务A周度检查表

> 用途：每周跟踪任务 A 的实验推进、结果结论、阻塞项和下周计划。  
> 上位文档：[任务A实验清单](任务A实验清单.md) / [实验设计总纲](../../实验设计总纲.md) / [实验记录填写规范](../../00-规范与记录/实验记录填写规范.md)

## 1. 使用说明

这份检查表不替代正式实验记录表，而是作为你每周推进任务 A 的执行面板。

建议使用方式：

1. 每周开始前，先填写“本周目标”。
2. 每做完一个实验，就在对应 `Exp ID` 后打勾，并补一句最核心结论。
3. 每周结束时，必须填写：
   - 本周新完成实验
   - 当前最优主线
   - 与预期相反的结果
   - 当前最大阻塞
   - 下周只做哪 1 到 2 件事
4. 正式数值结果仍然写入 [实验记录表.xlsx](../../00-规范与记录/实验记录表.xlsx)。

每周使用时建议固定两个原则：

- 周内只追 1 到 2 个核心问题，不要同时开太多实验支线。
- 每周结束必须留下可以复用的产物，而不只是“跑过了”。

建议每周至少归档以下 6 项：

- 本周固定的配置版本
- 本周完成的 `Exp ID`
- 当前最好 checkpoint 路径
- 一张结果汇总表
- 一句最重要结论
- 下一周唯一优先级最高的动作

## 6. 当前建议的优先级提醒

在任务 A 当前阶段，优先级固定为：

1. 先站稳单尺度主线
2. 再引入 hierarchy
3. 先做能回答问题的实验
4. 再考虑更先进 backbone

如果某周出现以下情况，下一周必须优先复盘而不是继续加实验：

- 多个 seed 方向不一致
- `A-Main-01` 不能稳定优于 `A-Base-03`
- physics loss 明显拖垮训练稳定性
- hierarchy 提速但精度明显崩坏
