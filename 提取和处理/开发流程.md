# 开发流程

### 项目总记

用“显式几何先验 +二次残差网络（QN）+ **物理约束的图神经网络主干** + 多模态临床融合”的多任务框架，预测时空血流动力学场（速度、压力、WSS、OSI 等），并进一步完成基于数字孪生体的临床终点风险预测（如EVAR后的髂肢闭塞、AAA破裂），实现预测时空场的秒级响应和数字孪生在医疗中的使用。

### 目标与子任务

- 物理场重建：基于AAA的点云数据集，预测时空速度与压力场，派生WSS、OSI、RRT 等指标。
- 临床终点预测：将图池化的全局物理表征与临床表格特征融合，输出风险概率或连续结局
- 结构优化：物理约束、显示几何特征、GNN 消息传递、 QN二次残差网络模块

### 模型架构（PI-GNN 多任务）

- 主干：**Graph Transformer** + PINN物理信息神经网络+QN二次残差网络
- 头A（物理场）：两个独立 MLP 分别回归速度场（N×3）与压力场（N×1）。由此可派生 WSS/OSI/RRT。
- 头B（临床）：池化得到 h_global，与临床向量拼接，经 MLP 输出概率（Sigmoid）。支持风险分层与可解释性分析（SHAP、校准曲线、DCA）。

---

### 流程：

当获得一个患者的三维几何模型后，进行显示几何特征工程，对几何血管进行处理做中心线提取后，获取一些几何特征，（可选：PointNet++做局部几何编码，获得局部几何特征）和时空坐标加全局边界条件进行特征融合，赋予对应的节点。然后进行训练QN二次残差网络和物理图神经网络，将该网络预测的压力场和速度场结合物理约束方程做损失函数，最终优化出效果最好的主干神经网络。然后数字孪生预测ILO破裂的时候，进行图池化获得全局特征，加上经过神经网络预测出来的速度场和压力场进行下游计算，获得WSS，TAWSS，OSI，RRT，结合临床数据做3个部分的特征融合后进行训练新模型，最终输出一个ILO的概率。

 

## **主干神经网络设计**

### 1.数据板块

**1.1 显式几何特征工程（Explicit Geometric Feature Engineering）**

目的：为复杂的动脉瘤几何建立一个标准化的“骨架”，提取具有明确物理意义的先验几何，解决不同患者几何形态各异、难以比较的问题，为模型提供强大的几何先验。

具体操作：对输入的3D模型（网格/点云），提取血管中心线，并为每个点计算出曲率、沿程位置等特征。

**数据内容和原因**

- 径向距离（Radial Distance）：该点到中心线的垂直距离。这个特征直接描述了该点所在位置的局部血管半径，是极其重要的几何信息。
- 测地线坐标（Geodesic Coordinate）：该点在中心线上投影点的位置（例如，从入口点开始沿着中心线走过的归一化长度）。这个特征告诉模型，这个点是位于动脉瘤的“上游”、“中段”还是“下游”。
- 中心线曲率（Centerline Curvature）：该点在中心线上投影点处的曲率大小。这个特征描述了该点是否位于血管的“拐弯处”。血流动力学在弯曲段和笔直段是完全不同的。
- 中心线扭率（Centerline Torsion）：描述血管三维空间扭曲的程度。

**使用技术栈：python+vmtk库，实现自动化处理中心线，并为每一个点云补充上对应的显示几何特征。**

**1.2 （可选）隐式几何特征工程**

目的：和组内工作对接，使用PointNet++来分层级提取局部几何特征，完善几何的输入。

具体操作：对输入的网格，先用一个轻量级的PointNet++结构来学习每个节点的局部几何嵌入。这个嵌入向量包含了比单纯(x,y,z)坐标丰富得多的局部几何信息。然后，将这个学习到的几何嵌入向量，而不是原始坐标，作为GNN的输入节点特征之一。GNN在此基础上，再进行“消息传递”，学习物理场在整个区域的传播和分布规律。

（可参考组内代码和数据完成这个部分）

### 2.模型架构

**图数据（Graph Data）- 描述几何与物理**

**节点（Nodes）**：重建的腹主动脉瘤（AAA）的点云数据集，一个点云代表一个节点。

**边（Edges）**：用 “k 近邻（k-NN）” 或 “半径邻域” 定义边（推荐 k=10-20，适配血管流场的局部关联性）**注意力机制（Attention）来做动态边的权重学习，**自然模拟血管内流场的局部传递关系。

**节点特征（Node Features）**：这是一个 N x F_node 的矩阵，其中 F_node 是每个节点的特征维度。对于每一个节点，其特征向量应包含：

**时空坐标**：[x, y, z, t]。t是离散的单个时间节点。

**边界条件信息**：[inlet_flow(t), outlet_pressure_L(t), outlet_pressure_R(t), ...]这些是全局信息，把它们复制给每一个节点。

**几何增强特征**：[radial_dist, geodesic_coord, curvature, ...]这些是基于中心线的特征。

**（可选）局部几何嵌入**：利用PointNet++的分层结构，在不同的半径尺度下，为每一个点提取一个局部的几何特征向量。这个向量的维度可以设定128维。它包含了比原始(x,y,z)坐标丰富得多的信息，比如该点周围的曲面是平坦、是弯曲、还是尖锐的。

**节点标签**（Node Labels / Targets）- 仅用于计算数据损失：这是一个 N x F_label 的矩阵，包含了CFD仿真出的标准物理场数据[velocity_u, velocity_v, velocity_w, pressure]。

**点标签：**每个节点增加一个“类型”特征**壁面点**还是**内部点**is_wall=1/0。

**物理约束（PINN）**：PINN的物理损失函数依然作用于GNN的最终输出上，保证物理规律的满足。

**图神经网络主干（GNN Backbone）**

作用：负责从输入的图数据中学习几何和物理场的时空表示。

输入：上述的图数据（节点、边、节点特征）。

**架构**：可以采用多层的Graph Transformer。在每一层，GNN通过消息传递机制，聚合邻居节点的信息来**更新自身的节点表示，不仅使用标准的MLP，还引入二次残差模块（QN）。**经过 L 层后，每个节点都会得到一个最终的、高度浓缩的特征嵌入向量 hv。这个向量 hv 已经隐式地包含了该点周围的几何信息和复杂的流场信息。

输出：一个 N x F_embed 的节点嵌入矩阵，其中 F_embed 是最终嵌入向量的维度，记该节**点嵌入矩阵为H**。

- 节点嵌入矩阵 H 被**同时**发送给两个独立的预测子头。
    - **子头A（速度预测头）**：
        - **架构**：一个独立的MLP，例如 MLP_velocity。
        - **操作**：它接收**整个** H 矩阵，并对**每一行 h_v** 进行处理。
        - predicted_velocity_for_node_v = MLP_velocity(hv)
        - **输出**：由于速度是三维的，这个MLP的输出层应该有**3个神经元**。因此，它的最终输出是一个 N x 3 的矩阵，代表了对**每一个节点**的速度向量 [pred_u, pred_v, pred_w] 的预测。**这就是一个完整的、预测出的速度场。**
    - **子头B（压力预测头）**：
        - **架构**：**另一个独立的** MLP，例如 MLP_pressure。
        - **操作**：它也接收**同一个 H 矩阵**，并对**每一行 hv**进行处理。
        - predicted_pressure_for_node_v = MLP_pressure(hv)
        - **输出**：由于压力是标量，这个MLP的输出层应该有**1个神经元**。因此，它的最终输出是一个 N x 1 的向量，代表了对**每一个节点**的压力值 [pred_p] 的预测。**这就是一个完整的、预测出的压力场。**

**结构总结**

1. **输入唯一且不含标签**：GNN主干只有一个统一的输入，只包含描述问题的特征。
2. **主干负责通用理解**：GNN主干的输出是“高浓缩的通用特征向量”（节点嵌入）。
3. **子头负责专业预测**：不同的预测子头（速度、压力）都是**独立的MLP**，它们都接收**同一个**通用特征向量作为输入，然后各自输出自己负责预测的物理量。
4. **最终输出是完整的物理场**：两个子头的输出合在一起，就是模型预测出的**一个完整的速度场和一个完整的压力场**。它们**不是**包含其他特征的矩阵或向量，而是**最终的、具有物理意义的预测结果**。

**损失函数设计**

**1. 总损失函数结构**

总损失 = 数据损失（所有点的速度 / 压力预测误差） + ω × 物理损失（按节点类型分配的约束残差）其中：

- **ω**：物理损失的总权重（推荐 1~5，可通过网格搜索优化，避免物理约束过强掩盖数据规律）；
- 核心改进：物理损失不再是 “所有点的统一残差求和”，而是 “按节点类型拆分残差，并赋予不同权重”。

**2. 各部分损失的具体计算（分节点类型）**

**（1）数据损失：所有点统一计算，保证预测值与 CFD 标签一致**

数据损失 = (1/N_total) × [ Σ（内部点：|u_pred - u_cfd|² + |p_pred - p_cfd|²） + Σ（壁面点：|u_pred - 0|² + |p_pred - p_cfd|²） ]

- **说明**：
    - 内部点：u 和 p 均以 CFD 标签为基准（u_cfd≠0，p_cfd 为真实压力）；
    - 壁面点：u 以 “无滑移条件” 为基准（目标值 0），p 仍以 CFD 标签为基准（壁面压力有真实物理意义，需拟合）；
    - N_total：所有点的总数，保证数据损失是全局平均误差。

**（2）物理损失：按节点类型拆分，避免过定**

物理损失 = (1/N_inner) × Σ（内部点：残差_NS） + (α/N_wall) × Σ（壁面点：残差_no_slip）

- **各残差定义**：① 残差_NS（内部点的 Navier-Stokes 残差）：残差_NS = |∇・u_pred|²（质量守恒残差） + |ρ(∂u_pred/∂t + u_pred・∇u_pred) - ∇・σ_pred|²（动量守恒残差）（σ_pred 为预测的 Cauchy 应力张量，σ = -pI + 2μD，D 为速度梯度张量）② 残差_no_slip（壁面点的无滑移残差）：残差_no_slip = |u_pred - 0|²（仅约束速度为 0，不涉及压力）
- **权重与数量说明**：① N_inner：内部点总数，N_wall：壁面点总数（按类型统计，避免某类点数量过多导致残差主导）；② α：壁面约束的权重系数（推荐 0.5~1）—— 因壁面点的无滑移约束已在 “数据损失” 中体现，物理损失中无需过高权重，避免重复约束；
- **关键优化**：仅内部点计算 Navier-Stokes 残差，壁面点仅计算无滑移残差，从根本上解决过定问题。
1. **标签生成**：从组内 CFD 数据或血管几何模型中提取 “壁面标签”——
    - 若有血管 STL 模型：计算每个点云点到 STL 表面的距离，距离 < 0.1mm（可根据血管尺寸调整）的点标记为`is_wall=1`，其余为`is_wall=0`；
    - 若仅有 CFD 数据：CFD 计算时会区分 “壁面边界” 与 “内部流场”，可直接导出每个点的 “边界类型” 标签，映射为`is_wall`。
2. **模型适配**：在 GNN 的输入层加入 “节点类型特征”（`is_wall`或`node_type`），并在 PINN 的损失计算层添加 “类型判断逻辑”——
    - 用 PyTorch 的条件判断（如`torch.where(is_wall == 1, 残差_no_slip, 残差_NS)`）实现差异化残差计算；
    - 避免硬编码：将标签作为模型输入的一部分，而非在代码中固定，方便后续扩展 “入口 / 出口点” 等其他类型的约束（如入口点的流量约束）。
3. **验证效果**：训练后通过 “残差分布可视化” 验证约束有效性 ——
    - 内部点：Navier-Stokes 残差应均匀分布在低数值区间（如 < 0.1），无明显高残差聚类（说明流场符合物理规律）；
    - 壁面点：无滑移残差应接近 0（如 < 0.01），且壁面附近内部点的速度梯度应连续（如从壁面 u=0 过渡到内部 u = 正常流速，无突变）。

### **步骤一：数据准备与预处理（适配“流体粒子图”）**

这是为您的独特框架量身定制的数据处理流程。

1. **输入原始数据**：
    - **病例集**：CFD仿真数据集，包含多个病例。
    - **数据增强**：对真实病例的**表面几何**应用SSM等方法，生成新的虚拟几何。然后，在这些新几何的**内部**进行均匀采样生成内部点，再对整个模型（壁面+内部）进行CFD仿真，获得新的训练数据。
2. **构建“预处理后”的点云文件（对于每一个病例的每一个时间点）**：
    - **目标**：创建一个统一的、信息丰富的点云文件（.npy或.h5格式），作为构建图的“原材料”。
    - **操作**：
        
        a. **加载数据**：加载CFD仿真输出的、包含壁面和内部点的**全量点云**（坐标、is_wall标签、标准化的速度/压力标签）。
        
        b. **提取中心线**：基于该点云中的**壁面点**，提取血管中心线。
        
        c. **计算显式几何特征**：为**全量点云中的每一个点**（无论是壁面还是内部），都计算其到中心线的距离、投影位置、对应位置的曲率等。
        
        d. **计算隐式几何特征**：将**全量点云的坐标**输入到一个预训练的PointNet++编码器中，为**每一个点**都生成局部几何嵌入。
        
        e. **整合与保存**：将所有信息整合在一起，保存为一个大的Numpy数组，维度为 N_total x F。其中F包含了：
        
        - x, y, z, t (4 dim)
        - is_wall (1 dim)
        - standardized_u, v, w, p (4 dim, **这是标签Y**)
        - explicit_geom_features (e.g., 4 dim)
        - implicit_geom_features (e.g., 64 dim)
        - *（全局边界条件可以在加载时动态添加）*
3. **数据集划分**：
    - **严格在“病例”层面**进行划分，将所有病例ID分为训练、验证、测试三组。

### **步骤二：模型训练（聚焦动态图的构建与PINN）**

这是整个流程的核心，包含了动态图构建和PINN损失的实现。

1. **数据加载与动态图构建（在DataLoader中完成）**：
    - 您需要编写一个自定义的PyTorch Geometric Dataset 类。在这个类的 __getitem__ 方法中，或者使用 torch_geometric.transforms，来实现在线（on-the-fly）的图构建。
    - **对于每一个要加载的样本（一个时空快照的点云文件）**：
        
        a. **加载预处理后的点云文件**。
        
        b. **分离特征X和标签Y**：
        
        - X = [x,y,z,t, is_wall, explicit_geom, implicit_geom, global_BCs]
        - Y = [standardized_u, v, w, p]
        
        c. **动态构建边 edge_index**：
        
        - 提取所有点的 (x,y,z) 坐标。
        - 使用高效的近邻搜索算法（如PyG内置的knn_graph或radius_graph），基于这些坐标，为每个点找到k个最近的邻居，并生成edge_index。
        
        d. **创建PyG Data对象**：将 X, Y, edge_index 等打包成一个 Data 对象并返回。
        
    - **DataLoader** 会自动将这些单独的 Data 对象，打包成一个大的、包含多个图的Batch对象，送入模型。
2. **模型与优化器初始化**：
    - model = Your_QR_GNN_Model()。这个模型的内部结构就是我们之前设计的，以QuadraticResidualGraphAttentionLayer为核心的堆叠式网络。
    - optimizer = torch.optim.Adam(model.parameters(), lr=0.001)。
3. **主训练循环与损失函数设计**：
    - for epoch in range(num_epochs):
        - for batch in train_loader: (DataLoader在后台已经为我们准备好了动态构建的图 batch)
            
            **a. 模型前向传播**
            
            - predicted_fields = model(batch.x, batch.edge_index)
            - predicted_vel = predicted_fields[:, 0:3]
            - predicted_p = predicted_fields[:, 3]
            
            **b. 计算数据损失 L_data**
            
            - loss_data = mse_loss(predicted_fields, batch.y)
            - **（可选）带权重的损失**：您可以给壁面点和内部点的数据损失赋予不同的权重，因为壁面上的物理量通常更受关注。
            
            **c. 计算物理损失 L_physics（服从物理约束方程）**
            
            - 这一步是PINN的核心，并且需要根据**节点类型**进行区分。
            - **分离节点**：根据batch.x中的is_wall特征，将预测结果和坐标分离为“壁面部分”和“内部部分”。
            - wall_nodes_indices = (batch.x[:, is_wall_index] == 1)
            - internal_nodes_indices = (batch.x[:, is_wall_index] == 0)
            - **计算壁面物理损失 loss_physics_wall**：
                - predicted_vel_wall = predicted_vel[wall_nodes_indices]
                - loss_noslip = mse_loss(predicted_vel_wall, torch.zeros_like(predicted_vel_wall))
                - （这部分就是对壁面点施加“无滑移”边界条件的物理约束）
            - **计算内部物理损失 loss_physics_internal**：
                - 提取内部点的坐标和预测场：
                    - internal_coords = batch.x[internal_nodes_indices][:, 0:4] (需要requires_grad=True)
                    - predicted_vel_internal = predicted_vel[internal_nodes_indices]
                    - predicted_p_internal = predicted_p[internal_nodes_indices]
                - **进行自动微分**：完全按照我们之前的讨论，将这些内部点的预测值代入Navier-Stokes方程，计算**连续性方程残差**和**动量方程残差**。
                - loss_continuity = mse_loss(continuity_residual, 0)
                - loss_momentum = mse_loss(momentum_residual, 0)
                - loss_physics_internal = loss_continuity + loss_momentum
            - **合并物理损失**：total_loss_physics = loss_physics_wall + loss_physics_internal
            
            **d. 计算总损失与优化**
            
            - total_loss = loss_data + lambda * total_loss_physics
            - 执行反向传播和优化步骤。

### **步骤三：模型评估**

- **流程**：与我们之前讨论的完全一致。
- **加载最佳模型**，在**独立的测试集**上进行评估。
- **输出指标**：
    - **基础物理场精度**：计算预测的速度场和压力场与CFD真值之间的**MRE, RMSE, R²**。
    - **衍生指标精度**：
        1. 从模型的速度场预测中，**后处理计算**出pred_WSS, pred_OSI等壁面衍生指标。
        2. 将这些“AI计算的衍生指标”，与您**预先提取好的、来自CFD的真实衍生指标标签**进行对比。
        3. 计算这些衍生指标的MRE, RMSE, R²。
- **评估指标**：
    - **可视化对比**：制作4D时空演化图、误差分布图、散点图等。

## **孪生体设计**

**1. 目标与问题定义**

- **临床问题**：对于接受EVAR治疗的AAA患者，能否在术后早期，就根据其解剖形态和血流动力学特征，预测其远期发生髂动脉支架肢体闭塞（ILO）的风险
- **机器学习任务**：这是一个**二分类问题**。
    - **输入（X）**：一个描述单个患者“解剖+血流动力学+临床”状态的综合特征向量。
    - **输出（Y）**：一个**连续的概率值（Soft Label）**，范围在 [0, 1] 之间。例如，输出 0.75，代表模型预测该患者有**75%的概率**会发生髂肢闭塞。

**2. 数据准备与特征工程（核心步骤）**

- **数据集**：来自论文的75名患者（25例闭塞和50例通畅）。
- **特征构建**：对于这75名患者中的**每一位**，需要构建一个特征向量。这个向量至少包含以下三组特征：
    - **组A：AI驱动的深度血流动力学特征（来自模块一中的神经网络）**
        1. **全局隐式特征 (h_global)**：将患者的术后早期三维几何模型的点云输入到您冻结的PI-GNN引擎中，对GNN主干输出的所有节点嵌入进行图池化（如平均池化，最大池化），得到一个高维向量（例如，512维）。这个向量是AI对整个流场模式的高维特征学习。
    - **组B：基于AI预测的压力场和速度场获得衍生指标**
        1. TAWSS
        2. OIS
        3. RRT
    - **组C：基础临床与解剖特征**
        1. 患者的人口统计学信息：Age（年龄），height（身高），weight（体重）。
        2. 常规解剖学指标：Proximal_neck_angulation（近端颈部成角）, Aneurysm_max_diameter（动脉瘤最大直径）。
        3. 手术相关信息：Stent_graft_diameter（支架移植物直径）, Stent_oversizing_ratio（支架过度尺寸比）。
        4. 合并症：Hypertension（高血压）, Smoking_status（吸烟状况）等。

**3. 模型选择与训练**

- **模型选择**：由于现在处理的是一个**表格数据**的分类问题（输入是一个融合后的特征向量），可以选择多种强大的机器学习模型：
    - **首选：梯度提升树模型（如 XGBoost, LightGBM）**。这类模型在处理异构的表格数据时通常表现最佳，并且自带特征重要性排序功能，可解释性强
- **训练流程**：
    - **数据划分**：将75名患者按比例（如80/20）划分为训练集和测试集。为了结果的稳健性，最好采用**交叉验证**（如5折或10折交叉验证）。
    - **特征缩放**：对所有数值型特征进行标准化（StandardScaler）。
    - **模型训练**：在训练集上训练您选择的模型。
    - **超参数调优**：使用网格搜索（Grid Search）或贝叶斯优化来寻找模型的最佳超参数（如树的深度、学习率等）。

**4. 评估与解释**

- **性能评估**：在测试集或通过交叉验证，评估模型的性能，指标包括：
    - **AUC (Area Under the ROC Curve)**：最重要的、综合评估模型区分能力的指标。
    - 准确率 (Accuracy), 精确率 (Precision), 召回率 (Recall), F1分数。
- **特征重要性分析（回答“为什么”）**：
    - 使用**SHAP (SHapley Additive exPlanations)分析。这是目前最先进的可解释性工具，它能告诉您，对于每一个独立的预测**，每个特征（无论是AI提取的，还是专家知识）是如何将预测结果从基线值“推”向最终值的。
    
    ### **模块二的完整流程**
    
    **Step 1: 影像采集与几何重建（数据输入端）**
    
    1. **输入**：患者X的腹主动脉CTA影像（DICOM文件）。
    2. **操作**：
    a. 使用Mimics等软件，进行三维重建，得到**血管壁的表面模型**（例如，patient_X_wall.stl）。
    b. **（关键一步）** 基于这个表面模型，在其**内部空间**生成大量的、均匀分布的**采样点**。这些点就是我们需要的“内部点”。
    c. 将**表面模型的顶点**（作为“壁面点”，is_wall=1）和**内部生成的采样点**（作为“内部点”，is_wall=0）合并，形成一个**完整的、只包含几何信息的“预测用点云”**。
    
    **Step 2: 特征工程（自动化脚本）**
    
    1. **输入**：上一步生成的“预测用点云”和表面模型。
    2. **操作**：运行一个自动化的特征工程脚本，为点云中的**每一个点**计算出GNN引擎所需要的完整输入特征：
    a. 提取**血管中心线**，计算每个点的**显式几何特征**（到中心线距离、曲率等）。
    b. （可选）使用PointNet++为每个点计算**隐式局部几何特征**。
    c. **（关键一步）** 对于临床风险预测，我们需要一个**心动周期**的血流动力学信息。因此，您需要将这个点云**“复制”20份**，并为每一份打上不同的时间标签t（t=0.00, 0.04, ..., 0.80）。同时，将对应时刻的**标准化边界条件**（例如，一个通用的入口流量波形）赋予它们。
    3. **输出**：20个可以输入到您的PI-GNN引擎的、**待预测的图数据对象**。
    
    **Step 3: 运行数字孪生系统（核心预测端）**
    
    1. **输入**：上一步生成的20个图数据对象，以及患者X的**临床表格数据**。
    2. **操作（全自动化）**：
    a. **运行“血流动力学引擎”**：将20个图数据对象输入到您冻结的**PI-GNN模型**中，模型会快速预测出这20个时间点的完整4D流场。
    b. **提取“临床预测特征”**：
    * 运行**后处理脚本**，从预测出的4D流场中，自动计算出我们之前定义的所有**“显式物理指标”**（平均WSS、OSI面积等）。
    * 同时，从GNN主干中**池化**出**“全局隐式特征” h_global**。
    * 将这两组血流动力学特征与患者X的**临床表格数据**进行**拼接**。
    c. **运行“临床风险预测模型”**：将拼接好的最终特征向量，输入到您训练好的**XGBoost/MLP模型**中。
    
    **Step 4: 输出与可视化（结果呈现端）**
    
    1. **输出**：模型会给出一个**单一的、量化的风险概率**，例如：“患者X的破裂风险预测为：**85%**”。
    
     2.（**可选，但极具价值）** 同时，系统还可以**可视化**由PI-GNN引擎计算出的4D流场（如WSS分布云图、涡流结构等），并高亮显示出那些由XGBoost模型判断为“最重要”的风险区域。这能为医生的决策提供直观的、可解释的图像证据
