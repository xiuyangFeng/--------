## 5+ 发散扩充候选方向（E-G–E-R · 2026-06-25）

> 同样遵守 §5 纪律：**单命题 · 0 重训 oracle / 失败归因先行 · M-E 四层门禁 · 新 exp_id 分表 · 本节不含执行/作业号**。
> 设计原则（从 §2 机制反推）：① **不再把 0.43 当作单一标量回归问题硬碰**；② 充分利用「压力可学（R²_p 0.93–0.96）、WSS 排序信号尚可（Pa p95 Spearman 0.559）、target 侧低秩（POD 50 模态 0.90）」这三块**已被证据支持的结构**；③ 把「**模式 / 位置**」与「**幅值 / 标定**」拆开，因为跨病例方差（GUO 0.57 / ZHANG 0.35 / CHEN 0.36）更像 **scale 不一致** 而非纯噪声。

### E-G · 量纲解耦：WSS「模式 + 病例标定」分离预测

| 项 | 内容 |
| --- | --- |
| **命题** | 把 `WSS(x)` 拆为 **无量纲空间模式** `ŵ(x)`（‖·‖ 归一）× **病例级标量尺度** `S_case`。网络主学**可跨病例迁移的相对分布**，`S_case` 由入口流量 / `BC_Inlet` / Poiseuille 估计 `~Qμ/R³` 单独回归或解析给定 |
| **机制依据** | §2.2 跨病例方差大（0.35–0.57）但 §2.3 Pa p95 **Spearman 0.559** 说明**秩/分布**比绝对幅值更可学；WSS 量纲随流量/Re 线性缩放，混在一个回归头里会让大流量病例主导 loss |
| **oracle 先验（0 重训）** | 在 GT 上对每例做 `WSS = S_case·ŵ`，检验：(a) 去尺度后**池化** R² 相对当前 0.43 是否抬升；(b) `S_case` 能否由 `BC/几何` 解析/线性预测到高 R² |
| **工程价值** | 直击 **L3 病例级**（消除单例 <0.50）与 **L4 跨域**；hotspot **位置**保真比绝对 Pa 更易达 **L2** |
| **主要风险** | 归一化口径（per-case vs per-domain）；`S_case` 误差会乘性放大；与 best_wss 早峰是否解耦未知 |
| **讨论焦点** | `S_case` 用解析（Q/R³）还是可学头；是否与 G5 的 Pa p95 排序叙事天然对齐；分量场是否也走同一拆分 |

### E-H · 解析/几何 WSS 先验 + 残差学习（physics-prior residual）

| 项 | 内容 |
| --- | --- |
| **命题** | 先用**解析估计**给每个壁面点一个 WSS 基线 `τ₀(x)`（Poiseuille `4μQ/πR³` × Dean/曲率修正 × 分叉先验），网络只学**残差** `Δτ = τ_gt − τ₀`，最终 `τ = τ₀ + Δτ` |
| **机制依据** | 直管/低曲率段 WSS 由 `半径 + 流量` 近似决定（已有 `NormRadius`/`Abscissa`/`Curvature`/`Tangent`/`BC_O1–O4`）；网络当前把「易解析部分 + 难高频部分」混学，残差化可把容量集中到分叉/瘤颈高频区 |
| **oracle 先验（0 重训）** | 在 GT 上计算 `τ₀` 与 GT WSS 的 R²（分 trunk / bifurcation / neck）；若 trunk 段 `τ₀` 已达 0.4–0.6，残差目标方差显著低于原始 → 立项 |
| **工程价值** | 强归纳偏置、低数据需求；残差低秩更利 **L1/L2**；可与 E-G、E-C 叠加（残差再做低秩） |
| **主要风险** | `τ₀` 在分叉/曲率奇异处偏差大，残差在最关心区域仍高频；流量分配估计误差 |
| **讨论焦点** | `τ₀` 公式取舍（是否含 Womersley/Dean）；残差是否仍走 (s,θ) 或 LB 谱（见 E-I）；与 E-B 互斥性 |

### E-I · 壁面流形谱表示（Laplace–Beltrami 内禀谱）

| 项 | 内容 |
| --- | --- |
| **命题** | 用壁面网格的 **Laplace–Beltrami 特征函数** 作几何内禀正交基，把 WSS 表为前 k 个 LB 模态系数；预测系数而非逐点标量。是 E-A/E-C 的**免对齐内禀版**（不需 `(s,θ)` grid 对齐，分叉拓扑自然处理） |
| **机制依据** | E-A 主风险正是「跨 case grid 对齐 + 分叉奇异 + 反映射抹峰」；LB 谱是几何自适应、低秩、平滑先验，回避展开/反映射 |
| **oracle 先验（0 重训）** | 对每例壁面 GT WSS 做 LB 谱投影，画 **重构 R² vs 模态数 k** 曲线；与 §5 E-C 的 `(s,θ)`-POD（10→0.54 / 50→0.90）**同口径对照**，看 LB 是否用更少模态达到更高重构上限 |
| **工程价值** | 若重构上限高且系数可由几何编码预测 → 同时利 **L1/L2/L4**（跨病例几何泛化） |
| **主要风险** | LB 基**逐 case 不同** → 跨 case 系数不可直接共享（需 canonical/learned 谱对齐，或仅用 LB 作平滑/谱损失先验）；特征分解成本 |
| **讨论焦点** | 预测「系数」还是「场 + LB 谱正则」；是否与 E-A 合并为「内禀谱 = 唯一壁面表示」单轨；与 functional map 跨 case 对齐可行性 |

### E-J · 壁面切向动量 / 压力梯度一致性（借力「可学的压力」）

| 项 | 内容 |
| --- | --- |
| **命题** | 利用近壁切向动量平衡 `μ ∂²u_t/∂n²\|_w = ∂p/∂s`：壁面**切向压力梯度** `∂p/∂s`（由已强的压力场可得）携带 WSS 空间结构/分离信息（逆压梯度↔低/回流 WSS）。把 `∂p/∂s` 作**结构化输入或一致性正则**约束 WSS 头 |
| **机制依据** | 与 Liao 2025 PINNs 同源但**WSS 定向**：压力是本项目**已被证据支持可学**的量（R²_p 0.93–0.96），而非像 E-B 依赖全速度场；区别于历史「近壁速度差分 5276 强 No-Go」 |
| **oracle 先验（0 重训）** | 在 GT 上：(a) 闭合审计 `∂p/∂s` 与 WSS 的局部物理关系是否成立（沿 I18 审计纪律，**先审计后上 loss**）；(b) 用 GT 壁面 `∂p/∂s + 几何` probe 对 WSS 的解释力是否高于纯几何 |
| **工程价值** | 物理一致、绕开全速度场；对分离/低 WSS 区（临床高风险）尤其有用 → **L2 hotspot** |
| **主要风险** | 离散壁面上 `∂p/∂s` 估计噪声；二阶关系在弯曲/二次流处假设偏弱；circ 分量仍可能不闭合 |
| **讨论焦点** | 作**输入特征** vs **soft 一致性 loss**；是否仅在 G4 规则网格上算梯度（比点云更可控）；与 E-B 是否合并为「压力主导的物理桥」 |

### E-K · 频带分解 coarse-to-fine（低频场 + 高频 hotspot 残差）

| 项 | 内容 |
| --- | --- |
| **命题** | 两阶段：阶段 1 预测**低频平滑 WSS**（≤k 模态 / 低通），阶段 2 在局部曲率/分叉/瘤颈特征上预测**高频 hotspot 残差**，用 **hotspot-aware 加权 loss**（top-k 高 WSS 区加权）防止平滑场主导 |
| **机制依据** | POD 10→0.54 / 50→0.90 说明**低频易、高频难**；当前单头 + 均匀 loss 下平滑场吃掉梯度，hotspot 系统性偏低（§2.3） |
| **oracle 先验（0 重训）** | 分频带统计当前预测误差谱：确认高频带是主要缺口；估计 hotspot-aware 加权对 L2 RMSE 的理论上界 |
| **工程价值** | 直击 **L2 区域 / hotspot RMSE 不系统性偏低**；可叠加于 E-A/E-H/E-I 任一表示 |
| **主要风险** | 残差阶段过拟合小样本高频；频带切分阈值敏感；阶段间误差耦合 |
| **讨论焦点** | 频带用 POD/LB/小波；是否与 E-F 异方差头共用 hotspot 权重 |

### E-L · 特征保持训练 / 早峰锚定正则（直击 feature_decay）

| 项 | 内容 |
| --- | --- |
| **命题** | I7 证据为 `feature_decay`（probe R² ep6 **0.214** → ep200 **−0.179**）。在 ep~6–10 锚定 backbone 的 **WSS 可预测子空间**：用**特征蒸馏正则**惩罚该子空间漂移，或 **课程式**先 WSS 主导、后引入压力时冻结/低 LR 该子空间。是 E-E 的**机制版**（针对子空间漂移，而非事后选 ckpt） |
| **机制依据** | E-E 的事后 EMA/选优已被 **TODO-20 证伪（Δ−0.040）**；但 I7 表明可预测特征在训练中被**主动覆写** → 应在**训练时**保护，而非事后挑 ckpt |
| **oracle 先验（低成本）** | 先用 ep6 ckpt 的 backbone 特征冻结 + 仅训 wss_head（探针级，1 seed），量化相对 best_wss 0.429 的 Δ；Go 门槛 Δ≥**+0.02** |
| **工程价值** | 成本低、机制明确；**单独预期 Δ 有限**，定位为 E-A/E-B/E-G 的**叠加增益**，非独立达 M-E 路径 |
| **主要风险** | 子空间界定（probe 方向 vs 全 backbone）；锚定过强抑制压力任务；天花板低 |
| **讨论焦点** | 蒸馏正则 vs 分支冻结 vs 课程；是否与 E-G 拆分头天然兼容 |

### E-M · 分支级量纲解耦：`S_case` 下钻到 `S_branch`

| 项 | 内容 |
| --- | --- |
| **命题** | 将 E-G 的病例级尺度进一步拆为 **分支级尺度**：`WSS_b(x)=S_branch·ŵ_b(x)`，其中 `S_branch` 由局部流量分配、半径、分叉角与出口阻力决定（近似 `~ μQ_branch/R_branch³`）。网络主学每段壁面的无量纲模式，而非让一个 `S_case` 承担全树差异 |
| **机制依据** | WSS 对半径三次方和流量分配极敏感；同一病例内 trunk / branch / neck 的尺度差异可能大于病例间平均尺度差异。当前 GUO / ZHANG / CHEN 的 L3 方差未必只由 slow/fast 全局幅值导致，分叉后 `Q_branch/R_branch³` 错配会直接表现为局部 hotspot 偏低 |
| **oracle 先验（0 重训）** | 基于已有中心线/branch 标识或近似分段，在 GT 上比较三种去尺度：全局 pooled、per-case、per-branch。若 per-branch 去尺度后 pooled R² 与 L3 逐例 R² 明显高于 per-case，说明主要缺口在**局部分支标定** |
| **工程价值** | 比 E-G 更直击 **L2 hotspot** 与 **L3 单例下限**；也为 E-D 主动选数定义 `Q/R³` 覆盖空间 |
| **主要风险** | branch 切分/中心线质量；出口流量分配不可观测时 `S_branch` 估计偏差大；过细分支可能放大噪声 |
| **讨论焦点** | `S_branch` 用解析阻力网络、可学 graph head，还是二者混合；是否把 branch-wise 模式 R² 与尺度误差写成独立门禁 |

### E-N · 血管树 circuit latent：0D/1D 流量分配桥接 3D WSS

| 项 | 内容 |
| --- | --- |
| **命题** | 先从几何 + inlet/outlet BC 学一个血管树 **circuit latent**（branch resistance、pressure drop、`Q_branch`），再由局部壁面 decoder 预测 WSS 模式/残差。即 `geometry/BC → vascular circuit → wall WSS`，而不是点云直接回归全场 WSS |
| **机制依据** | 血流动力学中 WSS 的一阶控制量是流量、半径、阻力和分叉分配；当前 direct head 把全局流量分配、局部曲率和壁面高频混成一个黑箱目标，容易在跨病例上失标定 |
| **oracle 先验（0/低重训）** | 从 GT 压力/速度或 BC 估计 proxy `Q_branch` / `Δp_branch`，检验其对 branch mean WSS、top-k WSS、ROI WSS 的解释力；若 circuit proxy 对 branch 统计量 R² 高，再立项结构模型 |
| **工程价值** | 可解释、跨病例泛化友好；与 E-H 残差、E-M 分支尺度天然合流；有助于把 M-E 从“黑箱点云拟合”变成可审计的数字孪生链路 |
| **主要风险** | 中心线/拓扑图构建成本；AG 数据的出口边界条件是否足够支撑可辨识的流量分配；0D proxy 与 3D 局部分离/二次流仍有 gap |
| **讨论焦点** | circuit latent 是显式物理参数还是隐式 branch graph token；是否先做 branch 统计量任务，再接壁面逐点 decoder |

### E-O · 时间相位系数建模：81 帧不再独立回归

| 项 | 内容 |
| --- | --- |
| **命题** | 将一个病例的 81 帧 WSS 表为少量时间基函数系数：`τ(x,t)=τ_mean(x)+Σ_k a_k(x)sin(kφ)+b_k(x)cos(kφ)`，模型预测空间系数而非逐帧独立场；或在 latent 中显式输入/预测心动周期相位 |
| **机制依据** | WSS hotspot 的空间位置在一个心动周期内通常具有连续性，幅值随相位变化；逐帧独立训练会浪费样本结构，并可能把相位/幅值误差误判为空间模式误差 |
| **oracle 先验（0 重训）** | 对 GT WSS 做 temporal Fourier / PCA：检查前 2–4 个谐波或低秩时间模态的重构 R²；再将当前 I6-diag 预测误差分解为空间错位、幅值低估、相位错位三类 |
| **工程价值** | 提升 L3 稳定性与时间一致性；可减少输出自由度，给 E-A/E-I/E-K 的空间表示减负 |
| **主要风险** | 当前 M-E 门禁尚未定义时间一致性；相位归一化/心动周期对齐错误会引入系统偏差；fast/slow 是否共享同一时间基需审计 |
| **讨论焦点** | 预测 Fourier 系数、temporal POD 系数，还是做时序 decoder；L3 eval 是否补充 per-case temporal consistency |

### E-P · Hotspot 检测 + 条件残差回归（结构路由而非单纯排序 loss）

| 项 | 内容 |
| --- | --- |
| **命题** | 将 WSS 头拆为三段：hotspot probability / background WSS / hotspot residual-amplifier。先定位高剪切候选区，再对这些区域使用更高容量或更高权重的局部 decoder 修正幅值 |
| **机制依据** | TODO-42 排序 loss No-Go 说明“只改 loss”不足；但 §2.3 可视化显示 hotspot 系统性偏低，像是平滑回归头在高值稀疏区欠拟合。需要让 hotspot 预测参与**结构路由**，而不是作为辅助指标挂在同一个 head 上 |
| **oracle 先验（0/低重训）** | 用当前预测计算 top-k hotspot 的定位指标（Dice / recall / center distance）与幅值误差：若定位尚可但幅值低估明显，则 P 路线成立；若定位也错，则优先 E-A/E-I 表示路线 |
| **工程价值** | 直击 **L2 hotspot RMSE 不系统性偏低**；可作为 E-K 高频残差阶段的具体实现 |
| **主要风险** | top-k 标签阈值敏感；小样本下热点分支容易过拟合；若 hotspot 位置本身不可学，结构路由会放大误差 |
| **讨论焦点** | hotspot label 用 per-case percentile、绝对 WSS、还是临床 ROI；路由是 hard mask、soft gate，还是 mixture head |

### E-Q · WSS 后验单调校准：诊断“排序已学、幅值未准”

| 项 | 内容 |
| --- | --- |
| **命题** | 在不改变主模型的情况下，对 `τ_pred` 做病例/分支/ROI 条件的单调校准：`τ_cal=f(τ_pred,Q,R,curvature,branch,t_norm)`，例如 isotonic regression、分段 spline、quantile mapping 或 monotonic MLP |
| **机制依据** | 当前 Pa p95 Spearman 有信号且 hotspot 系统性偏低，可能说明模型学到了相对排序/位置，但绝对幅值标定失败。单调校准是判断“模式 vs 幅值”的低成本探针 |
| **oracle 先验（0 重训）** | 使用 val 拟合校准器、test 只评一次：若 L2 hotspot RMSE 与 L3 逐例 R² 明显改善，同时 Spearman 基本不变，则后续主线应优先 E-G/E-M/E-H 标定；若改善有限，说明空间模式本身不足 |
| **工程价值** | 低成本区分失败类型；也可作为 M-E 前的工程兜底，但不能替代结构模型达标 |
| **主要风险** | 校准器若按病例拟合会泄漏；单调变换可能牺牲局部物理一致性；外推到新 Re/Wo 区间风险高 |
| **讨论焦点** | 校准变量取 case、branch 还是 ROI；是否把校准前/后指标都写入 M-E 表，防止“后处理冒充模型能力” |

### E-R · 目标重参数化与标签噪声地板：先确认 0.85 是否可达

| 项 | 内容 |
| --- | --- |
| **命题** | 系统比较 `τ`、`log(τ+ε)`、`Cf=τ/(0.5ρU_ref²)`、per-case/branch percentile field 等 target，并做 CFD/插值标签自一致性审计，确认 M-E 的 R²_wss≥0.85 是否被标签噪声与后处理链允许 |
| **机制依据** | WSS 正值、长尾、hotspot 稀疏，直接归一化回归容易压低高值区；若 CFD 网格、壁面插值或时间采样自身在 hotspot 区存在显著噪声，则统一点级 R² 门槛需拆成可信区/频带/ROI 门槛 |
| **oracle 先验（0 重训）** | (a) 对不同 target 做低秩重构、病例方差、hotspot 长尾统计；(b) 用同一 CFD 场的不同插值/网格/相邻时间步近似估计 self-consistency R²；(c) 比较当前模型在各 target 空间的误差结构 |
| **工程价值** | 给 E-G/E-K/E-P 提供更适合的监督目标；避免把不可达的标签噪声当成模型瓶颈；可校准 M-E 阈值 |
| **主要风险** | 缺少多网格 CFD 时 self-consistency 只能做 proxy；target 变换提升 R² 但降低物理可解释性；percentile target 需要单独恢复绝对幅值 |
| **讨论焦点** | M-E 是否保留原始 Pa 量纲硬门槛；target 变换只用于训练还是也作为中间门禁；标签噪声地板是否先于 GPU 主线立项 |

### 对现有方向 E-A–E-E 的强化建议（不改命题，调执行假设）

| 方向 | 强化点（讨论用） |
| --- | --- |
| **E-A** | 2D→3D **gap** 是真瓶颈：用 **branch-wise 局部图 + 重叠一致性缝合** 替代单一全局 `(s,θ)` grid；反映射端加 **peak-preserving / 超分** 解码与 hotspot 加权（接 E-K），缓解「抹平峰值」 |
| **E-B** | **不要预测全速度场**：只预测**固定法向偏移 δ 处的近壁切向速度** `u_t(δ)`，由壁面函数 `WSS≈μ·u_t(δ)/δ`（或可学壁函数）得 WSS，维度远低、与 WSS 直接耦合；并入 E-J 压力梯度一致性，规避 5276 direct vel_diff 失败模式 |
| **E-C** | 缺的是 **input→系数 probe 过线记录**：先补该 oracle；表示从 `(s,θ)`-POD 升级到 **LB 内禀谱（E-I）** 以解 θ-roll×POD 互斥与跨 case 对齐 |
| **E-D** | 数据轴定位为**乘子**：优先 **hotspot/高 WSS ROI 主动选数（H1）** 而非全量扩池；为 E-A/E-G/E-M 的跨病例/分支 scale 一致性提供更均衡的 `S_case/S_branch` 覆盖 |
| **E-E** | 用 **E-L 训练时锚定** 取代事后 EMA（TODO-20 已证伪）；若锚定仍 Δ<+0.02，转向 E-B/E-G 架构级拆分 |

### 5+.x oracle 波次清单与 Go 阈值（讨论初稿 · 阈值待共识）

> 用途：把上述方向的「先验判据」收敛为**第一波 0/低重训诊断**，用结果重排 §6、再决定谁进 GPU。**阈值为讨论初稿**，立项前由团队确认；产出统一写入 eval 脚本（与 §8.5 基建合并）。

| 编号 | 方向 | 0/低重训 oracle 问题 | 量化判据 | 试探性 Go 阈值 | 命中门禁 |
| --- | --- | --- | --- | --- | --- |
| O1 | **E-G** | 去尺度后空间模式是否更可学；`S_case` 是否可由 BC/几何预测 | 去尺度**池化** R²_wss；`S_case` 线性回归 R² | 模式 R² ≥ **0.60** **且** `S_case` R² ≥ **0.80** | L1/L3/L4 |
| O2 | **E-H** | 解析 `τ₀` 在各区的基线解释力；残差是否更低秩 | `τ₀` vs GT R²（分 trunk/bif/neck）；残差方差/原始方差 | trunk `τ₀` R² ≥ **0.50** **且** 残差方差 ≤ **0.6×** 原始 | L1/L2 |
| O3 | **E-I** | LB 谱重构上限是否优于 `(s,θ)`-POD（同口径） | 重构 R² vs 模态数 k 曲线 | k≤50 重构 R² ≥ POD 同 k；或更少模态达 **0.90** | L1/L2 |

| O6 | **E-L** | ep6 冻结 backbone + 仅训 wss_head 的增量 | test `wss_r2_wss` 相对 best_wss **0.429** 的 Δ | Δ ≥ **+0.02**（探针级，1 seed） | 叠加增益 |
| O7 | **E-M** | per-branch 去尺度是否优于 per-case 去尺度 | 去尺度 pooled/L3 R²；branch mean/top-k WSS 解释力 | per-branch 模式 R² 较 per-case **+0.05** 且单例下限抬升 | L2/L3/L4 |
| O8 | **E-N** | circuit proxy 是否解释 branch WSS 统计量 | `Q_branch/Δp/R` proxy → branch mean/top-k WSS R² | branch 统计 R² ≥ **0.70** 或较纯几何 **+0.10** | L2/L3/L4 |
| O9 | **E-O** | 时间低秩/谐波是否足够重构 81 帧 WSS | temporal Fourier/PCA 重构 R²；误差相位分解 | ≤4 个谐波或低秩模态重构 R² ≥ **0.90** | L3/L4 |
| O10 | **E-P** | 当前模型是 hotspot 定位错还是幅值低估 | top-k Dice/recall；hotspot bias/RMSE | top-k recall 可接受且 hotspot bias 显著为负 | L2 |
| O11 | **E-Q** | 单调校准能否显著改善幅值而不改排序 | val 拟合校准器，test L2/L3 指标 | hotspot RMSE 降低 ≥ **10%** 或 L3 最差病例 R² +**0.05** | L2/L3 |
| O12 | **E-R** | target 变换/标签噪声是否重塑 M-E 上限 | target 低秩性、长尾统计、self-consistency R² | 发现更优 target 或 self-consistency 接近门槛需重定 M-E | L1/L2/L3 |

**判读规则**：O1–O5、O7–O12 属 **0 重训**（GT/probe/后处理分析，不占长训 GPU），应**先于**任何 E-A/E-B 级重训立项；O6 为低成本探针。任一 oracle 先验未过线 → 对应方向**降级或封口**，不进 GPU 队列。

| **E-F 不确定性** | 兜底 | 低 | 低 | M-E 失败后的产品策略 |

**讨论建议（非执行序）**：把「**0 重训 oracle**」类（E-G/E-H/E-I/E-J/E-K/E-M/E-O/E-P/E-Q/E-R 的先验判据）作为**第一波低成本筛子**，尤其先回答 **scale/标定（E-G/E-M/E-Q）/ target-noise（E-R）/ hotspot 定位 vs 幅值（E-P）/ 时间低秩（E-O）** 四个问题，再决定谁进 GPU 立项；E-L 作低成本探针；E-A/E-B/E-N 为重投入主战场，应在 oracle 波次给出**目标方差/重构上限/branch 解释力**证据后再排。

## 8. 供多智能体讨论的开放问题

1. **M-E 阈值**：§3 初稿（R²_wss≥0.85 等）是否对齐业务？是否分 P/WSS/velocity 不同标准？
2. **主战场**：E-A（2D 壁面）vs E-B（速度场→WSS）是否互斥？能否分阶段组合？
3. **矢量 WSS**（裁决 §3「待定门禁」）：M-E 是否必须 circ/rad 分量达标，还是 magnitude + 方向误差即可？裁决结果决定是否把矢量分量并入 L1–L4 硬门禁。
4. **V3P vs V3D**：工程部署是否必须三域？若是，M-E 表是否迁到 V3D split？
5. **eval 基建**：L2/L3 区域 ROI、hotspot RMSE 脚本是否先于任何 GPU 立项？
6. **I6-a**：在 E-A/E-B 未动前，是否 worth 1 seed 探针，还是直接判为增量不足？
7. **CROWN 速度线**：E-B 是否借鉴 CROWN 速度 NMAE 路径，但换几何/BC 输入（V3P 优势）？
8. **模式 vs 幅值（E-G）**：跨病例缺口是否主要是 `S_case` 标定不一致？若 oracle 证实去尺度后池化 R² 显著抬升，是否应把「无量纲模式 R²」与「`S_case` 误差」拆成**两个独立门禁**，而非单一 `wss_r2_wss`？
9. **解析先验（E-H）**：`τ₀` 公式取到哪一档（Poiseuille / +Dean / +Womersley）才使残差方差最低？trunk 段先验 R² 多少算「值得残差化」？
10. **表示统一（E-A/E-C/E-I）**：`(s,θ)`-POD、LB 内禀谱、3D 点云三选一还是分工？是否存在「**一个壁面表示**」能同时满足 L1/L2 且免跨 case 对齐？
11. **oracle 波次优先级**：E-G/E-H/E-I/E-J/E-K 的 0 重训 oracle 是否应**先于任何 GPU 立项**统一跑一轮，用结果重排 §6？（与 §8.5 eval 基建合并立项）
12. **分支尺度（E-M/E-N）**：跨病例缺口是否应进一步拆到 `S_branch`？如果 per-branch 去尺度显著优于 per-case，M-E 是否新增 branch-wise 门禁？
13. **时间维（E-O）**：81 帧是否应作为周期场共同建模？若 temporal Fourier/PCA 低秩成立，L3 eval 是否补充时间一致性与相位误差？
14. **hotspot 机制（E-P）**：当前模型在高 WSS 区主要是**定位失败**还是**定位对但幅值低估**？这决定走表示换轨（E-A/E-I）还是条件残差/标定（E-P/E-Q）。
15. **target 与噪声地板（E-R）**：`τ`、`log τ`、`Cf`、percentile field 哪个更适合训练？CFD/插值 self-consistency 是否足以支撑 R²_wss≥0.85 的统一硬门槛？
16. **后处理边界（E-Q）**：若单调校准显著改善 L2/L3，它是工程兜底、诊断工具，还是可以作为 M-E pipeline 的正式组成？需要如何防止 val/test 泄漏？
