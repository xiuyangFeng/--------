## 0. 为什么路径 A–F 没把精度推上去（再压缩一层根因）

把已闭环的 No-Go 证据再压一层，能得到一个**比「继续单变量」更硬、也更乐观**的结论：

| 已被实验排除 | 留下的**真正约束** | 对应的结构性解法 |
| --- | --- | --- |
| 容量、loss 权重、4 维冗余、global WSS 表示微调（路径 A） | **(R1) 坐标系负担**：全局 xyz 下，网络要**隐式学每个 case 的血管朝向旋转**才能对齐 `wss_x/y`；60 case → 记忆而非泛化 | **架构级等变**（G1） |
| `dist_to_wall` / 法向梯度上下文 / 多档 kNN（路径 B） | **(R2) 噪声放大**：「预测场 → 差分推 WSS」把预测噪声逐点放大（VelGradB1 5170 / vel_diff 5276/5302 全负） | **物理先验 + 残差学习**（G2，让网络学修正量而非从零回归） |
| per-domain norm（+0.003）、扩 val（val15 强 No-Go） | **(R3) 小样本 + 高频目标**：`wss_x/y` 是 case-specific 二次流（Dean 涡 / 分叉），60–179 case 在标准监督下信息量不足 | **数据倍增 / 预训练**（G3） |
| 加 loss / 换小 head / 加正则（路径 F loss 线 5253/5266–69） | **(R4) 边界算子未写进结构**：`WSS=μ·∂u_t/∂n|_wall` 的边界结构没有进入网络归纳偏置 | **2D 壁面场算子学习**（G4） |

**关键判断（也是本方案的立项前提）**：

- 作者团队的 F0 noise-floor（三 seed CI 上界 0.399、残差分层未触发）是**在「同一 FieldPointNeXt local-pool + 全局 xyz + direct head」表示空间内测得的上界**。它**不约束**改变了表示/归纳偏置的 G1–G4——这正是值得继续冲的理由。
- 同时必须诚实：oracle v2（5265）显示 `wss_x/y` **部分可重建**（GT 近壁差分 Spearman ~0.61–0.63、`|∇p|` Spearman 0.73、POD 1D 10 模态 0.54），但**不是无限可学**。因此 G 路径的每一条都配一个**0 重训前置 oracle**，先证「表示层面天花板确实更高」，再烧 GPU。

| ID | oracle | 做法 | 触发判据（过则立项） |
| --- | --- | --- | --- |
| **G0-a** | **坐标系负担量化**（验证 R1） | 用 GT：把每个 case 的 `wss` 旋转进**中心线局部坐标系**（已有 `Tangent_*` + 估计法向 n̂），统计 local `wss_axial/circ/rad` 的**跨 case 分布方差** vs global `wss_x/y/z`；并用一个**线性探针**在 local vs global 下拟合 test R² | local 表示下 circ/rad 的可学性（线性探针 R²）显著 > global x/y（如 +0.1）→ **G1 必做** |

| **G0-d** | **残差基线可行性**（强化 G2） | 算每个壁面点的解析 WSS₀（见 §5 Womersley/Poiseuille 近似），看 `WSS − WSS₀` 的方差是否显著小于 `WSS`，且残差是否仍与几何相关 | 残差方差降 **>30%** 且残差有结构 → G2 残差头立项 |

## 4. G1 · 等变架构（最高优先级 · 本方案核心）

> **核心命题**：`wss_x/y` 失败的主因不是「学不到」，而是「网络要在 60 case 上**同时学旋转对齐 + 学 WSS**」。把旋转对齐变成**架构级保证**（等变），网络只需学 WSS 在局部不变特征上的依赖。这是文献中唯一**已验证**在血管几何上拿到低误差矢量 WSS 的家族（Suk 2024）。

分两档落地，**先轻后重**。其中 G1-a 不能再写成一个单变量黑箱：VN 头、切空间投影、局部基底张量积分别回答不同问题，必须拆成小阶梯，避免成功后无法归因。

### G1-a · TODO-54：Vector-Neuron 矢量 WSS 头（轻量入口 · 推荐第一跳）

- **改动阶梯（必须按顺序做）**：
  1. `V3P-G-54a-VNHeadPlain`：保留现 `FieldPointNeXt` backbone，仅把 `wss_head` 换成 **Vector Neurons 等变矢量头**（Deng 2021）；不引入法向、不引入局部基底、不做投影。这一档只回答“矢量等变 head 本身是否有增益”。
  2. `V3P-G-54b-VNHeadProj`：在 54a 基础上加入无参**切空间正交投影层**：$\vec{\tau}_{\text{final}} = \vec{\tau} - (\vec{\tau} \cdot \vec{n})\vec{n}$，硬性保证预测 WSS 位于壁面切流形内。这一档引入法向，回答“物理切空间硬约束是否有增益”。
  3. `V3P-G-54c-VNHeadLocalBasis`：在 54b 基础上，将 backbone 标量特征与壁面局部几何参照系基底（法向 $n$、两个正交切向 $t_1, t_2$）做张量积，作为 VN 头的等变矢量特征输入。这一档回答“局部基底方向信息是否弥补 backbone 标量特征的方向丢失”。
- **为何可能赢**：矢量输出对 SO(3) 等变 → `wss_x/y` 不再需要隐式坐标系；与 5277 PGradFeat 正交，可叠加。
- **口径**：54a 才是严格“仅换 head”单变量；54b/54c 是带法向和局部基底的结构增强，必须分表。global 标签不变，监督在等变输出上算 MSE；每一档都要单独对照 4957 和上一档。
- **Go（seed1）**：`wss_r2_wss > 0.42` 且 `wss_x` 或 `wss_y` test R² **> 0.10**（这是历史从未达到的信号）→ 补 seed2/3。
- **归因判读**：若 54a 无效、54b 有效，增益主要来自切空间物理约束；若 54b 无效、54c 有效，增益主要来自局部方向表示；若只有 54c 有效，不能宣称“VN head 单独解决问题”。
- **风险/对策**：等变头容量小可能欠拟合 → 允许 2–3 层 VN-MLP；法向/相对坐标构造要平移不变（复用 5303 审计结论 H5）。
- **容量错配防线**：如果 backbone (PointNeXt) 提取的特征依然高度依赖于全局坐标系，仅在最后一层强加等变性可能带不动。建议前置探针：对输入点云做 SO(3) 旋转，若 backbone 倒数第二层特征分布漂移巨大，则需直接跳至 G1-b，或考虑引入 PaiNN/MACE 等更强的 E(3) 等变网络。

### G1-b · TODO-55：SE(3)/E(3) 等变 backbone（全量换轨 · G1-a Go 后）

- **改动**：backbone 换 **EGNN / Vector-Neuron-DGCNN / 等变 message passing**（Suk 2024 的 gauge/SE(3) 等变思路的点云版），矢量场（坐标、相对位移、近壁速度差）走等变通道，标量（`p`、几何先验、BC）走不变通道。
- **依据**：Suk 2024 在表面网格上矢量 WSS 误差 7.6%、可条件化 BC；MICCAI 2023 证小数据上等变省样本——正中 R1+R3。
- **口径**：新 backbone = 新 split 无关、新表；**禁 warm-start** 旧 ckpt（等变权重不兼容，见发散 §5.3 对 TODO-29 的提示）。先 `V3P-G-55-EGNN-Probe`（单目标 WSS 上限），再双域主线。
- **Go**：三 seed `wss_r2_wss` 稳定 **> 0.45** 且 `r2_p ≥ 0.85` 不崩 → 成为新母版候选，进入临床/区域 eval。
- **成本**：算力已确认充足；预留等变网**收敛慢、对 lr/初始化敏感**的调参预算（建议先小子集过拟合自检）。

### G1-c · 计算成本与图结构澄清（立项前必读 · 防误区）

> **常见误区**：「等变是不是要把点云转成真实 mesh 图？那样会损失大量算力。」**不会。** 等变与「图结构」是两个**正交**维度：等变是**网络算子的性质**（特征在旋转下如何变换），与是否用图、用哪种图无关。

| 关注点 | 事实 | 成本 |
| --- | --- | --- |
| **要不要新建图 / 真实 mesh？** | **不要**。复用现 pipeline 已建的 **kNN `edge_index`**（采样点重建）；**不需要** Fluent 原始 mesh connectivity | 预处理侧**零额外开销** |
| **G1-a（VN 矢量头）** | 点级 + 全局池化，**不碰图**；只把 head 末层「标量线性」换成「矢量线性」（通道升成 3D 向量，线性组合 + 叉乘保持等变） | ≈ **3× 一个本就极小的 head**，可忽略 |
| **G1-b（EGNN backbone）** | 跑在**同一张 kNN 图**上，边数不变；消息里多带几条坐标相对向量通道做等变更新 | O(边数 × 几个向量通道)，与现 kNN pooling **同量级**，非数量级增长 |
| **真正贵的等变网（不推荐）** | **SE(3)-Transformer / Tensor Field Networks** 用**高阶球谐（l≥2）**才慢 | 高；本方案**刻意避开** |

**关键取舍**：EGNN / Vector Neurons / PaiNN 这一类只用**标量 + 一阶向量（type-1）**，靠线性组合与叉乘实现等变，**不引入球谐**，所以便宜。Suk 2024 引为「血管上矢量 WSS 可行」的**结论**依据；其 gauge-equivariant **mesh** CNN 需真实表面网格 + 平行移动，**不在本方案采用范围**——对点云用 EGNN/VN 是更省的等价替代。

> 成本排序：**G1-a 矢量头 ≈ 免费** < **G1-b EGNN（复用 kNN 图，同量级）** ≪ SE(3)-Transformer（高阶球谐，贵，不采用）。因此执行上先上零图改动的 G1-a，有 `wss_x/y` 信号再上 G1-b，全程复用现成 kNN 图。

> **关键归因纪律**：G1-a 与 G1-b 之间唯一变化是 backbone 是否整体等变；与 G2/G3 严格分表，避免重演「把所有改进归因于 PointNeXt 更强」的旧错（路线计划 §5.4）。

## 5. G2 · 物理残差 + 压力-WSS 耦合（高 ROI · 便宜 · 借鉴大模型残差思想）

> **命题**：与其让网络从零回归 WSS（高方差、噪声放大），不如让它学**对物理基线的修正量**（残差），并把已验证最强的物理信号 `|∇p|`（Spearman 0.73）从「特征 concat」升级为「结构耦合」。这把「残差连接」的思想用在**目标侧**而非仅网络内部。

### G2-a · TODO-56：解析基线 + 残差学习头（先过 G0-d）

- **先拆三档，禁止一次全上**：
  1. `G2-a0` **Cf-only / WSS₀ oracle**：不改模型，先在 train-only 统计下计算 $C_f = WSS / (0.5 \rho U_{\text{in}}^2)$、Re/Wo 分布、解析 WSS₀ 与 GT WSS 的相关、残差方差是否下降。此档只判断“无量纲化和解析基线是否值得进入训练”。
  2. `G2-a1` **固定 WSS₀ 残差头**：对每个壁面点用局部 Poiseuille/Womersley 近似估解析 WSS₀（轴对称近似，circ/rad 置 0），网络只学 $\Delta WSS = WSS - WSS_0$；推理时 $WSS = WSS_0 + \Delta WSS$。此档回答“固定物理基线是否降低目标方差”。
  3. `G2-a2` **自适应缩放残差头**：只有 2-a1 过线后，才引入网络同步预测的空间缩放标量 $\sigma(x) \in [0,1]$，拟合 $WSS(x) = \sigma(x) \cdot WSS_0(x) + \Delta WSS(x)$。此档用于病理区解析假设退化时自动降低 WSS₀ 权重。
- **无量纲化口径（可证伪假设，不是预设结论）**：网络可在 $C_f$ 空间训练，并把 Re/Wo 放入 `global_cond`，输出端再乘以病例特异性的 $0.5 \rho U_{\text{in}}^2$ 还原物理 WSS。它的目标是**缓解** AAA/AG/ILO 三域物理量纲失配与大域主导 loss，不应写成“彻底解决”；是否成立必须由 V3D 分域指标验证。
- **基线退化风险与替代方案**：若 G0-d 证实解析基线方差不降反升，固定 WSS₀ 残差线关闭；可改用 **低保真 CFD（粗网格/稳态）** 或 **纯几何 1D 降阶模型** 作为 teacher，但这将归入 G3/G7 预研，不再与 G2-a1 混表。
- **Go**：相对同架构、同 split、同训练预算的非残差/Cf 对照，`wss_r2_wss` +0.03 或 V3D 分域均不退且弱域提升；病例级 p95/Pa Pearson 必须不退化。

### G2-b · TODO-57：压力梯度 → WSS 交叉注意力头（升级 5277 / TODO-47）

- **改动**：把 `wss_head` 末层换成 **kernel-attention**：以壁面点为 query、近壁内部点的 `(p, ∇p, 速度差代理)` 为 key/value，核函数 `~1/r` 初始化（软 Green 函数）。
  - **流向有向无环图（DAG）积分算子**：基于边界层动量积分方程机理（壁面 WSS 是沿流向压力梯度累积和动量损失的空间积分结果），在图消息传递中利用流速方向构建有向无环图，约束信息只允许由“上游向近壁下游”传递，并对流向压力梯度做积分累积，作为对 WSS 的宏观流体力学软约束。
- **依据**：oracle `|∇p|` Spearman 0.73 是全项目最强物理信号；5277/5300/5301 已证输入侧 `|∇p|` 在 seed1/2 可复现 +0.007。
- **口径**：单变量；可与 G1-a 矢量头组合（注意分表先各自单跑）。
- **Go**：点级 ≥ 4957 且区域 top-k Dice / 病例级优于 4957。

## 7. G4 · 2D 壁面展开 + 算子学习（高上限 · G0-b 门控）

- **改动**：把壁面点按 **(Abscissa s, 周向角 θ)** 展开成规则 2D 网格，WSS 场变成 2D 图像，用 **2D U-Net / Geo-FNO** 学映射（MultiViewUNet 先例）。规则网格上 `μ·∂u_t/∂n` 的空间结构远比非结构点云易学。
- **门控**：必须先过 **G0-b（2D POD 重建 R²>0.6）**；分叉处展开奇异 → 需分支级展开或重叠拼接。
- **流形展开替代与避坑指南**：鉴于全局 2D 展开在分叉处存在拓扑奇异与严重的度量扭曲（Metric Distortion），若全局展开不可行：
  - **避坑（不要把 H4 oracle 扩展成谱图卷积）**：H4 仅用于单病例壁面谱基重建上限诊断。若把它扩展成跨病例**谱图卷积（Spectral Graph Convolution）**，会遇到不同血管几何拓扑带来的拉普拉斯特征基底“谱漂移（Spectral Shift）”，极难具备跨几何泛化能力，故不建议将谱卷积作为跨病例的 G4 替代方案。
  - **推荐方案：局部测地极坐标展开（Geodesic Patch CNN）**：以每个壁面局部极大 WSS 区域中心点为中心，沿壁面流形计算测地距离，并在其局部正切空间展开（Parameterization）为规则的 2D 贴片（Patch）。利用 2D CNN/U-Net 进行局部高精预测，而在不同 Patch 之间用图神经网络进行消息传递。这能完美规避血管分叉处的拓扑奇异性。
- **定位**：**换轨级**工作（V4 量级），作为 G1 若止步 0.45 时的高上限备选；算力充足可与 G1 并行预研预处理链。
- **Go**：2D U-Net test `wss_r2_wss` 明显 > G1 best，且可反映射回 3D 壁面点。

## 11. 风险与诚实边界

1. **等变网不是银弹**：训练更难、收敛慢、对超参敏感；建议先「小子集过拟合自检」确认实现正确再上全量。若 G1-b 调参到位仍 ≤0.42 且 G0-a 显示 local 表示可学性也低，则**坐标系负担不是主瓶颈**，应转 G3（数据）或 G4（换轨）。
2. **oracle 成立 ≠ 训练有效**（路径 F 已多次踩坑）：G0 只证「天花板更高」，每条 G 路径仍以**实际 test 三 seed**为准，单 seed 正信号不直接进母版。
3. **G2 残差基线偏差**：若解析 WSS₀ 与 GT 相关性低（G0-d 不过），残差学习退化为普通回归，不强推。
4. **不可约噪声**：`wss_x/y` 受入口微扰主导，存在**物理上不可学**的成分；即便结构性方法，`wss_x/y` R² 可能停在 0.1–0.3 而非 0.5+。届时 G5 异方差 + 区域 surrogate 是诚实且可发表的落点。
5. **分表纪律**：G1/G2/G3/G4 改了 backbone/target/数据，**一律新 exp_id + 新表**，禁止与 4957/0.394±0.005/0.243/0.406 混表。

## 13. 需要继续追问的问题与对策（面向后续精度提升）

> 本节把讨论中的疑问改写成**可验证问题**。它不是再新增一批并行试验，而是给 G0–G5 加一层审查门：先判断到底卡在数据、标签、表示、物理耦合还是评估口径，再决定是否开训。

### 13.2 推荐把问题收敛成四个 P0 闭环

1. **表示闭环**：Q1 + Q2。先证明 local / equivariant 表示确实让横向 WSS 变得可学，再开 G1-a。
2. **标签闭环**：Q3。先排除 wall mask、法向、量纲和异常病例问题，避免把脏标签当作架构瓶颈。
3. **物理闭环**：Q5 + Q6。先用 0 重训 oracle 判断压力梯度、残差基线和无量纲化是否有上限，再开 G2。
4. **评估闭环**：Q4 + Q7 + Q10。先固定“什么叫真的精度提高”，用病例级和高 WSS 区域指标约束点级 R²。

### 13.3 对后续执行的具体建议

- **不要同时开太多训练分支**：G0 / QA 过线后，第一批只开 `G1-a VNHead` 与 `G2-a ResidualCf` 两个 seed1 probe；G3 只做数据/预训练预研，不直接宣称指标。
- **每个新分支必须能回答一个问题**：例如 G1-a 回答 Q1/Q2，G2-a 回答 Q5/Q6，G4 只在 Q8/G0-b 过线后回答“壁面算子换轨是否必要”。
- **精度提升必须拆开写**：点级 R²、x/y 或 circ/rad 结构信号、病例级 Pa、高 WSS 区域四者分开报告；禁止用单个 `wss_r2_wss` 覆盖所有结论。
- **若 G1/G2 都只到 0.42 左右**：不继续扫小超参，优先判断是 Q3 标签噪声还是 Q8 拓扑缺口；若二者都无法解释，则把主线切到 G5 可信区间 + 区域 surrogate 叙事。
- **若出现单 seed >0.45**：先补 seed2/3 和 case bootstrap，再做病例/区域 eval；不要直接替换 4957 母版。

## 14. 额外发散想法观察池（暂不占 TODO 编号）

> 以下想法比 G1–G5 更发散，当前**不改变执行顺序、不直接开训、不新占 TODO 编号**。它们的价值在于：如果 G0 / QA / G1 / G2 之后仍卡在 `0.42–0.45`，可以作为下一轮换轨候选。每一项都必须先做低成本证据检查，再决定是否写入正式待办。

| 编号 | 发散想法 | 为什么可能有效 | 低成本前置检查 | 何时升级为正式路线 |
| --- | --- | --- | --- | --- |
| **H1** | **主动选数 / 主动 CFD 扩池** | 不是随机增加 case，而是用当前模型的 residual、uncertainty、几何聚类和高 WSS 失败区域，选择最能补信息的病例去补 CFD；若主瓶颈是覆盖不足，ROI 高于盲目扩数据 | 用 4957 / 5277 / 后续 G1 的误差图训练一个 case-level error predictor，检查高误差是否集中在少数几何簇、分叉形态或 Re/Wo 区间 | 若 top residual 簇解释 **>30%** WSS 平方误差，且候选池有可补病例，则作为 TODO-1 的“主动选数子策略”；固定 test 不动，防选择偏差 |
| **H2** | **拓扑条件化坐标系** | 当前 `Abscissa + Tangent + Curvature` 仍偏局部；高 WSS 往往和“距入口/分叉/出口的拓扑位置”有关。显式 `branch_id`、`dist_to_bifurcation`、branch-relative `s/θ/r` 可能比纯 kNN 更稳 | 用 GT WSS 做 branch-aware GBDT / KNN probe，比较只用现有几何特征 vs 加拓扑坐标后的 top-k AUC、p95 Pearson 和局部分量 R² | 若拓扑 probe 对高 WSS 区域或横向分量有显著增益，则升级为 G4 前置，优先做 centerline token / branch-aware wall unfolding |
| **H3** | **normal-ray 壁面管算子** | WSS 是法向梯度，kNN 近壁点不保证沿法向采样。对每个 wall point 沿内法向取固定距离的 1D ray / tube，让网络在法向线上学 `∂u_t/∂n`，比无序邻域更贴近物理算子 | 先用 GT 速度在法向 ray 上插值，计算 learned / finite-difference WSS proxy 与 GT WSS 的 Spearman、R²、p95 相关；同时检查法向翻转和插值失败率 | 若 ray proxy Spearman **>0.65** 且失败率可控，升级为 G2/G4 之间的物理算子路线；若法向/插值不稳，先回到 Q3 标签 QA |
| **H4** | **壁面 Laplace-Beltrami 谱基重建 oracle** | 2D 展开在分叉处有奇异点；单病例壁面谱基可作为低维重建上限诊断，判断 WSS 是否主要由低/中频曲面模式组成 | 对每个 case 壁面构图，做 20/50/100 个谱基**重建 GT WSS**，报 reconstruction R²、top-k Dice 和高 WSS 区域保真度；不训练跨病例谱卷积 | 若 50 模态重建 R² **>0.65** 且高 WSS top-k 不丢，只升级为 G4 的 0 重训 oracle / POD 替代诊断；除非能证明跨病例谱基可对齐，否则不作为谱卷积或谱系数预测主线 |
| **H5** | **两阶段 residual corrector / stacking** | 若 4957 或 G1 能学低频但高 WSS 残差呈稳定空间模式，可以先冻结主模型，再用轻量 residual corrector 修正局部误差，避免端到端训练破坏已学到的压力和低频 WSS | 用 cross-fit 方式在 train/val 内拟合残差，检查 residual 是否能被几何、pressure-gradient、拓扑位置解释；严禁直接用 test 残差调参 | 若 residual probe R² **>0.15** 或高 WSS top-k 明显改善，作为 G1/G2 之后的后处理路线；若只在 val 有效则关闭 |
| **H6** | **病例内时间 / 相位一致性建模** | 若同一 case 有多个 `t_norm`，WSS 的时相变化应连续，当前逐图训练可能浪费了病例内信息；加入 phase token / Fourier time embedding / temporal consistency 可降低横向分量噪声 | 统计同 case 不同 `t_norm` 的 WSS profile 相关、p95 相位曲线和残差相位漂移；判断病例内变化是否比跨 case 更可学 | 若病例内时序相关显著且数据覆盖足够，升级为 V3T 分支；若每 case 时间点稀疏或相位口径不齐，不立项 |
| **H7** | **低阶 hemodynamic teacher 蒸馏** | 粗网格 CFD 成本仍高；可以用 1D centerline flow、压力降、局部半径/流量解析模型输出低阶 teacher，让网络先学可解释的血流趋势，再学残差 | 对已有 case 生成低阶 teacher 特征，做 teacher-only probe 和 residual probe，检查是否解释 WSS 低频分量或病例 p95 | 若 teacher 对病例级 p95/Pa 有稳定解释力，则并入 G2-a 残差基线；若与 GT 弱相关，不作为训练监督 |
| **H8** | **不确定性分解驱动的可靠区域预测** | 点级 R² 不一定继续涨，但模型可以知道哪里可信。把 epistemic / aleatoric uncertainty 分开，有助于筛出不可学区域、主动选数和临床可信区间 | 用三 seed / dropout / heteroscedastic head 比较 uncertainty 与真实 absolute error 的相关、calibration curve 和高误差召回率 | 若 uncertainty 能召回 **>70%** 高误差区域，则并入 G5 和 H1；若校准差，只作为分析图不进主表 |
| **H9** | **隐式入口条件反演** | 入口流量/波形/相位若记录不全或和真实 CFD 边界不完全一致，网络可能缺关键条件。可从压力场、速度场或几何反推出 latent BC，再用 FiLM 调制 WSS head | 训练一个 latent BC probe，检查其是否解释 case-level residual、p95 误差或分域差异；和显式 Re/Wo 做对照 | 若 latent BC 明显降低病例级残差，升级为 G2/G3 的条件化模块；若只记忆病例 ID，关闭 |
| **H10** | **高 WSS 区域的弱监督检测头** | TODO-42 排序 loss 已 No-Go，但失败的是直接把排序 loss 混入主训练。可以改为冻结主干后训练一个区域检测头，专门服务 top-k / AUC，不干扰点级 WSS | 在冻结 4957/G1 特征上训练 high-WSS binary head，评估 top-k Dice/AUC，同时确认点级 WSS 不被重写 | 若区域指标显著提升且点级主输出不退化，作为 G5 论文叙事；不用于宣称点级精度突破 |
| **H11** | **流向-跨流向本征物理坐标系 (Streamwise-Cross-stream Frame)** | 全局 xyz 或几何局部分量缺乏物理意义；WSS 的绝大部分幅值在流向（低频），二次涡流在跨流向（高频）。拆分这两个方向在物理上是解耦的。 | 用中心线向量在壁面切平面的投影定义 $t_{stream}$，与之正交定义 $t_{cross}$，转换 GT 并评估预测这两者的 test R² | 若这两个物理分量比 x/y 更容易拟合，说明坐标系表示错位是主因，可全面替代现有的目标坐标系 |
| **H12** | **上游历史感受野注入 (Upstream History Injection)** | 二次流高度依赖上游几何历史；若 kNN 的球状感受野无法有效回溯足够远的上游中心线曲率，网络在信息论上就无法预测。 | 对高 WSS 点做 Gradient Attribution 检查感受野；若仅局限于局部，说明缺乏长程感知 | 若证实感受野不足，引入 Centerline-Attention，让壁面点 Query 上游的中心线 Token，作为并行宏观通道 |
| **H13** | **基于边界层动量积分的软约束 (Physics-Informed Soft Loss)** | 差分损失（TODO-30）易引起噪声放大；利用冯·卡门动量积分关系式（$WSS \\propto \\mu U_e / \\delta$）作为自洽正则化更稳健。 | 让网络同步输出 WSS、边界层厚度 $\\delta$ 和自由流速度 $U_e$，检查它们在已知 CFD case 中的物理一致性 | 若软约束稳定收敛，可替代直接差分，作为 G2 的物理损失扩展 |
| **H14** | **入口边界条件微扰导致的不确定性 (Irreducible Aleatoric Uncertainty)** | WSS_x/y 可能高度依赖入口流速剖面的微小分布；若仅有几何，横向分量在信息论上不可预测。这本身是个重大的科学发现。 | 在 CFD 中用同一几何微调入口速度分布，若 downstream x/y 剧变而 z 不变，证明其存在不可逆的不确定性 | 若证实，直接支持 TODO-61（异方差输出），并论证几何输入对二次流预测的理论极限，作为破局文章论点 |

### 14.1 我建议优先保留的五条

1. **H3 normal-ray 壁面管算子**：最贴近 `WSS=μ·∂u_t/∂n`，且能直接回答为什么 5276/5302 的无序差分失败；建议跟 G0-d / Q3 标签 QA 连在一起先做 oracle。
2. **H4 曲面谱基重建 oracle**：用于判断壁面 WSS 是否有低维曲面模式上限；由于跨病例谱漂移风险高，只建议先做 0 重训重建诊断，不直接写谱卷积或谱系数预测模型。
3. **H1 主动选数**：如果后续证明标签干净、表示也改了但仍卡住，最可能的真实瓶颈就是覆盖不足；主动选 CFD case 比随机扩池更符合 ROI。
4. **H11 流向-跨流向本征物理坐标系**：摒弃无物理意义的 xyz，回到流体力学本质。预测解耦的流向/跨流向分量比目前的 global xyz 甚至纯几何 local 都要更合理。
5. **H14 不确定性证明**：如果网络真学不到 `wss_x/y`，通过 CFD 扰动实验把它坐实为“信息论上的不可学”，这会是一篇非常有价值的流体力学/深度学习交叉论文，不再纠结于 R² 刷榜。

### 14.2 暂不建议马上投入的方向

- **H5 residual corrector**：只有主模型已经稳定后才有意义；现在太早做容易变成对 val residual 的后处理过拟合。
- **H6 时间建模**：取决于同 case 多时相数据是否足够整齐；若 `t_norm` 稀疏或相位口径不一致，工程量大于收益。
- **H9 latent BC**：有潜力，但最容易退化成 case ID 记忆；必须有严格的 case-level split 和反泄漏检查。
- **H10 弱监督检测头**：适合论文临床区域叙事，不适合作为点级 `wss_r2_wss` 突破主线。
