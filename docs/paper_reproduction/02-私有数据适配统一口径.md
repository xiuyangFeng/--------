# 私有数据适配统一口径

## 1. 统一目标

任何外部论文代码进入本项目私有数据集前，都必须先通过一个“中间数据层”，避免每篇论文各自读取 `processed/`、`predictions_test/` 或历史 run 文件，导致 split、单位、归一化和 mask 不一致。

## 2. 最小样本字段

建议每个 graph/frame 导出为统一记录：

| 字段 | 含义 | 必须性 |
| --- | --- | --- |
| `case_name` | 病例名 | 必须 |
| `domain` | AAA / AG / ILO 或现有域标签 | 必须 |
| `time_index` / `t_norm` | 时间步 | 必须 |
| `split` | train / val / test | 必须 |
| `points_xyz_norm` | 训练坐标，归一化空间 | 必须 |
| `points_xyz_mm` | 可视化与物理映射坐标，mm | 强推荐 |
| `wall_mask` | 壁面点 mask | 必须 |
| `near_wall_mask` | 近壁内部 mask | 强推荐 |
| `features` | `t, BC, Abscissa, NormRadius, Curvature, Tangent, is_wall` 等 | 按模型 |
| `targets_field` | `u,v,w,p` | 视任务 |
| `targets_wss` | `wss, wss_x, wss_y, wss_z` | WSS 任务必须 |
| `surface_mesh` | STL/VTK 面片路径或 surface graph | 2D/mesh/WSS 必须 |
| `norm_params_path` | 归一化参数 | 必须 |

## 3. 输出任务分层

| 层级 | 输出 | 适用 baseline |
| --- | --- | --- |
| A | `u/v/w/p` 全场或采样点流场 | PointNetCFD、PINGS、PINN 类 |
| B | 壁面 WSS 瞬态场 | E(3)-equivariant AAA WSS、Lannelongue GNN、V3P 类 |
| C | TAWSS / OSI / RRT | MultiViewUNet、任务 B 后处理 |
| D | 病例级/区域级 surrogate | G5 叙事、临床指标表 |

## 4. 不能混用的口径

- `V3P split_AG_v1` 与 `V3D split_data_new_v3` 禁止混表。
- 2D grid 指标不能替代 3D 壁面点指标。
- 归一化空间 RMSE 不能直接写成物理单位。
- 插值回面片后的云图不能替代同点逐点评估。
- 只预测速度/压力的论文不能被写成“解决 WSS baseline”，除非完整走通后处理 WSS 链并报告误差。

## 5. 建议的 baseline 统一指标

| 指标层 | 指标 |
| --- | --- |
| 点级流场 | `rmse_u/v/w/p`、`r2_u/v/w/p`、`r2_vel_mag` |
| 壁面 WSS | `rmse_wss`、`r2_wss`、`r2_wss_x/y/z`、向量角误差 |
| 近壁质量 | `near_wall rmse_vel_mag`、边界层速度剖面误差 |
| 周期指标 | TAWSS、OSI、RRT 的点级/区域级误差 |
| 病例级 | mean WSS、p95 WSS、Pa p95 Pearson/Spearman |
| 区域级 | high-WSS top-k Dice / IoU / AUC |
| 工程指标 | 推理时间、显存、是否依赖 mesh、是否可处理新拓扑 |

## 6. 第一轮适配建议

先不要把外部仓库直接改进主项目训练脚本。推荐流程：

1. 在只读方式下审计外部仓库输入格式。
2. 设计本项目统一导出格式。
3. 为每个外部 baseline 写独立 adapter。
4. 每次 adapter 只服务一个论文仓库。
5. 训练结果统一回收成项目内部 `baseline_results.csv/json`，再进入论文图表。
