# 外部论文梳理记录规范

> 上位文档：[paper_reproduction/README.md](README.md) · [01-复现优先级与适配策略](01-复现优先级与适配策略.md)  
> 关联：[外部baseline实验记录规范](../00-规范与记录/外部baseline实验记录规范.md)

## 1. 「梳理」指什么

老师要求的梳理 **不是** 在论文原作者 benchmark 上 1:1 数值复现，而是：

1. 该方法是否值得在本项目私有 AAA / WSS 数据上试；
2. 按计划实验矩阵跑完后，**在我们数据上** 能不能学到目标场、相对内部路线有无信号；
3. **为什么论文里精度高、换到我们数据集就不行**（任务、数据、特征、架构、指标口径差异）。

表述建议：**「PointNetCFD-style 适配 + ablation 梳理」**，勿写「已复现论文 Table X」。

## 2. 何时写梳理记录（重要）

| 动作 | 是否写 `梳理记录.md` |
| --- | --- |
| 单次 run 结束 | **否**（只更新 `experiments/<name>/实验分析记录.md` + `analysis_report.md`） |
| **一轮计划实验矩阵全部跑完** | **是**（写一篇批次梳理总结） |
| 中途只完成 1–2 组 | **否**（等矩阵齐再写） |

「一轮矩阵」= 该论文 README / 适配策略里预先定义的一批对照（如 PointNetCFD 四组：original / wall / geom_vp / geom_pwss）。其他论文矩阵规模可以不是 4 组，但 **必须事先在 `papers/<id>/README.md` 写清本轮矩阵包含哪些配置**。

## 3. 文档路径

每篇论文目录固定结构：

```text
docs/paper_reproduction/papers/<paper_id>/
├── README.md              ← 方法要点、适配路线、本轮实验矩阵定义
└── 梳理记录.md            ← 每完成一轮矩阵写/追加一个批次节（勿每 run 一条）
```

复制模板：[`papers/_template/梳理记录.md`](papers/_template/梳理记录.md)

## 4. 与实验记录三层的关系

| 层级 | 路径 | 粒度 |
| --- | --- | --- |
| 单次 run | `outputs/.../<run_dir>/analysis_report.md` | 每个 Job |
| 实验族 | `external_baselines/<baseline>/experiments/<name>/实验分析记录.md` | 每个配置族 |
| **梳理（本文档）** | `docs/paper_reproduction/papers/<id>/梳理记录.md` | **每轮矩阵 1 次** |
| 总账 | `docs/02-推进与变更/代码修改与实验推进记录.md` 文首 | 矩阵完成 + 梳理写完时 1 条 |

**顺序**：矩阵跑齐 → 写/更新 `梳理记录.md` → 推进记录文首 1 条（含梳理链接）。

## 5. `梳理记录.md` 必填结构

每个 **批次** 使用二级标题 `## 批次 YYYY-MM-DD（可选：Job xxx）`，节内按序填写：

1. **梳理目标**：本轮要回答的 1–2 个问题  
2. **是否严格论文复现**：否 + 与原文差异（数据 / 代码 / 输入输出 / 指标）  
3. **本轮实验矩阵**：表（配置名 · 输入 · 输出 · 目的）  
4. **汇总结果**：test 关键指标（注明归一化或物理单位）  
5. **在我们数据上为什么行 / 不行**：分点解释 gap（勿只堆数字）  
6. **与内部路线关系**：是否与 V3P/V3D 同表；若对照，写清禁止混表口径  
7. **Go / No-Go**：是否继续投入、作为论文哪类 baseline  
8. **下一步**：≤3 条  
9. **详细指标链接**：`external_baselines/.../experiments/` · 各 `run_dir`

## 6. 新论文接入 checklist

- [ ] `papers/<id>/README.md` 写清 **本轮计划矩阵**（几组、各组目的）  
- [ ] `external_baselines/<id>/` 代码与配置就绪  
- [ ] 矩阵全部跑完  
- [ ] 填写 `papers/<id>/梳理记录.md`（从 `_template` 复制）  
- [ ] `docs/02-推进与变更/代码修改与实验推进记录.md` 文首 1 条 + 梳理链接  
- [ ] （可选）更新 `00-文献筛选总表.md` 该论文「适配状态」列  

## 7. 常见错误

1. 每跑一个 Job 就改 `梳理记录.md` → 应等 **整轮矩阵** 完成再写。  
2. 把梳理写成严格复现结论 → 应写 **适配判读** 与 **gap 原因**。  
3. 梳理里贴全量病例表 → 细节放 `experiments/`，梳理只保留 **汇总 + 判读**。  
4. 与 V3P AsymW-a 等同表且不注明 split / 任务差异。
