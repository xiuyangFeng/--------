## 批次 2026-06-15（Job 5610–5613）

### 1. 梳理目标

- 在不使用图结构（忽略 `edge_index`）时，PointNetCFD-style 点云方法能否在本项目 **AG · split_AG_v1** 上学到可用的 **u/v/w/p**？
- 加 **is_wall / 显式几何** 能否解释「论文思路在我们数据上变弱」的问题？
- **p + WSS 直预测** 是否值得作为 WSS 外部 baseline？

### 2. 是否严格论文复现

- **否**（方法适配 + ablation 梳理，非原文 benchmark 数值复现）
- 差异摘要：
  - **数据**：私有 AAA `data_new/AG`，非论文公开算例
  - **代码**：自写 PyG adapter + 简化 MLP（非官方 repo 直跑）
  - **输入**：除 paper-close 的 `x,y,z,t,BC` 外，另有 wall / 几何 / WSS 扩展组
  - **指标**：z-score 归一化空间 MSE / RMSE / MAE / R²

### 5. 在我们数据上为什么行 / 不行

1. **压力能学、速度难学（original）**：AAA 不规则几何 + 仅坐标 + global max pooling，难以表达近壁边界层；与论文在相对规则 CFD 几何上的设定不对等，**不是单纯代码跑错**。
2. **wall / 几何单调改善速度**：说明 gap 主要来自 **任务需要的信息量**，方法核心并非完全无效；geom_vp 相对 original RMSE 约 **−18.6%**。
3. **病例 OOD 一致**：GONG_HUI_XIA 各组最好，GUO_XI_JIANG / ZHANG_JUN_HUA 最差，与 V3 数据难度观察同向。
4. **WSS 直预测不行**：PointNetCFD 设计目标是 **体场 u/v/w/p**，非壁面矢量 WSS；geom_pwss epoch1 即最优，WSS 轴向 R²≈0，**不能**替代 V3P WSS 主线（AsymW-a wss_r2≈0.399 **禁止同表混列**）。

### 6. 与内部路线关系

- 与 V3P / V3D：**分表**；仅作外部「不建图点云」对照。
- **geom_vp** 可作为论文 **VP sanity baseline**；**不可**作为 WSS SOTA 对照。

### 7. Go / No-Go

| 方向 | 结论 |
| --- | --- |
| 论文 VP 外部 baseline（geom_vp） | **Go** |
| paper-close original 继续投入 | **No-Go**（仅保留下界对照） |
| geom_pwss / WSS 直预测 | **No-Go** |
| 继续 PointNet 家族 | **No-Go** 作为主路线；可转向 mesh / 等变 / 2D 壁面 P0 |
