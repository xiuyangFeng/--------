# 点云预测值与真值回插到面片方法

> 建档日期：2026-06-23  
> 定位：只讨论 **GNN/CFD 点云标量如何映射、插值、采样回 STL/VTK 面片，并用于病例级可视化与同面片对比**。本文不讨论模型结构优化，也不把插值后的图当作训练方法改进。

## 1. 要解决的问题

本项目的预测结果通常是点云：

```text
x, y, z, is_wall,
u_cfd, v_cfd, w_cfd, p_cfd, wss_cfd,
u_pred, v_pred, w_pred, p_pred, wss_pred,
err_p, err_wss, ...
```

老师关心的是：怎样把这些点云上的 `wss_pred / wss_cfd / err_wss` 或 `p_pred / p_cfd / err_p` 画回病例的 STL/面片上，让单个病例的预测分布能被看清楚。

这里必须区分三件事：

| 目的 | 推荐口径 | 说明 |
| --- | --- | --- |
| 数值指标 | 原始同点点云上算 | 最公平，不引入插值误差 |
| 病例级可视化 | 点云标量插值到 STL/VTK 面片 | 适合论文图、汇报图、老师肉眼检查 |
| CFD 原生对照 | Fluent/CFD-Post 从 `.cas + .dat` 画 | 最接近工程 CFD 习惯，但需要原始解文件 |

**核心原则**：真值和预测如果都来自同一个 CSV 点云，应使用同一个目标面片、同一种插值方法、同一组参数回插；如果 CFD 真值来自 Fluent 原生网格，则图中必须说明它与 GNN 插值面片不是同一离散口径。

### 1.1 展示时间帧怎么选（merged-1146）

病例级云图需要 **空间梯度足够** 的时刻；不宜默认用周期端点帧。

| 帧 ID | Fluent step | `t_norm` | 周期内位置 | 医学 / 脉搏相位（近似） | 本项目用途 |
| --- | ---: | ---: | --- | --- | --- |
| `merged-1120` | 1120 | 0.00 | 第 1/81 帧 · 周期端点 | 舒张末期 / 低脉动 | ❌ 单帧 R² 易失真，云图梯度弱 |
| **`merged-1146`** | **1146** | **~0.16** | **第 14/81 帧** | **收缩期上升段 / 射血早期** | ✅ **V3P/CROWN 汇报主展示帧** |
| `merged-1200` | 1200 | 0.50 | 第 41/81 帧 | 周期中段 | 可选对照，非默认 |
| `merged-1280` | 1280 | 1.00 | 第 81/81 帧 · 周期端点 | 与 1120 首尾相接 | 同 1120，不作主图 |

- **时间定义**：`t_norm = (step − 1120) / (1280 − 1120)`（见 `pipeline/convert_to_graph.py`）；81 偶数步覆盖 **一个心动周期**（任务 E 文档 §2.1）。
- **图注必写**：`merged-1146 · t_norm≈0.16 · 收缩期上升段（近似）`；**数值指标**仍优先 **81 帧 pooled**，不单报 1146 单帧 R²。
- **交付索引**：[`outputs/field/postview/README.md`](../../outputs/field/postview/README.md)
- **落盘规范**：[`docs/00-规范与记录/点云回插面片可视化目录说明.md`](../00-规范与记录/点云回插面片可视化目录说明.md)
- **Agent 技能**：[`.cursor/skills/postview-surface-viz/SKILL.md`](../../.cursor/skills/postview-surface-viz/SKILL.md)（批量出图 / ParaView 包时优先读）

## 2. 方法选择总览

| 方法 | 输入 | 输出 | 优点 | 风险 | 本项目建议 |
| --- | --- | --- | --- | --- | --- |
| 最近邻映射 | 点云 CSV + STL 顶点 | STL 顶点标量 | 简单、可解释、速度快 | 块状、锯齿，受采样密度影响 | debug / 快速预览 |
| k 近邻 IDW | 点云 CSV + STL 顶点 | 平滑面片标量 | 可控、好实现、比最近邻平滑 | 会平滑 hotspot | 推荐作为主力自实现方法 |
| Gaussian kernel | 点云 CSV + STL/VTK mesh | 平滑面片标量 | VTK/PyVista 原生支持，参数可记录 | 半径不当会过度平滑或空洞 | 推荐用于论文/汇报主图 |
| Shepard / RBF | 点云 CSV + STL/切面 | 平滑标量场 | 数学上更平滑 | 大点数内存重，外推风险 | 只在局部 patch 或低点数尝试 |
| `sample()` / Probe | 带 cell topology 的 CFD/VTK mesh + 目标 mesh | 目标 mesh 上的场 | 利用单元内插值，工程上更正统 | 需要源数据有网格单元 | 有 `.vtu/.vtk/.cas+.dat` 时优先 |
| ParaView `Resample With Dataset` | source 数据 + destination geometry | 目标几何上 point data | 图形界面友好，可批量脚本化 | source 若只是点云时需要配合点插值器 | 用于最终出图/复核 |
| CloudCompare scalar field 插值 | 两个点云或 mesh 顶点 | destination scalar field | GUI 简单，适合人工核验 | 自动化和论文记录弱 | 辅助检查，不作主链 |

## 3. 本项目最推荐的链路

### 3.1 壁面 WSS / TAWSS

推荐链路：

```text
预测 .pt
  -> export_for_cfdpost.py
  -> __wall.csv
  -> 将 wss_pred / wss_cfd / err_wss 插值到同一个 STL 顶点
  -> 输出 VTP
  -> ParaView / CFD-Post 统一色标截图
```

当前已有入口：

- `tools/cfdpost_cloud_export/export_for_cfdpost.py`
- `tools/cfdpost_cloud_export/map_to_stl_surface.py`
- `tools/cfdpost_cloud_export/export_pointcloud_vtp.py`
- `tools/cfdpost_cloud_export/run_all_routes.sh`

现有 `map_to_stl_surface.py` 做的是 **最近邻映射**。它适合第一版图和排查坐标是否对齐，但下一步如果要画得更像 CFD 面片云图，建议扩展成 `nearest / idw / gaussian` 三种方法。

### 3.2 压力 p

压力可以有两种图：

| 图 | 推荐目标 | 说明 |
| --- | --- | --- |
| 壁面压力 | STL / wall VTP | 与 WSS 一样插值到壁面 |
| 腔内压力切面 | 体网格或规则切面 | 若只有点云，用局部 RBF / Delaunay / IDW；若有 CFD 体网格，用 Probe/sample |

注意压力有参考零点问题。如果为了病例级对比做了 per-case offset correction，必须在图注或指标表里写清楚。

### 3.3 真值和预测一起回插

如果 CSV 里同时有 `wss_cfd` 和 `wss_pred`，不要只回插预测。推荐一次性回插：

```text
wss_cfd
wss_pred
err_wss = wss_pred - wss_cfd
abs_err_wss
p_cfd
p_pred
err_p
```

这样同一个 `*.vtp` 里就可以切换变量，保证 CFD 点云真值和 GNN 预测用了同一目标 STL 和同一插值参数。

## 4. 插值方法细节

### 4.1 最近邻映射

对每个 STL 顶点 `q`，找到最近的点云点 `p_i`：

```text
f(q) = f(p_i),  i = argmin ||q - p_i||
```

建议记录：

- `map_dist`
- `map_valid`
- `max_dist`
- `valid_ratio`
- `p50 / p90 / p95 / max` 距离

适用情况：

- 快速确认点云和 STL 坐标系是否一致；
- 预测点和 STL 顶点密度接近；
- 不需要很平滑的展示图。

不适合：

- 稀疏点云；
- high-WSS hotspot 需要平滑边界；
- 老师要求“像 CFD 云图”的展示。

### 4.2 k 近邻反距离加权

对每个 STL 顶点 `q`，找最近 `k` 个点云点：

```text
d_i = ||q - p_i||
w_i = 1 / (d_i + eps)^power
f(q) = sum(w_i * f(p_i)) / sum(w_i)
```

推荐默认参数：

```text
k = 8 或 12
power = 2
max_dist = 2~5 mm
eps = 1e-8
```

如果 `k` 个邻居里有点超过 `max_dist`，可选择：

- 丢弃超半径邻居；
- 若半径内无点，标为 NaN；
- 或 fallback 到最近邻，但必须在报告里记录。

优点是可解释、可复现、实现简单。缺点是没有利用血管表面拓扑，欧氏距离可能跨过薄壁、分叉或相邻管腔。

### 4.3 Gaussian kernel / VTK PointInterpolator

VTK `vtkPointInterpolator` 的思路是：用目标几何的点作为查询点，从 source 点云中找邻域点，再用指定 kernel 计算新值。VTK 支持 Voronoi、Gaussian、Shepard、SPH 等 kernel，并可以按半径或最近点数找邻域。

PyVista 的 `mesh.interpolate(point_cloud, radius=..., sharpness=...)` 是对这类点云插值的高层封装。PyVista 文档说明 `interpolate()` 使用 Gaussian interpolation kernel，并用 `radius` 和 `sharpness` 控制核；如果源数据有 CFD/FEM cell topology，文档也明确建议优先用 `sample()`。

推荐默认参数：

```text
radius = 3.0 mm
sharpness = 2.0~4.0
strategy = mask_points 或 closest_point
```

解释：

- `radius` 太小：面片出现空洞；
- `radius` 太大：局部高 WSS hotspot 被抹平；
- `sharpness` 越大，远点影响越小；
- `mask_points` 适合严谨报告覆盖率；
- `closest_point` 适合想避免空洞的展示图，但必须说明 fallback。

### 4.4 RBFInterpolator

SciPy `RBFInterpolator` 适合 scattered data 的平滑插值，但官方文档提醒：全局 RBF 的内存随点数近似二次增长，超过约千点就可能不现实；可以设置 `neighbors` 用局部近邻 RBF。

本项目壁面点通常上万，RBF 不建议作为全局默认。它更适合：

- 局部小 patch；
- 规则切面；
- 需要比较平滑的少量点验证图。

### 4.5 `sample()` / Probe / Resample With Dataset

如果 source 不是单纯点云，而是带单元拓扑的 CFD mesh，例如 `.vtu/.vtk/.cas+.dat` 导出的体网格或面网格，应优先使用：

- PyVista `sample()`
- VTK `vtkProbeFilter`
- ParaView `Resample With Dataset`

原因是这类方法会在 source mesh 的 cell 里找查询点，并用单元形函数/插值函数计算点值，比“离散点云距离加权”更符合 CFD 网格数据的含义。

但如果 source 只有 CSV 点云，没有 cell topology，就不能假装它是 CFD 网格；应使用 PointInterpolator / IDW / Gaussian。

## 6. 病例级图怎么排

### 6.1 WSS 主图

推荐三列：

| CFD 真值 | GNN 预测 | 误差 |
| --- | --- | --- |
| `wss_cfd` 或 Fluent 原生 WSS | `wss_pred` 回插到 STL | `err_wss` 或 `abs_err_wss` |

要求：

- CFD 真值和 GNN 预测两列使用同一色标范围；
- 误差图使用单独的发散色标或绝对误差色标；
- 图注写清楚插值方法和参数，如 `Gaussian radius=3 mm, sharpness=2, valid=98.9%`；
- 图注写清楚 **时间帧与周期相位**，如 `merged-1146, t_norm≈0.16, early systole rising phase (approx.)`；
- 不要只展示预测，不展示误差；
- 对 high-WSS 区域可加局部放大图。

### 6.2 压力主图

推荐三列：

| CFD 真值 | GNN 预测 | 误差 |
| --- | --- | --- |
| `p_cfd` | `p_pred` | `err_p` |

要求：

- 如果做了压力零点校正，必须写 `per-case offset corrected`；
- 颜色范围可按病例内共同 min/max，也可按全 test 统一范围；不要 CFD 和 Pred 自动各自缩放。

### 6.3 点云图作为补充

点云直接 VTP 图仍然有价值，尤其用于：

- 检查预测点是否覆盖壁面；
- 看 `err_wss` 是否集中在分叉、入口、出口或瘤颈；
- 和 STL 插值图对照，判断插值是否制造了假平滑。

但论文/汇报主图最好用面片云图，因为老师更容易按 CFD 习惯读图。

| 工具 | 本项目定位 | 使用建议 |

```text
批量生产：Python/PyVista/VTK
最终出图：ParaView 或 CFD-Post
原生 CFD 真值：Fluent/CFD-Post
人工检查：CloudCompare
不推荐主链：MATLAB
```

## 8. 本项目建议实现的脚本接口

建议在现有 `map_to_stl_surface.py` 基础上扩展：

```bash
python tools/cfdpost_cloud_export/map_to_stl_surface.py \
  --csv tools/cfdpost_cloud_export/output/slow__GUO_XI_JIANG__result_features_merged-1146__wall.csv \
  --stl data_new/AG/slow/GUO_XI_JIANG/GUO_XI_JIANG.stl \
  --method gaussian \
  --radius 3.0 \
  --sharpness 2.0 \
  --fallback mask \
  --scalars wss_cfd,wss_pred,err_wss,p_cfd,p_pred,err_p \
  --output-dir tools/cfdpost_cloud_export/output/route_interp
```

建议支持参数：

| 参数 | 说明 |
| --- | --- |
| `--method nearest` | 当前已有最近邻 |
| `--method idw` | kNN 反距离加权 |
| `--method gaussian` | Gaussian kernel，接近 PyVista/VTK |
| `--k` | IDW 或 kNN kernel 邻居数 |
| `--radius` | 半径内邻居，单位与坐标一致 |
| `--power` | IDW 幂次，默认 2 |
| `--sharpness` | Gaussian 衰减 |
| `--fallback mask/nearest/null` | 无邻居点处理策略 |
| `--report-json` | 输出覆盖率和距离统计 |

## 9. 质量控制清单

每张病例级图出图前检查：

- [ ] STL 与 CSV 坐标单位一致；
- [ ] 点云和 STL 没有明显错位、旋转、镜像；
- [ ] `valid_ratio` 足够高，建议大于 95%；
- [ ] `map_dist p95` 小于血管局部半径的一小部分；
- [ ] CFD 真值和预测用了同一插值方法；
- [ ] CFD/Pred 色标一致；
- [ ] 图注写明 `method / radius / k / sharpness / coverage`；
- [ ] 主指标仍在原始同点 CSV 上计算；
- [ ] 误差图没有被二次插值后再和一次插值图混比；
- [ ] 分叉、薄壁、邻近管段处检查是否发生欧氏近邻跨面。

## 11. 一句话给老师

可以这样解释：

> 我们的数值指标在原始同点点云上计算，避免插值影响误差；病例级云图则把 `wss_cfd/wss_pred/err_wss` 用同一套局部核插值映射到同一个 STL 面片顶点，并记录半径、邻居数、覆盖率和映射距离。这样既保证指标公平，也能得到接近 CFD-Post 风格的面片云图。
