- val `r2_vel_mag` ≈ **0.586**
- 对照 `A-Opt-05` seed1 同 epoch：val `r2_vel_mag` ≈ 0.467，val `rmse_vel_mag` ≈ 0.750（5540 的 RMSE 仍更差）

### 2.2 报错（完整栈顶）

```text
File "training/core/losses.py", line 512, in _subsample_physics_batch
    return batch[sel]
...
IndexError: index 7 is out of bounds for dimension 0 with size 2
```

触发链：`train_field.py` → `trainer.fit` → `run_epoch` → `PhysicsConstraintLoss.build_loss` → `_subsample_physics_batch`（当 `epoch > warmup_epochs` 且图节点数 `N > max_physics_nodes`）。

### 2.3 根因

文件：`training/core/losses.py`

函数 `_subsample_physics_batch`（约 L488–512）：

```python
sel = ...  # shape [K]，K 为节点下标，0..N-1
return batch[sel]   # ❌ 错误
```

在 **PyTorch Geometric `Batch`** 上，`batch[idx]` 的 `idx` 表示 **batch 内第几张图（graph index）**，不是 **节点 index**。
本任务 `data.batch_size=1`，但单图节点数常 **>2048**，`sel` 里会出现 7、100、500 等节点号，被误当成「第 7 张图」→ `IndexError`。

### 2.4 连带设计缺陷（修 P0 时一并处理）

同文件 `PhysicsConstraintLoss.build_loss`（约 L633–651）：

1. 先 `physics_batch = _subsample_physics_batch(...)` 缩小节点；
2. 随后 `base_x = batch.x.detach()` 用的是 **未子采样的原 batch**；
3. `physics_batch.x = torch.cat(cols, ...)` **覆盖**为全图节点数。

结果：即使子采样函数写对，**physics 前向仍对全图计算**，`max_physics_nodes=2048` **起不到省显存作用**，大图上 ep21 仍可能 **OOM**（5536–5538 旧作业即 ep21 OOM）。

**正确顺序**：

1. 节点级子采样得到 `physics_batch`（节点数 ≤ `max_physics_nodes`）；
2. `base_x = physics_batch.x.detach()`（子采样后的 x）；
3. 在子采样后的坐标上建 `requires_grad` 叶子，再赋回 `physics_batch.x`。

### 2.5 推荐修法（实现要点）

**A. 节点子采样（替换 `return batch[sel]`）**

任选其一（需与当前 PyG 版本一致）：

- `batch.subgraph(subset_mask)` / `batch.subgraph(sel)`（若 API 可用）；
- 或手动对 **节点维张量** 索引：`x`, `y`, `y_wss`, `edge_index`, `pos` 等，并重映射 `edge_index`（仅保留两端都在 `sel` 的边）；
- **不要**使用 `batch[integer_tensor_of_node_ids]`。

约束：

- 配置已设 `data.batch_size=1`；若 `batch.batch` 存在，子图后应仍为单图或显式设 `batch.batch = zeros(N_sub)`。
- 优先保留壁面点逻辑（现有 wall / interior 采样策略可保留）。

**B. 修正 `build_loss` 坐标叶子来源**

```python
physics_batch = _subsample_physics_batch(batch.clone(), self._max_physics_nodes, self.is_wall_idx)
base_x = physics_batch.x.detach()   # 必须是子采样后的 x，不是 batch.x
# 后续 coord_leaf / physics_batch.x 重建逻辑不变，但维度与 physics_batch 节点数一致
```

**C. 配置（勿改除非显存仍不够）**

| 字段 | 当前值 | 文件 |
| --- | --- | --- |
| `data.batch_size` | `1` | `A-PINN-01-cont_seed1.json` 等 |
| `physics.max_physics_nodes` | `2048` | 同上 + `training/core/config.py` 默认值 |
| `physics.warmup_epochs` | `20` | 前 20 ep 不启 physics |

1. **单元/烟测（推荐）**：构造小型 fake `Data`/`Batch`（如 N=5000 节点，`max_physics_nodes=2048`），在 `epoch=21` 调用 `build_loss`，断言不抛 `IndexError`，且 `physics_batch.x.size(0) <= 2048`。
