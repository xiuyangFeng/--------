## 1. 一句话结论

这篇论文和我们当前任务A最相关的地方，不是“脑动脉瘤分类”，而是：**在病例数很少的脑血管 CFD 数据上，用点云网络 + 物理方程约束 + 点云重采样，快速预测三维速度场和压力场**。

它和我们当前工作高度相似：

- 都是患者特异性血管几何；
- 都是 CFD 结果作为监督；
- 都希望用深度学习替代耗时 CFD；
- 都用点云/三维空间点作为主要数据表达；
- 都关心脑血管或血流动力学参数；
- 都面临小样本、几何差异大、边界附近物理量难学的问题。

但它和我们当前工作的关键差异也很大：

- 论文主要预测 **速度三分量 `Vx/Vy/Vz` 和压力 `P*`*；
- 我们当前最难的是 **壁面 WSS，尤其 `wss_x/y` 或横向/周向分量**；
- 论文没有把 WSS 作为主要输出指标，而是用速度/压力进一步计算 FR、PD 等临床指标；
- 论文的高精度 NMAE 不能直接等价为“WSS 点级 R² 已经能做好”。

因此，这篇论文最适合给我们提供三类启发：**有限数据下的物理约束、点云预处理、临床指标叙事**；但不能直接证明我们当前 WSS 精度瓶颈已经被别人解决。

## 2. 论文做了什么

### 2.1 数据对象

论文使用 51 个右侧颈内动脉 ICA 的患者病例，其中 49 例有脑动脉瘤，2 例无动脉瘤。研究区域是 ICA C3-C7 段，重点包括 carotid siphon 及其附近复杂弯曲区域。

训练/测试划分：

- 40 例训练；
- 11 例测试；
- 另外做了 4 组随机划分的 Monte Carlo cross-validation。

CFD 设置：

- 稳态不可压 Newtonian 流体；
- 控制方程是 Navier-Stokes；
- 速度入口，入口 WSS 调整到 1.5 Pa；
- 出口流量按出口面积平方律分配；
- 使用 ANSYS Fluent；
- 每例约 50 万 mesh elements；
- 壁面 no-slip；
- 血液密度 1050 kg/m3，黏度 0.004 Pa s。

### 2.2 输入与输出

输入：

- 每个点的三维坐标 `(x, y, z)`；
- 论文主要强调“坐标到流场”的映射。

输出：

- `Vx`
- `Vy`
- `Vz`
- `P`

注意：论文没有把 WSS 作为网络的直接输出。

### 2.3 点云预处理

论文认为 CFD mesh 点在壁面附近密度很高，空间分布不均匀，这会影响点云网络训练。因此它做了：

- voxelization，体素大小 0.05 mm；
- 在体素内随机生成新点；
- 用同一体素内原始点按距离加权插值出 `Vx/Vy/Vz/P`；
- 每个病例重采样到 10,000 个点；
- 用 uniform loss 衡量点云均匀性；
- uniform loss 从 26366 降到 606，论文称均匀性改善 97.70%；
- 用 Hausdorff distance 和速度分布可视化说明重采样没有明显改变流场分布。

这一点和我们当前工作相关：我们也一直在处理点云/图采样、壁面点、近壁特征、重图和归一化问题。论文给出的启发是：**点云分布本身可能是一个可控变量，不只是网络结构问题**。

### 2.4 网络结构

网络基于 PointNet 思路，包含三部分：

1. Module 1：用 shared Conv1d 提取局部/点级特征，再用 max pooling 得到全局特征；
2. Module 2：拼接局部特征和全局特征，输出 `Vx/Vy/Vz/P`；
3. Module 3：PINNs 物理约束模块，用自动微分计算 Navier-Stokes 方程残差，并加入壁面 no-slip 边界条件。

总 loss：

```text
loss = loss_data + omega * loss_phy
```

其中：

- `loss_data` 是预测速度/压力和 CFD 真值之间的监督误差；
- `loss_phy` 包括连续性方程、动量方程残差、壁面 no-slip 条件；
- `omega` 每 10 epoch 更新一次，按 data loss 和 physics loss 的比例动态调整。

训练设置：

- PyTorch 2.3.0；
- Python 3.8.19；
- NVIDIA Tesla P100 12GB；
- batch size = 1；
- epoch = 15000；
- Adam 初始学习率 0.003；
- ReduceLROnPlateau；
- dropout = 0.5。

## 3. 论文主要结果

论文设计了四组对照：


| 组别           | 点云预处理 | PINNs |
| ------------ | ----- | ----- |
| SOTA model 1 | 无     | 无     |
| SOTA model 2 | 无     | 有     |
| 本文消融 1       | 有     | 无     |
| 本文主模型        | 有     | 有     |


### 3.1 压力预测

论文表 1 报告的 pressure NMAE：


| 模型           | Pressure NMAE mean |
| ------------ | ------------------ |
| 无预处理、无 PINNs | 12.55 ± 4.91%      |
| 无预处理、有 PINNs | 10.05 ± 5.03%      |
| 有预处理、无 PINNs | 15.16 ± 7.20%      |
| 有预处理、有 PINNs | **6.63 ± 2.80%**   |


观察：

- 单独点云预处理反而让压力变差；
- PINNs 对压力帮助明显；
- 点云预处理和 PINNs 组合后提升最大。

### 3.2 速度预测

论文表 2 报告的 velocity NMAE：


| 模型           | Velocity NMAE mean |
| ------------ | ------------------ |
| 无预处理、无 PINNs | 11.15 ± 2.78%      |
| 无预处理、有 PINNs | 9.90 ± 2.82%       |
| 有预处理、无 PINNs | 12.30 ± 3.79%      |
| 有预处理、有 PINNs | **7.79 ± 2.14%**   |


观察：

- 速度也同样是“预处理 + PINNs”组合最好；
- 仅预处理不够，甚至可能因为数据点减少/插值损失导致纯数据驱动性能下降；
- 物理约束是这篇论文的关键增益来源。

### 3.3 临床指标

论文还从预测的速度/压力中计算：

- FR：aneurysm neck inflow rate 与 aneurysm inlet flow rate 的比值；
- PD：aneurysm neck 与 inlet 压力差除以入口动态压。

结果：

- FR：`R2 = 0.9624, P < 0.001`
- PD：`R2 = 0.9355, P < 0.001`

但论文也承认：

- FR 最大偏差约 20%；
- PD 最大偏差约 60%；
- 目前没有对应的破裂/复发标签，因此还不能直接证明临床分类效果。

这对我们很重要：**即使点级指标不是完美，也可以通过病例级/区域级临床 surrogate 指标形成叙事**。这和我们当前 G5 病例级 Pa p95、区域 top-k 等思路是一致的。

## 4. 我不完全理解或需要追问作者/老师的点

### 4.1 论文没有直接预测 WSS，为什么摘要和引言反复强调 WSS？

论文背景中多次强调 WSS 对动脉瘤形成、破裂和治疗很重要，但模型输出是 `Vx/Vy/Vz/P`。WSS 只是 CFD mesh 设计和医学意义中的背景变量，并没有作为主要预测指标报告。

可以追问：

- 作者有没有从预测速度场进一步计算 WSS？
- 如果计算了 WSS，误差是多少？
- 壁面速度梯度对点云重采样和插值是否敏感？
- 论文的速度 NMAE 低，是否能保证 WSS 也低？

这对我们很关键，因为我们现在最难的正是 WSS，而不是压力或速度低频主场。

### 4.2 PINNs 的物理残差是在所有点上算，还是只在采样后的 10,000 点上算？

论文说 Module 3 使用预测的 `Vx/Vy/Vz/P` 和坐标，通过自动微分计算 N-S 方程残差。但如果只在重采样后的 10,000 点上计算，那么物理约束的空间分辨率受采样影响。

可以追问：

- 物理残差点是否就是训练点？
- 壁面 no-slip 点如何从重采样点中确定？
- voxel 内随机点如果落在壁面附近，如何保证 wall point mask 正确？
- 对 near-wall gradient 和 WSS 是否有额外处理？

### 4.3 点云预处理为什么单独使用会变差？

论文解释为数据缩减和纯数据驱动不稳定。但这背后可能还有几个问题：

- 10,000 点是否损失了壁面边界层高梯度信息？
- 距离加权插值是否平滑了局部极值？
- 对速度/压力还可以接受，但对 WSS 这种梯度量可能更危险；
- 如果目标是 WSS，是否应该保留更多壁面/近壁点，而不是追求全域均匀？

这和我们 G4/近壁壳层/壁面展开非常相关。

### 4.4 论文用 NMAE，不用 R²，会不会掩盖局部高误差？

论文不用 MSE，理由是壁面点速度为 0 会影响误差评价。NMAE 更容易描述整体误差，但可能掩盖：

- 高 WSS 区域；
- 分叉附近；
- aneurysm neck；
- outlet；
- Pcom 小直径区域；
- 二次流/横向速度分量。

可以追问：

- 有没有 per-component R²？
- 有没有高速度/高压力梯度区域误差？
- 有没有壁面附近单独误差？
- 有没有 aneurysm neck 局部区域误差？

### 4.5 入口边界条件比较经验化，泛化到别的中心是否稳定？

论文入口速度被调整到入口 WSS = 1.5 Pa，出口按面积平方律分配。这是常见 CFD 设定，但不是患者特异性流量。

可以追问：

- 如果有 Doppler/4D-flow MRI 的患者特异入口流量，模型是否还能泛化？
- 模型学到的是几何到“标准化 CFD 场”的映射，还是几何到真实生理场的映射？
- 多中心数据时，BC protocol 不一致会不会导致泛化失败？

论文自己也把 multi-center 和 patient-specific BC 列为未来工作。

### 4.6 训练 15000 epoch 且 batch size = 1，是否容易过拟合？

论文说 loss 约 10000 epoch 后稳定到 0.015，但没有看到太多关于：

- best checkpoint 如何选择；
- val/test 是否严格隔离；
- 多 seed 或多 split 方差；
- 是否存在 early peak；
- 训练后期是否对压力/速度任务有不同影响。

这和我们当前观察到的 `best_wss` 很早达到峰值、后期退化的问题有关。

### 4.8 论文/答辩里的「好看云图」怎么画？1 万采样点如何和 Fluent 全网格真值对比？

**背景矛盾**：论文每例 CFD 约 50 万 mesh elements，但预处理后只保留 **10,000 个均匀点** 用于训练/测试；深度学习预测也只在这 1 万个坐标上有值。而答辩或论文里常见的 CFD 云图，往往是 Fluent/CFD-Post 在**完整壁面/体网格**上做的光滑 Contour。两者分辨率不一致，对比时到底怎么做？


| 环节                    | 做法                                                                     | 是否插值                               |
| --------------------- | ---------------------------------------------------------------------- | ---------------------------------- |
| **数值指标（NMAE 等）**      | 在**同一套 10,000 点**上算：真值标签在预处理阶段已由体素内**距离加权插值**从原始 CFD 点得到；网络预测也在这些坐标上输出 | 真值侧已在预处理时插值；指标是**同点逐点**对比，不需要回到全网格 |
| **CFD 参考云图（「正统」真值图）** | Fluent 读 `.cas` + `.dat`，在**原生壁面/体网格**上 Contour                        | 不经过 1 万点；展示的是**全网格 CFD 解**         |
| **GNN 预测云图（要「好看」）**   | 1 万离散点本身不够光滑，通常需要**再映射/插值到连续表面**后再 Contour                             | **需要插值**                           |


**论文未写清、建议追问老师/作者：**

- 正文 Fig. 里的 CFD 参考图，是**原生 Fluent 全网格 Contour**，还是也画在 10,000 重采样点上？
- GNN 预测图是**点云直接画**，还是**插值回壁面/体网格**后再 Contour？若插值，用的是 Fluent 内置插值、VTK/ParaView，还是其他？
- 若左全网格、右 1 万点插值并排，作者是否检查过**插值平滑**对视觉「显得更像」的影响？是否提供过**同点集**点云对比作为补充？
- 体腔内速度/压力截面图：截面是在 Fluent 原生网格上取，还是把 1 万点预测**插值到体网格**后再切面？
- 对 WSS（若后处理计算）：1 万点上的速度梯度 + 插值到面片，误差会不会比速度/压力更敏感？

### 4.7 临床终点是 WSS，但均匀化是否损害未来实际应用？

我们最终关心的是临床血流动力学指标（WSS、高剪切区域、病例级风险 surrogate），而不是单纯的速度/压力场重建。论文为改善训练稳定性，把原本壁面/近壁过密的 CFD 点均匀化到 10,000 点，主输出仍是 `Vx/Vy/Vz/P`，WSS 未直接预测，临床叙事走 FR/PD。这会引出一个对我们很关键的应用链问题：

**均匀化 → 只预测 v/p → 临床需要 WSS 时再数值求梯度**，这条部署路径是否可靠？

可以追问：

- 均匀化后，壁面点与近壁点各占多少？内部点占比是否显著上升？高 WSS 区域（瘤颈、分叉、小直径段）的点密度是否被稀释？
- 距离加权插值是否会系统性**平滑 WSS 峰值**（p95/max）和**热点定位**？有没有单独评估过 top-k 高 WSS 区域保真？
- 若只预测速度/压力，临床端再由预测场求壁面速度梯度得到 WSS，**梯度误差相对速度 NMAE 会放大多少**？作者是否做过这条后处理链路？
- 点分布偏向内部、远离壁面时，近壁速度梯度样本不足，是否会使 WSS 精度比速度/压力**更差一个量级**？
- 若临床目标是 WSS 或 OSI，是否应**保留更多壁面/近壁点**（而非追求全域均匀）？均匀化是否只适合 v/p 主线、不适合 WSS 终点？

与我们项目的对应：

- V1 `A-*-wss-multi` 说明「有 WSS 监督」不等于 WSS 可学，采样口径是关键变量；
- `V3P-F-30a-VelSup`、`V3P-F-30c-VelDiffInfer` 等已验证：**速度场改善 ≠ WSS 改善**，后验梯度/差分推 WSS 不稳；
- V2/V3 改 **wall-rich（~13000 壁面 + ~2000 近壁）**、G4 **(s,θ) 壁面 2D 展开**，都是在把表示重心拉回壁面，而非先均匀化再间接求 WSS；
- G5 **Pa p95 / 区域 top-k** 病例级 surrogate 仍可能成立，但若均匀化抹平热点，病例级指标也会一起退化——不能只借 FR/PD 叙事替代 WSS 临床终点。

## 5. 我们项目路线的真实演变

### 5.1 最早不是直接做 WSS，而是速度场和压力场

这点需要在给老师汇报时讲清楚：我们最早的任务A/V1路线并不是一开始就“退到 WSS”，而是和这篇论文一样，先做了**血流速度场 + 压力场重建**。

V1 冻结卡和实验清单中固定的预测目标是：


| 目标索引    | 目标名       | 含义     |
| ------- | --------- | ------ |
| `[0:3]` | `u, v, w` | 三维速度分量 |
| `[3]`   | `p`       | 压力     |


当时的主指标也不是 WSS，而是：

- `interior.RMSE_|v|`：内部节点速度模长误差；
- `R2_p / RMSE_p`：压力预测；
- 分区域速度误差：wall / near_wall / high_curvature / bifurcation / trunk 等。

V1 结果说明了两件事：

1. **压力是相对容易学的**：baseline 和优化线中 `R2_p` 多数能到 `~0.92` 档。
2. **速度场不是完全没信号，但内部主流区和复杂区域仍然很难**：例如 `A-Main-01` 之后，显式几何能改善速度误差，但速度分量 R² 仍不算高；`A-Opt-05` 作为 V1 母版后，`r2_vel_mag` 约 `0.58`，压力仍约 `0.92`。

所以我们的路线不是“看到这篇论文才想到预测速度/压力”，而是：**我们早期已经完整做过速度/压力场路线，后来发现单纯速度/压力场精度与端到端 WSS/临床指标目标之间仍有落差，才逐步转向壁面 WSS 和病例级/区域级 surrogate。**

### 5.2 为什么后来转向压力 + 壁面 WSS

后续 V2/V3 文档中已经明确写过：如果速度场相关指标难以上升，后续优先服务 **压力场 + 壁面 WSS 快速预测**。这个转向的原因不是速度场不重要，而是：

- 速度场整体预测不等于 WSS 梯度质量好；
- WSS 是壁面切向速度梯度，比分布较平滑的速度/压力更高频；
- 临床端更关心高 WSS 区域、壁面血流动力学和病例级风险 surrogate；
- 直接优化速度场有时会压制 WSS 学习。

V1 的 `A-*-wss-multi` 说明，在 Transformer + 几何 + 场监督基础上加 WSS 头，壁面 WSS R² 可以到 `~0.46` 档；但后续 PointNeXt / wall-rich 采样 / 纯 p+WSS / 弱速度辅助路线并没有稳定复现这一优势，说明问题不是“有没有监督 WSS”这么简单，而是**采样口径、目标权重、backbone、表示方式和边界算子共同决定 WSS 可学性**。

### 5.3 V3 已经专门检验过“加速度是否帮助 WSS”

这也要单独说明，因为这篇论文的核心输出是速度/压力，老师可能会自然问：既然论文速度/压力做得好，我们能不能先预测速度再推 WSS？

我们在 V3 中已经做过多条相关验证：


| 实验 / 路线                  | 问题                          | 关键结果                                                                       | 结论                       |
| ------------------------ | --------------------------- | -------------------------------------------------------------------------- | ------------------------ |
| `V3P-Probe-VWSS-01`      | 速度上下文是否帮助 WSS               | `wss_r2_wss=0.343`，低于 WSS 单目标 `0.397`；`r2_vel_mag=0.074`，也低于速度单目标 `0.294`  | **速度监督同时压低两目标**，主线不含速度监督 |
| `V3P-F-30a-VelSup`       | 加内部速度/壁面 no-slip 监督是否改善 WSS | best_wss `wss_r2_wss=0.364`，比 4957 母版 `0.399` 低 `0.035`；速度场本身有改善           | **速度改善不等于 WSS 改善**       |

因此，这篇论文对我们最有价值的启发不是“重新回去把速度场做大”，而是：

- 可以借鉴它的物理约束和点云预处理思想；
- 但不能假设速度/压力预测好了，WSS 自然就好了；
- 对我们当前瓶颈来说，WSS 需要自己的表示、采样、目标和评估口径。

## 7. 对我们有用的启发

### 7.1 物理约束不是单纯加特征，而是直接约束输出场

论文的 PINNs 模块不是把物理量 concat 到输入，而是让预测场满足：

- 连续性方程；
- 动量方程；
- 壁面 no-slip。

这对我们当前的 I18 PDE 残差正则、G2 物理残差、G4 2D 壁面算子有参考价值。不过要注意：论文约束的是速度/压力场，而我们如果直接约束 WSS，需要非常小心近壁梯度、法向、壁面插值和离散噪声。

建议：

- 不要直接把这篇论文理解为“PDE loss 一定能救 WSS”；
- 可以先做 GT PDE/边界残差闭合审计；
- 若速度/压力预测场不够光滑，WSS 梯度约束可能放大噪声；
- 在 G4 规则网格上做物理残差可能比在非结构点云上更可控。

### 7.2 点云均匀性是一个值得记录的变量

论文最有意思的一点是：它没有只换模型，而是强调点云预处理。我们当前也有类似问题：

- 近壁点是否过密；
- 壁面点/内点比例是否合理；
- graph 重建是否改变训练分布；
- 2D 展开 occupancy 是否足够；
- 高 WSS 区域是否被采样稀释。

建议在我们后续文档/实验中补充类似统计：

- 每 case 壁面点数、近壁点数、内点数；
- 采样后高 WSS top-k 覆盖率；
- G4 `(s,θ)` grid occupancy；
- 2D->3D 反映射后高峰是否被平滑；
- 训练点分布和评估点分布是否一致。

### 7.3 论文支持“病例级/临床指标叙事”

论文并没有只停留在点级 NMAE，而是计算 FR 和 PD，与 CFD 做一致性。这与我们 G5 的病例级 Pa p95 和区域 surrogate 非常一致。

对我们来说，如果点级 WSS 长期只能从 0.425 小幅提升到 0.45 左右，但病例级 Pa p95、top-k 高 WSS 区域、热点定位明显改善，仍然可以形成论文叙事：

> 模型不一定完美重建每个点的 WSS 向量，但能稳定捕捉临床更关心的高风险区域或病例级 surrogate。

这篇论文可以作为“从预测场进一步计算临床指标”的外部叙事参考。

### 7.4 它间接支持我们转向 G4 2D 壁面表示

论文通过点云均匀化说明：数据表示会显著影响训练稳定性。我们当前 G4 的基本逻辑是：

- WSS 是壁面上的场；
- 壁面场在 `(s,θ)` 规则域中更低秩；
- F0 `pod_2d = 0.672` 已经说明 target-side 表示天花板更高；
- 单 case 2D U-Net 过拟合 R² 0.954 说明链路可训练。

论文没有做 G4，但它支持“不要只在原始 CFD 点云上硬学，应该重新设计点云/场表达”的思路。

## 8. 不能从论文直接搬过来的地方

### 8.1 不能把它的 NMAE 直接和我们的 WSS R² 对比

论文的 `7.79% velocity NMAE`、`6.63% pressure NMAE` 是速度/压力指标，不是 WSS。

WSS 是速度在壁面法向上的梯度，往往比速度本身更高频、更敏感：

```text
WSS = μ · ∂u_t / ∂n | wall
```

因此，速度场预测看起来很像 CFD，不代表 WSS 点级预测一定好。

### 8.2 不能简单照搬“均匀重采样”

对速度/压力，均匀重采样可能有助于整体空间学习；但对 WSS，壁面和近壁梯度是关键信息。过度均匀化可能：

- 稀释壁面点；
- 平滑高 WSS 峰；
- 损失边界层信息；
- 降低局部极值和热点定位能力。

如果借鉴论文做预处理，必须单独检查 WSS top-k 和近壁梯度保真。

### 8.3 不能直接把 PINNs loss 加到现有 WSS head 上

论文的 PINNs 是对 `Vx/Vy/Vz/P` 求空间导数，再算 N-S 残差。我们当前如果只预测 WSS，没有完整速度场，就无法直接套同样的 PDE loss。

可行方向更像：

- 先预测速度/压力，再由物理/几何算子得到 WSS；
- 或在 G4 中对壁面 WSS 加局部物理一致性约束；
- 或用压力梯度、近壁速度差分、法向 ray 作为结构化输入；
- 或做残差学习：网络只学物理基线无法解释的部分。

## 10. 对我们后续工作的具体建议

### 10.1 短期：把论文作为“外部对照”，不要改当前主线

当前主线已经在 G4 Phase 2 Probe，且 I 系已经形成 oracle-first 队列。建议不要因为这篇论文立刻打断现有路线。

短期可做：

- 在汇报中新增一页“相关工作对照：Liao 2025”；
- 强调它证明有限数据下 PINNs + 点云预处理可行；
- 同时指出它未直接解决 WSS；
- 用它支撑我们转向“物理约束 + 表示换轨 + 临床指标”的合理性。

### 10.2 中期：给 G4/I 系增加两个低成本诊断

建议补两个不需要大量 GPU 的诊断：

1. **G4 grid 采样保真诊断**
  检查 `(s,θ)` 网格化前后，高 WSS top-k 点是否被保留，峰值是否被平滑。
2. **WSS vs velocity/pressure 可解释性诊断**
  用现有 GT 速度/压力/压力梯度/几何特征做 probe，看它们对 `τ_s/τ_θ` 或 global `wss_x/y` 的解释力。这个问题和论文 PINNs 方向直接相关。

### 10.3 如果要借鉴 PINNs，优先做“闭合审计”，不是直接上 loss

建议顺序：

1. 先在 GT 场上验证局部 PDE/边界残差是否闭合；
2. 再检查离散点云/2D 网格上的导数估计是否稳定；
3. 再做小样本过拟合；
4. 最后才进入正式训练。

这和当前 I18 的最终审查口径一致：PDE 残差正则有价值，但不能直接加。

## 11. 可放进汇报的简短表述

> Liao et al. 2025 在 51 例脑血管 CFD 数据上，用 PointNet-style 点云网络、voxel-based 均匀重采样和 PINNs 物理约束预测三维速度/压力场，40 例训练、11 例测试，达到 velocity NMAE 7.79% 和 pressure NMAE 6.63%，并能进一步计算 FR/PD。这个方向和我们项目早期 V1 路线高度相似：我们最早也是做 `u/v/w/p` 血流场重建，并取得了压力 `R2_p~0.92`、速度中等可学的结果；但后续 V2/V3 实验证明，速度/压力场做好并不自动带来 WSS 点级提升，速度监督甚至多次压低 WSS。因此该论文不能直接对标我们当前 WSS R² 瓶颈；它更适合作为“有限数据下点云表示 + 物理约束 + 临床 surrogate”的相关工作依据。对本项目的主要启发是：继续坚持壁面表示换轨、物理一致性审计和病例级/区域级指标叙事，而不是简单回到原始速度场预测。

## 12. 结论

这篇论文对我们有参考价值，而且老师说“和你现在工作类似”是合理的：它确实也是脑血管 CFD 场预测、小样本、点云、物理约束。

但阅读时要抓住边界：

- 它解决的是速度/压力场，不是直接解决 WSS；这一点和我们 V1 初始路线相似，但和当前 V3 主瓶颈不同；
- 它的“精度不错”主要来自 NMAE，不是我们当前用的点级 WSS R²；
- 我们内部已经验证过“加速度监督/速度差分/近壁速度上下文”不能稳定提高 WSS；
- 它支持我们的 G4/G5/I18 等方向，但不能替代我们自己的 WSS 实验证据；
- 最值得学习的是“有限数据下，数据表达和物理约束必须一起设计”。

对当前项目的下一步判断：**继续按现有 G4 Phase 2 + I 系 oracle 队列推进，不建议临时改主线；但可以把这篇论文纳入相关工作和汇报材料，作为有限数据脑血管 PINNs/点云预测的直接对照文献。**
